Language Description
XPath standard XML Path language, uses a "traditional" language to navigate through the XML
XSLT (eXtensible Stylesheet Language Transformation) transforms and styles XML to HTML
XQuery finds and extracts elements and attributes from XML documents (superset of XPath)
XLink creates hyperlinks in XML documents
XPointer allows links to point to specific parts of an XML document
XML DTD (Document Type Definition) describes the structure of an XML document (same as Schema but different syntax)
XML Schema describes the structure of an XML document (same as DTD but different syntax)

Attribute / Method Description
namespace
<tag xmlns="URI"> </tag> defines a default namespace, the URI is the identifier of the namespace (all children element belong to this namespace) (attributes do not inherit default namespaces!)
<tag xmlns:prefix="URI"></tag> defines a prefixed namespace, the URI is the identifier of the namespace (only elements or attributes that has this attribute belong to this namespace)
parse / serialize DOM
new DOMParser() returns an object which has the ability to parse a string into a DOM
DOMParser.prototype.parseFromString(string, MIME_type) parses a string (DOMString (html, xml, svg)) into a DOM and returns it
new XMLSerializer() returns an object which has the ability to parse a DOM tree into a string
XMLSerializer.prototype.serializeToString(DOM_obj) parses a DOM tree into a string and returns it
PCDATA / CDATA
<![CDATA[unparsed characters]]> entered data ignored by the XML parser (CDATA = unparsed data / PCDATA = parsed data) (only restricted data is ]]> because it ends the element)
XMLDocument.createCDATASection("unparsed characters") creates a CDATA Node (unparsed data by the XML parser) with its value

Description and Demonstration :

- XML (eXtensible Markup Language), is a language for storing and transporting data, it is designed to be both machine and human readable

- XML is a hardware and software independent tool symplifies data storage and exchanges for many IT systems

- XML does not do anything (like data type checking), it simply stores data in plain text (wrapped in tags)

- XML file extension is .xml, the basic MIME type is text/xml but because XML can be presented in many formats there are many other MIME types (like application/xhtml-xml and many others)

- XML is a W3C recommendatin since 1998 

SYNTAX :    <?xml version="1.0" encoding="UTF-8"?>                <-- prolog must come first but it is optional, it should contain the character encoding to avoid errors (UTF-8 is the default)  
            <rootelement>                                         <-- required only one root element!
              // elements --------------------------------------
                <tag attr="value"> text </tag>                    <-- elements can have text content, attributes and other elements
                <tag />                                           <-- must have proper open and closing tags (except the prolog), this is a valid "self-closing tag" (element with no content)
                <Tag> text </Tag>                                 <-- elements are case sensitive (<tag></tag> is different from <Tag></Tag>)
                <_2a-.> text </_2a-.>                             <-- elements can contain a letters (any) / 0 digits / _ underscores / - hyphens / . periods / (spaces are not allowed)
                <_tag> text </_tag>                               <-- elements must start with a letter or an _ underscore
                <xmltag> text <xml/tag>                           <-- elements cannot start with xml (this is illegal) 
              // attributes ------------------------------------
                <tag attr="value"> text </tag>                    <-- attributes must have attribute/value paris, (booleans are NOT allowed)
                <tag attr='value'> text </tag>                    <-- attribute values must be properly quoted (" double or ' single quotes)
                <tag attr='&tl;'> text </tag>                     <-- entities can be used as attribute values
                <tag attr="val" attr="val"> text </tag>           <-- same attribute identifiers in the same element are NOT allowed (this is illegal)
              // nesting ---------------------------------------
                <tag>
                    <chTag> text <chTag>                          <-- all elements must be properly nested
                </tag>
              // comments --------------------------------------
                <!-- comment -->                                  <-- comments are NOT parsed by the XML parser  
              // text content  ---------------------------------            
                <tag>                                             <-- all white spaces are preserved (both significant (inside element) and insignificant (arround elements)) and new lines are registered as "LF" (linefeed)  
                    some text    some text                        (handling whites spaces is the responsability of the application which uses the XML file but XML preserves all white spaces!)
                       some text <chTag>chText</chTag>           
                </tag>
            </rootelement>

- unlike HTML, XML tags are not defined, the author "invets" them and the XML Syntax is very strict compared to HTML 

    <?xml version="1.0" encoding="UTF-8"?>
    <biography>
      // elements -------------
        <_name-21.2> Robert </_name-21.2>                         // tagname started with underscore, (these characters are authorised but only letter or underscore can be used as first character)   
        <selfClose />                                             // self-closing tag (camelCase used, remember tag names are case sensitif)
        <xmlStuff></xmlStuff>                                     // -> this is NOT allowed tag cannot start with "xml"
      // attributes -----------
        <pet dog="l'amour&amp;"> pottyos </pet>                   // combine quotes + entity used in attribute value
        <pet dog></pet>                                           // -> this is NOT allowed boolean attribute will fail
      // nesting --------------
        <b><u> mancs </b></u>                                     // -> this is NOT allowed elements must be properly nested (this might work in HTML but not in XML)
      // comment --------------
        <!-- comments go here -->
      // text content ---------
        <structured>                                              // white spaces are preserved (might not shown by the browser but XML preserves all white spaces!)
            apple       2.19
            pear        3.25
            plum        5.21
        </structured>
    </biography>

using entities in XML

- in XML only 5 entity references are allowed, but any entity numbers can be used no problem

- entities in tag names are NOT allowed! 
SYNTAX :    <   &lt;        less than                             <-- strictly illegal
            &   &amp;       ampersand                             <-- strictly illegal
            >   &gt;        greater than                          <-- not illegal but recommended to replace 
            '   &apos;      apostrophe (single quote)
            "   &quot;      quotation mark
    
    <?xml version="1.0" encoding="UTF-8"?>
    <rootElement>
        <testEntity> &lt; &gt; &amp; &apos; &quot; </testEntity>  // pre-defined entitites references used 
        <testEntity2> &#169; &#174; &#182; &#38; <testEntity2>    // using entity numbers 
        <&lt;></&lt;>                                                    // !! this is illegal entities cannot be used in tags (parser will throw an error)
    </rootElement>

name spaces

- namespaces (sort of scopes) useful when we mix multiple xml source in the same document

- in XML there are 2 type of namespaces are allowed to avoid name conflicts, default and perfixed namespace 

SYNTAX :  // default namespace ---------------------------------
            <?xml version="1.0" encoding="UTF-8"?>
            <tag xmlns="URI_1">                                   <-- this element and all its children elements belong to the URI_1 namespaces
                <tag attr="val"> text </tag>                      <-- attributes do NOT inherit default namespaces (careful) 
                <tag xmlns="URI_2 >                               <-- this element and all its children elements belong to the URI_2 namespaces
                    <tag> text </tag>
                    <tag> text </tag>
                </tag>
            </tag>
          // prefixed namespace --------------------------------  
            <?xml version="1.0" encoding="UTF-8"?>
            <tag xmlns:prefix_A="URI_1">                          <-- prefix namespace is declared 
                <prefix_A:tag attr="val"> text </prefix_A:tag>    <-- this element is in the URI_1 namespace but the attribute is NOT! (ending tags are prefixed as well!)
                <tag prefix_A:attr="val"> text </tag>             <-- only the attribute is in the namespace URI_1 
                <prefix_A:tag prefix_A:attr="val"> text </prefix_A:tag>    <-- both the element and the attribute are in the namespace URI_1 
            </tag>

- the URI is an identifier what makes the namespace unique (not parsed by the parser)

    <?xml version="1.0" encoding="UTF-8"?>
    <dogs xmlns="http://white" xmlns:black="http://black">        // -> "http://white" default namespace declared               // -> "http://black" prefixed namespace declared
        <dog1 origin="europe"> peppy </dog1>                      // children element belong to the "default" namespace         // !! but its attribue doesn't (it has no namespace declared)
        <black:dog2> bodry                                        // prefixed element belong to the "prefixed" namespace 
            <size> small </size>                                  // this children element belongs the the "default" namespace  // -! there is no prefix inheritage!
        </black:dog2>  
        <dog3 black:origin="usa"> puffy </dog3>                   // the element belongs to the default namespace               // but the attribute belongs to the prefixed one     
    </dogs>

parse / serialize DOM 

SYNTAX :    new DOMParser()                                       // returns an object which has the ability to parse a string into a DOM
            DOMParser.prototype.parseFromString(string, MIME_type)  // parses a string (DOMString) into a DOM and returns it 
                - string : string containing DOM Source 
                - MIME_type : type to parse too (text/html / text/xml / image/svg+xml)
            
            new XMLSerializer()                                   // returns an object which has the ability to parse a DOM tree into a string
            XMLSerializer.prototype.serializeToString(DOM_obj)    // parses a DOM tree into a string and returns it
    
// parse froms string to DOM -----------------    
    var xmlDOMString = "<person><name>Attila</name> <suraname>Peter</suraname> <age /> </person>";        // string contains an XML source
    var domXML = new DOMParser().parseFromString(xmlDOMString, "text/xml");                               // -> XML DOM returned
    
// parse DOM to string -----------------------   
    new XMLSerializer().serializeToString(domXML);                              
        // -> "<person><name>Attila</name> <suraname>Peter</suraname> <age /> </person>"                  // DOM tree parsed back to string

Parsed VS Unparsed Data PCDATA / CDATA

- PCDATA (Parsed Character Data) = text parsed by the XML parser

- CDATA (Unparsed Character Data) = text NOT parsed by the XML parser

SYNTAX :    <![CDATA[unparsed characters]]>                       // entered data ignored by the XML parser (CDATA = unparsed data / PCDATA = parsed data) (only restricted data is ]]> because it ends the element)
            XMLDocument.createCDATASection("unparsed characters") // creates a CDATA Node (unparsed data by the XML parser) with its value 
    
// XML source ------------------
    <?xml version='1.0' encoding='UTF-8'?>
    <person>
        <name>Karsai</name>
        <![CDATA[ unparsed data = <<l;kj;lk__`~ ]]>               // data not parsed 
    </person>
    
// JavaScript ------------------
    var myCdata = XMLDom.createCDATASection(" unparsed data = <<l;kj;lk__`~ ");   // creates a CDATA node (same as in the XML source code)
    XMLDom.getElementsByTagName("person")[0].appendChild(myCdata);                // append the CDATA node
