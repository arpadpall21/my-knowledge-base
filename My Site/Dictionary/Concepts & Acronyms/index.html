General
abstraction - hiding (complex) system parts, so on a higher level only essential parts are exposed
- that's why we say 'abstraction over a system|API|ect...'
agile developement - breaking the developement in pieces (sprints) so all these spirints can be developed separately
- Scrum, Kanban, XP (Extreme Programming)
agnostic - the ability to work with an underlying system without knowing anything about its implementation

APIPA - Automatic Private IP Addressing
ARIA - Accessible Rich Internet Applications
ASGI - Asynchronous Server Gateway Interface
atomic - multiple operations that are undivisible or uninterruptible, threated as single unit without any interference
- like database transactions, concurrent tasks that maintain data consistency and prevent race conditions
big ball of mud - entangled code
binary (code) - executable binary file which is not readable by human (compiled program)
black box testing VS white box testing - black box testing -> test that only considers the external behavior of the system
- white box testing -> test that considers the internal behavior of the system

BLOB - Binary Large OBject

business intelligence - collecting -> analysing -> parsing data in order to help business make better decisions
CA
RCA

- Root Certificate Authority (highest level of trust and authority in the certificate hierarchy, issues digital certificates, either directly to entities or to intermediate Certificate Authorities)
- Intermediate Certificate Authority (operates under the trust and authority of a Root Certificate Authority (Root CA))

CDN - Content Delivery Network
CI - Continous Integration

CMS - Content Management System
code golf - express a code in a shortes possible way
compiling - language-to-language translations in a deep level
concurrency VS parallelism - concurrency -> single thread that is passed where execution needed and freed up where is not (using single cpu) (async/await style programming)
- parallelism -> running multiple threads at the same time (using multiple cpu)
configuration drift - unintended changes in configuration settings caused by human error, software update, hardware upgrade or other factors
copy-on-write (CoW) - make copy link reference without creating a new file (if the new file is modified then it is created)
compiler - program what turns human readable code into machine readable code (soruce code to binary(executable))
CRUD - Create Read Update Delete (basic storage operations)
CSS - Cascading Style Sheets
CSV - Comma Separated Variables
daemon / service - background process (service) on Unix-like systems
DAO - Data Access Object (ORM term, one record modelized as object)
- (in SQL ORMs usually represents a single table row) (in NoSQL usually represents a document)
DDL - Data Definition Language (in SQL this is basically a query which instructs the databse to do some operation like CREATE TABLE ...)
debouncing - limit (regulate) a frequently called task in order to gain performance
declarative programming VS imperative programming - declarative programming -> we are passing commands then the program handles the logic
- imperative programming -> we are handling the logic
dependency - relation between two modules/components/libraries where one of them depends on the other to function
DevOps - Developement Operations (from developement to deploying)
dinamically typed language - at compile time there's no type checking (variables are typeless)
distributed system - system whose components are located on different networked computers
DLL - Dynamic Link Library
DOM - Document Object Model
encapsulation - hiding the actual data from the public (like in a class) then creating methods and conditions to access/modify this data
- so the data is not directly accessible to the public but only the methods that access/modify this data
endianness - CPU sequential byte order to represent a number (big endianness (most significants first) 0x00 0xFF = stored as 0x00 0xFF) (little endianness (less significant first) 0x00 0xFF = stored as 0xFF 0x00)
executable - a file containing an executable script or program (.exe executed by C++, .ps1 executed by PowerShell)
ECC memory - Error-Correcting Code memory
fault tolerance - system continues to function when a part of it fails and can recover from failure (ex: service error, disk falutre, etc..)
federated - a group of systems joined together to form a single one
FHS - Filesystem Hierarchy Standard
FIFO - First In First Out (items retrieved the same order they entered the pipe)
file handle - temporary file name or identifier assigned to an open file which is used by the operating system
FOSS - Free Open Source Sortware (OS Standard like Linux, Android, Fedora, Debian)
FQDN - Fully Qualified Domain Name (Fully location in the DNS hierarchy)
framework - a fully structured tool that dictates the way how the app should be build (Django)
garbage collection - removing values that are not referenced my any variable
GIL - Global Interpreter Lock
globbing - targeting files on the file system
GUI - Graphical User Interface
happy path - happens everithyng as expected
- we mainly using this concept in testing, fist we make sure that we have a 'happy path' then start testing errors and edge casses
hook - place in code that allows to tap in the API ('hook into the code')
- through a hook we can alter the behaviour of the API in question
hot swappable - hardware can be swapped while the computer is running (without stoping the computer)
HTML - Hyper Text Markup Language
hypervisor - virtual machine manager, (creates, runs and monitors virtual machines)
IEEE - Institute of Electrical and Electronics Engineers (global organization that works on electonic device inovations)
IETF - Internet Engineering Task Force (open international community that develops and defines standards for internet technologies)
IDE - Intergrated Development Environment
idempotent - operation that can be applied multiple times without changing the result after the first operation
- ex: a PUT request updates the resource on the server once called, repeating this operation won't have any effect (after the first one)
intrinsic / extrinsic (in CSS) - intisic = element is sized based on its content
- extrinsic = element is sized based on its parent
IPC - Inter Process Communication
IT - Information Technology
JSONP - JavaScript Object Notation with Padding
- technology used 2005 - 2009 (before CORS) allowed to for <script> tags to request cross-origin resources if they serve JSON data bypassing same-origin policy
- the idea was that JSON cannot run by the browser
JSX - JavaScript XML
kernel (OS) - closest software layer to the hardware
KISS (principle) - Keep It Simple and Stupid
LAMP stack - Linux Apache MySQL PHP
library - a collection of pre-written code offering functionalities
load factor - a number from 0 to 1 indicating how empty of full something is
- (ex: 0.1 = 10% full / 0 = empty / 1 = full / 0.52 = 52% full)
LRU - Last Recently Used (a stack that sorts most recently used items on top)
- We mainly use this concept for LRU Caching (for fast recently used items access)
LTS - Long Term Support (Sofware version supported for long period of time)
MEAN stack - MongoDb Express.js Angular Node.js
message broker - service that facilitates communication between multiple applications or systems by receiving, routing, and transmitting messages between them
memory leak (memory loss) - badly allocated memory (RAM) which is not used or cannot be accessed
metadata - data about data
meta programming - programming the programming language
microservice architecture - software architecture made of multiple independent 'micro' modules (services) that are loosely-coupled (monolitic VS microservice)
MIME - Multipurpose Internet Mail Extensions (indicates the nature and format of a document or file)
minifier - program that reduces file size
MIT (license) - Massachusetts Institute of Technology (permissive free software license)
mixin (mix-in) - API or class which methods and properties are not directly implemented
- on runtime creates (mixes) an object, class, etc... from the intended sources
- (ex: the fetch API has the Body mixin implemented on both Request and Response, so the Body mixin looks like inherits from both classes)
mocking - faking a specific behaviour (like replacing a 'mocked' function with the original)
monolitic architecture - software architecture made of one large block (monolitic VS microservice)
namespace - container or scope
NP-complete (problem) - Nondeterministic Polynomial
- complete = already tested
- hard problem which reqires considerable resources to solve (like the travelling sales person problem)
opaque response - Cross-Origin response with no CORS (headers)
Operating System - software between the hardware and the user software
opinionated VS unopinionated - opinionated -> rigide structure that imposes to do thinks in a certain way (structure over configurability)
- unopinionated -> flexible structure that allows to do things in different ways (configurability over structure)
optimization problem - a problem which requires the best solution (finding the minimal or maximal answer)
orchestrator - central control point service that manages multiple tasks/services
ORM - Object Relational Mapping (technique for converting data between incompatible systems using object-oriented approach)
package - chunk of code (ready to install program)
parser - reads a piece of text and produces a data structure
PEP - Python Enhancement Proposal
- document providing information to the Python community, or describing a new feature for Python or its processes or environment
pointer - a memory address that points to a data
polymorphism - overwriting properties/methods in the inheritance chain
polling (event handler system) - program put the event in an action queue than periodically check the queue and executes it
portable / standalone aplication ready to run program (does not need to be installed)
POSIX - Portable Operating System Interface (OS standard specified by IEEE)
- POSIX compliant apps will run on any POSIX compliant system
predecessor - the pervious element in a hierarchy (ex: hierarchy A -> B -> C the predecessor of C is B etc...)
preflight request - a CORS pre request in order to figure out what kind of access the third party server provides for cross-origin requests
- an OPTION HTTP request with Access-Control-Request-Method Access-Control-Request-Headers and Origin headers

- the goal is to verify the validity of an idea before investing more significat resources in it
proprietary software - owned sofrware (not open-source)
pure function - function that has no side effect and always returns the same value for the given input
PWA - Progressive Web Application (Web Application progressively enhanced)

repository "repo" - central storage location for packages, metadata, ect...
REST API - REpresentation State Transfer

- (specification documents (IETF) Internet Engineering Task Force)

- specifications and organizational notes about topics related to the internet and computer networking, such as routing, addressing and transport technologies

scalar - single value, (primitive) (not an object or some similar complex type)
scripting programming language - script runs directly (does not need to be compiled) (interpreted by a program)
SDL - Schema Definition Language (language tha works through schema (ex: GraphQL))
SEO - Seach Engine Optimization

- vertical sharding (scaling) upgrading the capacity of a single server
- horizontal sharding (scaling) distirbuting the dataset among multiple servers

sourcecode human readable program (the program before compiled)
sourcemapping host recognizes the original file through a compiled or transpiled file
SPA - Single Page Application
spying - spying on a specific behaviour (like spying on a function if it was called or not)

stabbing - faking a specific content (like faking some kind of content that exists)
stale VS fresh stale -> outdated content
fesh -> up to date content
statefull - unit having inernal storage (state) so it can remember previous states
stateless - unit not having internal storage (state), so it cannot remember any previous states (restarts default state every time it started)
successor - the next element in a hierarchy (ex: hierarchy A -> B -> C the successor of A is B etc...)
swapping (in memory management) - moving processes out|in the RAM to|from the disk to give space to other processes
syntax sugar - easier to read syntax (code)
tarball - Linux Archive file
TDD - Test Driven Developement (Writing the test first then the Program)

TLD - Top Level Domain (Server)

tree shaking - dead code elimination (dead code removal)

TTY - TeleTYpe (referes to the terminal)
type casting - data type conversion (usually between 2 programming languages)
UEFI - Unified Extensible Firmware Interface
UI - User Interface
unit testing - testing a part of a software
upsert - Insert (if necessary) + Update
vendor lock-in - customer becomes dependent on a vendor for products and services, making it difficult to switch to another vendor without incurring significant costs or inconvenience
vertext ('vertices' for multiple) - a point represented in 2D or 3D space
warehousing data - the process of collecting organizing and managing large data
waterfall developement - sequential software developement
wildcard character - character that represent one or more other characters (ex: *)

Cloud

elasiticity - scale up and down as you need

provisioning - allocating and configuring cloud resources
SaaS - (Software as a Service) service that handles the application (application, data)

serverless - the server architecture is handeled by the platform (ex: AWS) we just write the code for the service

Security & Cryptography
ACL - Access Control List
AEAD - Authenticated Encryption Associated Data
- encryption mode where we can pass additional data which also provides authenticity to the encryption
AES - Advanced Encryption Standard (commonly used encryption in TLS)
authenticated encrytion - encryption which also provides authentication (in addition of secrecy)
- like an encryption mode (block cipher mode) (ex: GCM, CCM, OCB, etc...)
authenticity - making sure that the message comes from the intended sender
block cypher - encrypting one block at the time (ex: takes one 8 byte block at the time to encrypt, if the block is not a full 8 byte it will be padded)
CORS - Cross Origin Resource Sharing
CRC - Cryclic Redundancy Check (algorithm that generates a checksum for data integrity check, it's not suitable for cryptograpich authentication)
- same aproach as creating checksums but parses the data on bit level (the following number defines the strength and the output length in bits (ex: CRC-32 generates a 32 bit checksum))
CSR - Certificate Signing Request (SSL Certificate Signing Request)
CSRF - Cross-Site Request Forgery (token to prevent cross-site attacks (mostly used in forms))
cypher - encryption algorithm
cypher suite - a set of algoritms that describes which algoritms should be used to secure network communication (ex: )
DDoS - Distributed Denial of Service (attack)
- overloading a server with requests from multiple sources
DHE - Diffie Hellman (Perfect Forward Secrecy algorithm) (the additional 'E' is appended for traditional abbreviation)
- Key exchange protocol
ECDHE - Elliptic Curve Diffie Hellman (Perfect Forward Secrecy algorithm) (the additional 'E' is appended for traditional abbreviation)
- key exchange protocol
ephemeral (TLS) - new certificate-key pair is used for each TLS session
fingerprint - a unique identifier that is generated from a block of data using a specific algorithm (unique to a particular set of data)
FIPS - Federal Information Processing Standard
- it's a US government security standard for cryptographic modules (security standard for Federal systems)
hash / checksum - encrypted key generated with the original file
- it can be used by the client (which pairs the checksum and the original file) to verify that the original file has not been modified
HMAC - keyed-Hash Message Authentication Code (superset of MAC where we add a pirvate key on top of MAC)
HSM - Hardware Security Module (a physical device that manages security solutions, like cryptographic key generator, etc...)
- ex: AWS CloudHSM is a service that provides a cloud-based physical hardware security module
identity provider (IdP) - trusted 3rd party system that provides authentication services
initialization vector / Nonce - some data which is added to a key in order to generate a new unique key
- allows the reuse of the same
integrity - making sure that the message has not been changed between the two parties
least privilege access - give just enough priviledges to to perform the desired tasks (no more)
MAC - Message Authentication Code (basically the message checksum)
OCSP - Online Certificate Status Protocol (obtaining the revocation status from CA of an x509 digital certificate)
PFS - Perfect Forward Secrecy (for each TLS session a new certificate-key pair is used)
PKI - Public/Private Key Infrastructure
PSK - Pre Shared Key (pre shared keys are used for Symmetric Encryption)
RSA - Rivest Samir Adleman (3 dudes who invented a protocol that provides authenticity to the Diffie Hellman key exchange (sign/verify the exchanged public keys))
salting - adding some data (pading the data) to the original data then hashing it, this way the generated hash is even harder to guess
SHA - Secure Hash Algorithm
- an algorithm that generates a unique hash (checksum, digest) by using the data
- SHA families MD5 SHA-0|1|2|3
SNI - Server Name Indication (TLS extension) (same server used for multiple hostnames, the server can present different SSL certificates and different content)
SPKAC - Signed Public Key And Challenge (Netscape SPKI)
- encryption used by old browsers (developend by Netscape) (used the HTML5.2 deprecated <keygen> element)
SSL - Secure Sockets Layer
SSO - Single Sign On (authentication scheme that allows a user to log in with a sigle ID into serveral independed systems)
stream cypher - encrypting one bit or byte at the time (encrypting the data as it streams)
Trust Store / Root Store - Collection of trusted Root Certificats what the client uses in order to verify the server's trust
TLS - Transport Layer Security
XSS - Cross-Site Scripting
