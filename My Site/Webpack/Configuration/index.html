Compilation-level

Chunk-level
[id] - ID of the chunk

[chunkhash] - hash of the chunk, including all elements of the chunk
[contenthash] - hash of the chunk, including only elements of this content type (affected by optimization.realContentHash)
Module-level
[id] - ID of the module
[hash] - hash of the module
[contenthash] - hash of the content of the module
File-level
[file] - filename and path, without query or fragment
[query] - query with leading ?
[fragment] - fragment with leading #
[base] - only filename (including extensions), without path
[path] - only path, without filename
[name] - only filename without extension or path
[ext] - extension with leading
URL-level
[url] - URL

Webpack Config (webpack.js.org/configuration/)

Remember This :

Description and Demonstration :

- we can configure Webpack through the config file which is by default webpack.config.js (other config files can be used on build time through the --config flag)

- from v4 Webpack does not require a config file, by Default the entry is src/index.js, the output is dist/main.js and the output code is optimized for production (minimized, etc...) 

// webpack config file ----------------------------------------------------------------
    
    module.export = { // webpack options... }                                          // exports an object which holds webpack configurations    
    
    // function -----------------------------------------------------------------------
    module.export = function(env) {                                                    // the env receives the key/value pair passed through the --env flag   
        return { // webpack options... }                                               // exports a function which returns an object holding webpack configurations    
    }

I only list options that I understand here because there are crazy amounts of options available ([likend] the original page)

Entry and Context [link]

- specifies entry points

Options 
    module.exports = {
        context: filePath:str                                                          // directory where entry files are looked for (must be an absolute path) (Default: current directory)   
        
        entry: path:str|obj|arr|fn

            
 - str - a file specified as entry point 

                

            
 - arr - array contains multiple entry points 

            
 all these entries are treated as one entry (webpack will ouptut a single bundle) 

                

            
 - obj - object contains key/value pairs where the values are the entry points 

            
 in this case as many outputs are generated as entry points specified (keys will be the new file names) 

                

            
 - fn - function executed when looking for entries (on make event) 

            
 this is dynamic loading which is NOT an entry point 

        
                                                    // entry (entries) where Webpack starts building the bundle 
    }
Example :

webpack.config.js (file) 

    const { CleanWebpackPlugin } = require('clean-webpack-plugin');  
    
    module.exports = {  
        mode: 'development',
        entry: ['./src/index.js', './src/index_.js', './src/index_2.js'],              // all 3 modules are treated as 1 single top-level module (1 bundle is outputted only!)   
        output: {
            path: __dirname + '/dist',
            filename: '[name].bundle.js'
        }, 
        plugins: [new CleanWebpackPlugin()]
    }

webpack.config.js (file) 

    const { CleanWebpackPlugin } = require('clean-webpack-plugin');  
    
    module.exports = {  
        mode: 'development',
        context: __dirname + '/src',                                                   // context specifies the base directory where Webpack starts looking for entries    
        entry: {
            bl1: './index.js',                                                         // each entry is treated as individual top-level entry point (3 bundles will be outputed)    
            bl2: './index_.js',                                                          // outputted files will be 'bl1.bundle.js', 'bl2.bundle.js' and 'bl3.bundle.js'    
            bl3: './index_2.js'
        },
        output: {
            path: __dirname + '/dist',
            filename: '[name].bundle.js'
        }, 
        plugins: [new CleanWebpackPlugin()]
    }

Output (and output substitusions) [link]

- specifies how and where should output the bundle(s), assets, etc...

- Webpack Substitutions are used in output template strings, Webpack replaces these substitution with specific values on output

Options 
    module.exports = {
        output: {
            charset: bol                                                               // adds charset="utf-8" to the HTML script tag (Default: true)    
            
            
            filename: str|fn                                                           // output bundle name, (substitution required if multiple bundles are outputted ex: [name].bundle.js)    
                                                                                           // can create subdirectories as well (ex: './sub/[name].bundle.js')    
                                                                                           // does not affect dynamically loaded chunks   
            assetModuleFilename: str                                                   // name of each output for asset modules output    
            path: filePath:str                                                         // the output directory as absolute path    
            publicPath: filePath:str                                                   // public directory what the browser will reference to server assets    
            
            
            hashDigest: hash:str

                
 'hex' (Default) | 'base64' | 'latin1' 

            
                                                      // encoding to use when generating a hash 
            hashDigestLength: nr                                                       // the appended hash length (Default: 20)    
            
            
            sourceMapFilename: str                                                     // name of source map files (Defautl: [file].map[query]))    
            
            
            clean: bol                                                                 // fully cleans the output directory's content before emitting the new bundle(s) v5.20.0 
        }
    }
Example :

webpack.config.js (file) 

    module.exports = {  
        mode: 'development',
        entry: {
            in1: './src/index.js',                                                     // there are 2 entries 
            in2: './src/moduleX.js'
        },
        output: {
            path: __dirname + '/dist',
            filename: '[name].bundle.js'                                               // the '[name]' substitusion here will use the entry object keys 
        },                                                                               // outputs 2 files 'in1.bundle.js' and 'in2.bundle.js'  
    }

Mode [link]

- sets Webpack in Development of in Production mode

Options 
    modules.exports = {
        mode: 'development'|'production'|'none'

            
 'developement' 

            
 - process.env.NODE_ENV is set to 'development' 

                

            
 'production' 

            
 - process.env.NODE_ENV is set to 'production' 

            
 - the output code is optimized for performance, minified, etc...) 

            
 - the following plugins are used for production build FlagDependencyUsagePlugin FlagIncludedChunksPlugin,ModuleConcatenationPlugin, NoEmitOnErrorsPlugin and TerserPlugin 

                

            
 'none' 

            
 - opts out any default optimization options 

        
                                       // Development or Production mode 
    }
Example :

webpack.config.js (file) 

    module.exports = {  
        mode: 'development',                                                           // Webpack runs in Development mode 
        entry: {
            in1: './src/index.js', 
            in2: './src/moduleX.js'
        },
        output: {
            path: __dirname + '/dist',
            filename: '[name].bundle.js' 
        }
    }

Module [link]

- defines how different types of modules should be treated

Options 
    module.exports = {
        module: {
            noParse: str|regExp                                                        // matching files won't be parsed and loaded     
            
            rules: arr                                                                 // loaders used to load specified files (see loaders) executed from right-to-left    
        }
    }
Example :

webpack.config.js (file) 

    module.exports = {  
        mode: 'development',
        context: __dirname + '/src',
        entry: function(){ return './index.js'},
        output: {
            path: __dirname + '/dist',
            filename: '[name].js',
        }, 
        module: {
            noParse: /exclude.css/,                                                    // files including 'exclude' won't be loaded 
            rules: [{test:/\.css$/i, use:['style-loader', 'css-loader']}]
        }
    }

index.js (file) 

    import './styles.css';
    import './exclude.css';                                                            // won't be loaded because of the 'noParse' option

Resolve [link]

- defines module resolution

Options 
    module.exports = {
        resolve: {
            alias: obj                                                                 // contains path aliases

            
 - a reference which holds a path 

            
 (key = alias) (value = referenced path)    
            
            
            extensions: arr                                                            // attempts to resolve extensions in the specified order (when no file extension is specified) (Default: ['.wasm', '.mjs', '.js', '.json'])    
            enforceExtension: bol                                                      // requires file extensions in file paths (Default: false) 
            
            
            mainFiles: arr                                                             // list of files used when resolving directories (Default: ['index'])  
            modules: arr

                
 - relative path = will be searched similar to Node.js scan (looks through the current directory and its ancestors) 

                
 - absolute path = only the given directory is searched 

            
                                                              // list of directories to be searched when resolving modules (Default: ['node_modules'])
            
            
            symlinks: bol                                                              // symbolic links are resolved to their real path (Default: true)    
            
            
            restrictions: [str|regExp, ...]                                            // list of files that Webpack prevents to be resolved    
        }
    }
Example :

webpack.config.js (file) 

    const { CleanWebpackPlugin } = require('clean-webpack-plugin');  
    
    module.exports = {  
        mode: 'development',
        resolve: {
            alias: {
                Util: __dirname + '/src/utilities',                                    // path aliases specified 
                Temp: __dirname + '/src/templates'
            }
        },
        context: __dirname + '/src',
        entry: function(){ return './index.js'},
        output: {
            path: __dirname + '/dist',
            filename: '[name].js',
        }, 
        plugins: [new CleanWebpackPlugin()]
    }

index.js (file) 

    import 'Util/index';                                                               // path alias 'Util' resolved to "__dirname + '/src/utilities'"
    import 'Temp/index';                                                               // path alias 'Temp' resolved to "__dirname + '/src/templates'"

webpack.config.js (file) 

    const { CleanWebpackPlugin } = require('clean-webpack-plugin');  
    
    module.exports = {  
        mode: 'development',
        context: __dirname + '/src',
        entry: function(){ return './index.js'},
        output: {
            path: __dirname + '/dist',
            filename: '[name].js',
        }, 
        resolve: {
            mainFiles: ['index', 'main']                                               // if a directory is required these files will be looked for inside 
        },
        plugins: [new CleanWebpackPlugin()]
    }

index.js (file) 

    import './templates'                                                               // resolves to './templates/index' or './templates/main'

Optimization [link]

- from Webpack v4 optimization depends on the chosen mode, but we still can manually configure optimization if we want to

Options 
    module.exports = {
        optimization: {
            minimize: bol                                                              // minimizes the output bundle (Default: false on developement | true production)    
            minimizer: arr                                                             // list of minimizers to use (Default: [TerserPlugin])    
            
            
            nodeEnv: bol|str                                                           // sets the process.env.NODE_ENV value (Default: mode setting)  
        }
    }

Plugins [link]

- specifies Webpack plugins to use on build process

Options 
    const plugin = require(plugin)                                                     // importing the plugin    
    
    module.exports = { 
        plugins: [new plugin() , ...]                                                  // at runtime Webpack uses the plugin  (a new instance is required this way the same plugin can be used with different settings)    
    }

DevServer [link]

- development server (webpack-dev-server) options

Options 
    module.exports = {
        devServer: {
            compress: bol                                                              // gzip compresses everything served 
            
            
            contentBase: dirPath:str|bol                                               // where (static) content should be server from (devServer.publicPath takes precedence over this) (Default: current directory)    
            publicPath: dirPath:str                                                    // bundled files will be available in the browser under this path    
            
            
            host: str                                                                  // host to use (Default: 'localhost')    
            port: nr                                                                   // port to listen to (Default: 8080)    
            index: str                                                                 // filename that considered the index file (Default: index.html)    
            headers: obj                                                               // adds headers to all response 
            
            
            lazy: bol                                                                  // compiles requested bundles only (disables file watching)    
            liveReload: bol                                                            // automatically reloads the page when file changes are detedted (Default: true)    
            
            
            onListening: fn(server)                                                    // executes a function when the dev-server starts listening    
            open: bol                                                                  // opnes the default web browser when the dev-server starts 
            openPage: path:str                                                         // navigates to the specified page when opening the browser    
        }
    }
