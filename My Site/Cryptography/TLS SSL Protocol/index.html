Description and Demonstration :

- the TLS/SSL protocol estabilishes an encrypted connection between two computers (server - client)

- TLS (Transport Layer Security) and SSL (Secure Sockets Layer) are the same (version history: SSLv2 -> SSLv3 -> TLS1 -> TLS1.1 -> TLS1.2 -> TLS1.3)

Perfect Forward Secrecy

- it's an additional (required in v1.3) TLS feature which generates a new key pair for each TLS handshake (so each connection is encrypted with different keys)

Session Renegotiation

- renegotiaton of certain aspects of a TLS session (not allowed in v1.3)

Session Resumption

- a TLS handshake is relatively slow, so there are 2 ways to resume a session

Session Identifiers (NOT supported by v1.3)

- the Server generates a unique ID for each connection and sends it to the client, both the Client and Server save the session state

- when the Client wants to reconnect sends the ID of the saved session state to the Server

- if the Server has the same Session Id they can resume, otherwise the Server will create a new connection

Session Ticket

- the Server encrypts the entire session state and sends it to the client as a 'ticket'

- when the Client reconnects sends the state to the Server (no need for Serve-Side caching)
