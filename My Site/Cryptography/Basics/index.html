Cryptographic Algorithms
Algorithm Description
AES (Advanced Encryption Standard) - symetric encryption algorithm
- replaces the obsolete DES (Data Encryption Standard)
- operates on fixed-size block of data
- supports 128, 192, 256 bit key sizes
DES (Data Encryption Standard) - symetric encryption algorithm (developemd by IBM in 1970 and adopted as a standard by the US government) (obsolete)
- operates on 64 bit blocks and uses 56 bit key
3DES (Triple DES) - enhanced version of DES (kind of obsolete)
- applies DES algorithm 3 times with different keys
|
RSA (Rivest–Shamir–Adleman) - very popular asymetric encryption algorithm
- used for secure communication, digital signatures and encryption
- popular key sizes 1024, 2048, 4096
ECC (Elliptic Curve Cryptography) - asymetric encryption offers good security with short key sizes
- mostly used in systems with limited resources
- popular key sizes 192, 224, 256, 384
DSA (Digital Signature Algorithm) - asymetric encryption mostly used for digital signatures
|
SHA-2 (Secure Hash Algorithm 2) - hash algorithm that includes SHA-224, SHA-256, SHA-384, and SHA-512
- replaces the obsolete HSA-1 algorithm
SHA-1 (Secure Hash Algorithm) - hash algorithm (no longer considered secure)
MD5 (Message Digest Algorithm 5) - hash algorithm (obsolete)
|
DH (Diffie Hellman) - key exchange algorithm
- allows to securely exchange keys over an insecure channel
- commonly used in protocols such as SSL/TLS, VPN, secure messaging
ECDH (Elliptic Curve Diffie Hellman) - does the same as DH with shorter key sizes but providing the same level of security
- useful for systems with limited resources
|
HMAC (Hash-based Message Authentication Code) - it's a technique (not algorithm) for hasing a message in order to achieve authenticity
- uses a hash algorithm and a cryptographic key

Symmetric VS Asymmetric encryption

- Symmetric encryption uses one "symmetric" key for encryption and decryption (fast encryption)

- Asymmetric encryption uses 2 keys, public key for encryption and private key for decryption (slow encryptions) (the public key can only encrypt the data but cannot decrypt it)

Sign & Verify

- This method assures the integrity and authenticity of the data

- The sender signs the data with the private key

- The receiver verifies the signed data with the public key or certificate

Hashing

- Hashing is a one-way encryption technique, meaning we pass the content to a hash algorithm (like SHA-256) which outputs an encrypted result (that's the hash or also called checksum or digest)

- Hashing Characteristics:

- the generated has cannot be decrypted to get the original data back (one way encryption)

- the generated hash has a fixed length no matter the data size (length defined by the hasing algorithm)

- a small change in the original data drastically changes the hash (ex: changing a single character in the data will change at least 50 of the hash)

Salting

- the salt is basically a secret that is added to the original data before hashing

- we keep the salt in secret and it's needed in order to get the same hash result

Key Exchange Protocols DH / ECDH / RSA

- exchange keys between pairs while the man in the middle cannot discover the exchanged keys

- DH (Diffie Hellman) is a very popular key exchange protocol

- ECDH (Elliptic Curve Diffie Hellman) is the same protocol as the DH but internaly uses an elliptic curve (basically it's more secure and CPU efficient than DH)

- RSA (Rivest Samir Adleman) provides authenticity to the Diffie Hellman key exchange (sign/verify the exchanged public keys)

Digital Certificate

- a digital certificate is used to estabilish trust between entities (through the Chain of Trust)

- the Cerficate contains these main informations so the client can verify the server's identity:

- Identity information: The identity of the certificate owner, such as their name, organization, domain, or email address

- Public key: associated with the certificate owner, used for encryption and digital signatures

- Issuer's information: identity of the Certificate Authority (CA) that issued the certificate, which includes their name and organization details

- Serial Number: assigned by the CA

- Validity: validity star and end dates

- Digital signature algorithm: algorithm used by the CA to sign the certificate (such as RSA, DSA, or ECDSA)

- Issuer's digital signature: The CA's digital signature, which is created by signing the certificate content with the CA's private key (this is how the client can verify the authenticity and integrity by using the CA's public key)

File Extensions

*.pem (Privaci Enhanced Mail)

- contains an ASCII base64 encoded key (any key! certificate, private|public key etc...)

*.der (Distinguished Encoding Rules)

- contains a binary encoded key (any key! certificate, private|public key etc...)

*.cer / *.crt (CERtificate)

- contains a digital certificate (can be der (binary) or pem (ASCII base64) encoded)

*.pfx / *.p12

- contains both the certificate and the private key, in binary encoded format

Block cypher VS Stream cypher

- Block Cypher = encrypting one block at the time (ex: takes one 8 byte block at the time to encrypt, if the block is not a full 8 byte it will be padded)

- Stream Cypher = encrypting one bit or byte at the time (encrypting the data as it streams)
