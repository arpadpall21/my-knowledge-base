Config File

- the config file has 3 main sections: metadata, specification and status (the status is automatically generated, Kubernetess uses it to compare the desired VS actual status from here)

- components are referenced by giving labels then reference those lables 

// ---------------------------------
    apiVersion: apps/v1
    kind: Deployment
    metadata:                             // metadata section
        name: nginx-deployment
        labels:
            app: nginx                    // deployment label
    spec:                                 // specification section
        replicas: 2
        selector:
            matchLabels:
                app: nginx
        template:
            metadata:
                labels:
                    app: nginx            // template label
            template:
                metadata:
                    lables:
                        app: nginx
                spec:
                    // ...
    
// ---------------------------------
    apiVersion: v1
    kind: Service
    metadata:
        name: nginx-service
    spec:
        selector:
            app: nginx                    // selector refers to deployment lables
        ports:
            // ...
