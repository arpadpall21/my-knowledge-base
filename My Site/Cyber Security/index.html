Cyber Attack Types
Cyber Attack Description Prevention
man-in-the-middle attack - a hacker evasdropping between to pairs on the network - use secure encryption techniques like HTTPS
- use VPN
- be wary of using public WiFI
|
(Distributed) Denial Of Service ((D)DOS) - Overloading a system on the network with large amounts of request until it stops working
- DDOS (Distributed) means using different (lots of) network nodes to overload a targeted system 		- use CDN, distribute your service, it's harder to target sevaral distributed services
- use DDOS anti softwares, these softwares can recognize and block traffic from malicious sources
- use load balancers, these guys distribute traffic between servers and can be configured to filter out thse types of attacks
- use firewalls, to monitor and blacklist these type of sources
- use rate limiting, this technique limits the number of request from a single client
- have an incident response plan, plan what to do in case of a DDOS attack
|
SQL injection - injectin SQL script in the input field what the sql server execute as SQL command - parametrize SQL queries, provide the user with interfaces where sql parameters can be configured instead of sql commands
- imput validation, implement input validation on both client & server side (ex: allow only non comand characters)
- use least privilege access, so the user have access only what it needs
- setup security measures on DB side like instruction detection, firewall ect...
|
cross-site scripting (XSS) - injecting malitious javascript code into another website (the injected code can steal sensitive informations)
- ex: injecting script through comments

- a website allows users to post comments on articles, the comment section allows users to enter text, which is then displayed on the web page for other users to see

- the attacker injects the following script on the comment section <script>alert('Your account has been hacked!');</script>

- when another user views the comments, the script is execute in his browser the 'Your account has been hacked!' message is displayed

- imput validation, implement input validation on both client & server side (ex: allow only alphanumerical characters)
- output encoding, making sure that the content is not interpreted as HTML or JavaScript code
- HTTPOnly cookies, cookies are accessible through HTTP request only (not accessed by client side scripting)
- using sanitization libraries, the libs sanitize user input and filter out XSS attacks (DOMPurify, OWASP Java Encoder)
- implementing CSP (content security policies), implement CORS policies
cross-site request forgery (CSRF) - attacker sends a malicious form request in order to change the server content
- we assume that the attacker cracked the session token for this attack 		- implement CSRF protection

- the server generates a CSRF token and sends it to the client

- the client indludes the CSRF token in all form request (as hidden form field)

- with all form requests the server verifies the CSRF token along with the session token

(we usually include CSRF protection in form requests where the client modifies the server content (ex: for getting data no CSRF protection needed))
- do not transmit CSRF tokens in cookies (recommended to use a custom header)
|
phising - getting the user's credentials by some soft hacking way (like fake emails, phone calls, etc..) - educate your staff (like sending tricky mails so they are awaire of this)
- enable spam filter
- enable 2 factor authentication
|
malware - malicious software in the system - use antivirus
- be cautious of emails and downloads
- backup your data
randsomware - malicious software that lock the user's computer, the hacker asks for money to unlock it (asks for randsom) - use antivirus
- be cautious of emails and downloads
- backup your data
|
password attack - cracking the user's password with different malicious techniques
- brute forcing / dictionary (trying out common passwords) / social engineering 		- use strong password (combination of letter, numbers, symbol characters, etc...)
- use password manager
- enable 2 factor authentication
|
insider attack an emloyee (insider) uses its credentials for malicious activites - use least privilege access, so the user have access only what it needs
- monitor (suspicious) activity
- educate your staff about regulations and consequences
- conduct background check on employees
- have an incident response plan, plan what to do in case of a DDOS attack
|
zero-day attack - exploiting previously unknown vulnerabilities before it was patched (can occure the same day the vulnerability is discovered 'zero day') - update your software to the latest version
- implement network segmentation (segment your system)
- conduct regular security assesment
|
rogue software - malicious software disguised as legitimate application
- does malicious actions on the background
- tricks the user into believing that their computer is infected with malware and offer to sell solutions 		- use antivirus
- be cautious of emails and downloads
- enable pop-up blocker
- educate your staff
|
crypto jacking - the hacker uses the victims computing resources to mine cryptocurrencies - use antivirus
- monitor the system performance for unusual CPU usage
|
Advance Persistance Threat (APT) - malware that infiltrates a system and stay undetected over a long period of time
- tyipcally a large scale attack 		- implement network segmentation (segment your system)
- conduct regular security audits
- implement strong access control (2 factor authentication, strong password policies)
- backup your data
- have an incident response plan
|
IoT attack (Internet of Things) - gaining unauthorized access into a device connected to the internet (ex: home camera) - implement strong access control (2 factor authentication, strong password policies)
- secure your home network
- never use public WiFi to connect these type of devices
|
botnet attack - a group of compromized diveces (bots) are used to carry out malicious activities - monitor network activities
|
drive by (download) attack - malicious script that downloads and installs itself on the user's computer
- occures when the user visits a website that is compromized with malicious code 		- disable plugins, such as Java or Flash
- look out for suspicious downloads
|
Evil Twin Attack - hacker sets up a fake (wireless) access point that looks like a legitimate one to the user
- once the user connects to the fake access point, the hacker can intercept and manipulate the user's traffic, steal sensitive information 		- make sure the network you're connecting to is legit
- use 2 factor authentication
