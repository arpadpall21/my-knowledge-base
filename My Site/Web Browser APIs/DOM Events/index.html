Remember This :

- unlike in Node.js event listener functions are synchronously called in the event queue so they can block each other

- when the event listener is set by an attribute in an HTML element the "event" object must be passed as event

Description and Demonstration :

- DOM events are 'things' happening to HTML elements when something or the user invokes it, the Browser notifies JavaScript about these events

- most event types "propagate" to ancestor/child element, (Default: 'bubbles' = propagated to ancestor element)

- we can listen to these events in JavaScript and perform various tasks by registering listener functions (event listeners) on these events

- unlike in Node.js event listener functions are synchronously called in the "event queue" so they can block each other

Event Listeners and Event Handler Properties

- an Event Handler Property is an HTML element property in JavaScript which triggers its function when the specified DOM event is emitted (prefixed with on (ex: click event has the onclick Event Handler Property)

- the Event Handler Property is also present in the HTML element as attribute which directly executes JavaScript code as its value (ex: click attribute)

Methods and Properties

- properties and methods are specific depending the event type, but the following ones are available on every event because they are inherited from the Event constructor which sits on the top of the event chain

The composedPath() / composed / bubbles / stopPropagation() / cancelBubble / stopImmediatePropataion() / target / srcElement / currentTarget / eventPhase methods

SYNTAX:     Event.NONE                                                                 // returns 0 (means no event is proceded at this time)
            Event.CAPTURING_PHASE                                                      // returns 1 (means element is in the caputring phase when the event is happening)
            Event.AT_TARGET                                                            // returns 2 (means the event is produced by this element)
            Event.BUBBLING_PHASE                                                       // returns 3 (means element is in the bubbling phase when the event is happening)
            
            eventObj.composedPath()                                                    // returns an Array of objects of the possible (not live) event propagation path  
            eventObj.composed                                                          // indicates if the event crosses the "shadow-DOM" boundary
            eventObj.bubbles                                                           // returns if the event is a bubbling (bubbles up in the DOM tree) event (ex: onfocus is not "false")
            
            eventObj.stopPropagation()                                                 // prevents the event propagation to other elements
            evetnObj.cancelBubble                                                      // gets or sets the event propagation of the element
            eventObj.stopImmedieatePropagation()                                       // prevents the event propagation of the same event type
            
            eventObj.target                                                            // returns the element in the event propagation chain that trigered this event
            eventObj.srcElement                                                        // returns the element in the event propagation chain that trigered this event (predecessor of the target property)
            eventObj.currentTarget                                                     // returns the element itself the listener is registered on
        
            eventObj.eventPhase                                                        // returns a number which depicts where is the element in the event propagation chain when the event is happening (0 no event / 1 capturing phase / 2 target / 3 bubbling phase)
DEMO 
    <p id="demo"> myParagraph <u>childElement</u> </p>
<script>
    document.getElementById('demo').addEventListener("click", parentFunc);             // listener set up 
    document.getElementById('demo').addEventListener("click", parentFuncB);            // second listener set up with the same event type
    
    function parentFunc(e) {
        e.composedPath();                                                              // -> [u#demoChild, p#demo, body, html, document, Window]   // when clicked on the child element (event propagation chain)
        e.compsed;                                                                     // -> true                                                  // the click event crosses the "Shadow-DOM" boundary
        e.bubbles;                                                                     // -> true                                                  // bubbling event type (propagates the event)
        //  ------------------------------
        e.target;                                                                      // -> <u>childElement</u>                                   // when clicked on the child element (returns the element what fired this event)
        e.target;                                                                      // -> <p id="demo"> myParagraph <u>childElement</u> </p>    // when clicked on this element (returns the element what fired this event)     
            e.srcElement                                                               // does the same as "e.target" property
        e.currentTarget;                                                               // -> <p id="demo"> myParagraph <u>childElement</u> </p>    // no matter where is clicked (always returns this (current) element!)      
        //  ------------------------------
        e.eventPhase;                                                                  // -> 2                                                     // when clicked on this element (target phase) this element produces the event
        e.eventPhase;                                                                  // -> 3                                                     // when clicked on the child element (bubbling phase) other element propagates the event and this element is in the bubbling phase to that event    
        e.eventPhase;                                                                  // -> 1     (if the listener would set up "capturing") (capturing phase)
        //  ------------------------------
        e.stopPropagation()                                                            // event propagation is prevented, the child element event is no fired!
        e.cancelBubble = true;                                                         // does the same as "stopPropagatoin()" method 
        e.cancelBubble;                                                                // -> ture                                                  // because the event propagation is prevented                       
        e.stopImmediatePropagation();                                                  // prevents the second listener to fire of the (same event type)
    }
    
    function parentFuncB(e) {}                                                         // this function won't execute because the same event type propagation is prevented above "e.stopImmediatePropagation()"
</script>

The preventDefault() / defaultPrevented / cancelable / isTrusted methods

- many events has default actions, by using the preventDefault() we can prevent them to execute but there are some default browser behaviours what cannot be prevented!

SYNTAX :    eventObj.cancelable                                                        // returns a boolean value whether the default event behaviour can be prevented or not    
            eventObj.preventDefault()                                                  // prevents the default behaviour of the event (if it is cancelable)
            eventObj.defaultPrevented                                                  // returns a boolean value if the default behaviour of the event is prevented or not
                
            eventObj.isTrusted                                                         // returns a booleand value indicating if the event call is [trusted = user called] or [not trusted = script called]
DEMO 
    <a href="demoSite.html" id="preventDemo"> mySite </a>
<script>
    document.getElementById('preventDemo').addEventListener("click", preventLink);
    
    function preventLink(e) {
        e.cancelable;                                                                  // -> true      // because the "click" event is preventable on a link
        e.preventDefault();                                                            // default behaviour is prevented
        e.defaultPrevented;                                                            // -> true      // because we prevented this default event
        e.isTrusted;                                                                   // -> true      // when user clicks on the link (would be false if a script trigers this event)
    }
</script>
