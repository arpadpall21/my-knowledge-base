FOCUS EVENTS
Event Description Tag support DOM
blur fires when an element loses focus All HTML elements, EXCEPT: <base>, <bdo>, <br>, <head>, <html>, <iframe>, <meta>, <param>, <script>, <style>, and <title> 2
focus fires when an element gets focus All HTML elements, EXCEPT: <base>, <bdo>, <br>, <head>, <html>, <iframe>, <meta>, <param>, <script>, <style>, and <title> 2
focusin fires when element or its child gets focus All HTML elements, EXCEPT: <base>, <bdo>, <br>, <head>, <html>, <iframe>, <meta>, <param>, <script>, <style>, and <title> 2
focusout fires when element or its child loses focus All HTML elements, EXCEPT: <base>, <bdo>, <br>, <head>, <html>, <iframe>, <meta>, <param>, <script>, <style>, and <title> 2
FOCUS EVENT PROPERTY
Property Description
event = new FocusEvent(type, init) create FocusEvent Object (Not Tested)
eventObj.relatedTarget returns the related element that trigered the event (See Mouse Events) (Experimental Technology 2018-10-02)


INPUT EVENT
Event Description Tag support DOM
input fires every time when the element's content changes (ex: each button hit) <input type="color">, <input type="date">, <input type="datetime">, <input type="email">, <input type="month">, <input type="number">, <input type="password">, <input type="range">, <input type="search">, <input type="tel">, <input type="text">, <input type="time">, <input type="url">, <input type="week"> and <textarea> 3
INPUT EVENT PROPERTY
Property Description
event = new InputEvent(type, init) create InputEvent Object (Not Tested)
eventObj.data returns the inserted characters (Experimental Technology 2018-10-02)
eventObj.dataTransfer renturn a DataTransfer object (apparently has no use here [2018-10-02]) (Experimental Technology 2018-10-02)
eventObj.inputType returns the type of the input itself such as insert, delete, paste, ... (Experimental Technology 2018-10-02)
eventObj.isComposing returns a boolean value if the event is fired between the compositionstart and the compositionend events

FocusEvent()

- the FocusEvent() is derived from the UIEvent() which is derived form the Event() constructor

- this is how you can create a Focus Event (Not Tested!) MDN create FocusEvent Obj

SYNTAX :    event = new FocusEvent(type, init)     // NOT TESTED!

Focus Events 

SYNTAX :    focus                           	   // fires when element gets focus
            blur                                   // fires when element loses focus 
            focusin                                // fires when element or its child gets focus
            focusout                               // fires when element or its child loses focus
Example :

onfocus / onblur and onfocusin / onfocusout

CODE : 
	<input type="text" onfocus="this.style.background='orange'" onblur="this.style.background='lightgreen'">
	<input type="text" onfocusin="this.style.background='red'" onfocusout="this.style.background='blue'">

Focus Event Property 

SYNTAX :    eventObj.relatedTarget                 // returns the related element what trigered the current event
    
    <input type="text" onfocusout="focusOut(event)">
<script>    
    var focusOut(event) {
        event.relatedTarget;                       // -> null       // because not another element makes this element's focus to lose
    }
<script>

Input Event

SYNTAX :    input                                  // fires every time when the element's content changes

field change count = 0

Input Event Properties and Methods 

SYNTAX :    eventObj.data                          // returns the inserted characters
            eventObj.dataTransfer                  // renturn a DataTransfer object (apparently has no use here [2018-10-02])
            eventObj.inputType                     // returns the type of the input itself such as insert, delete, paste, ...
            eventObj.isComposing                   // returns a boolean value if the event is fired between the compositionstart and the compositionend events
    
    <input type="text" id="demo">
<script>
    document.getElementById('demo').oninput = function(event) {
        event.data;                                // -> a              // "a" button pressed
        event.inputType;                           // -> "insertText"   // "a" button pressed
    }
</script>

onchange VS oninput

- the onchange after element change fires when element loses focus

- the oninput fires upon element change

This field will turn green immediately when content change

This field will turn green after content change but when you click away
CODE : 
    <input type="text" oninput="this.style.background='green'"> 
	<span> This field will turn green <u>immediately</u> when content change </span> <p></p>
	<<input type="text" onchange="this.style.background='green'"> 
	<span> This field will turn green after content change but when you <u>click away</u> </span>
