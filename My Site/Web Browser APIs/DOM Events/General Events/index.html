General Events 

SYNTAX :  // Form --------------------------------------
            change                                     // fires when the content of a form element, the selection, or the checked state have changed   
            invalid                                    // fires when an element is invalid
            reset                                      // fires when a form is reset
            search                                     // fires when the user writes something in a search field
            select                                     // fires after the user selects some text
            submit                                     // fires when a form is submitted
            
          // Media ------------------------
            abort                                      // fires when the loading of a media data is aborted 
            canplay                                    // fires when the browser can play the media file (buffered enough to start playing)
            canplaythrough                             // fires when the browser estimates that the file can be played through without stopping for buffering
            durationchange                             // fires when the browser determines the new duration of the media file
            emptied                                    // fires when some error happens (like the file is suddenly not available)
            ended                                      // fires when media reached the end
            error                                      // fires when an error occured durring the media loading
            loadeddata                                 // fires when the data for the current frame is fully loaded
            loadedmetadata                             // fires when the metadata for the specified media file is loaded
            pause                                      // fires when the media is paused either by user or programmatically
            play                                       // fires when the media is started (no longer paused)
            playing                                    // event occurs when the media is playing after having been paused or stopped for buffering
            progress                                   // fires when the browser is downloading the media file
            ratechange                                 // fires when the media speed has been modified
            seeked                                     // fires when the user releases the seekbar after seeking
            seeking                                    // fires when user start seeking a new position on the seekbar
            stalled                                    // fires when the browser tries to get the media data but it is not available
            suspend                                    // fires when the browser intentionally not getting the media data
            timeupdate                                 // fires when the time indicated by the currentTime attribute is changing (about 200ms)
            volumechange                               // fires each time when the user changes the volume, mutes or unmutes 
            waiting                                    // fires when media stops because it needs to buffer the new frame
            
          // Other ------------------------
            afterprint                                 // fires after the print dialog box
            beforeprint                                // fires before the print dialog box
            beforeunload                               // fires before exiting or reloading the page a warning message which is not customizable
            load                                       // fires when an object has been loaded
            toggle                                     // fires when the <details> element opens or closes
            unload                                     // event occurs once a page has unloaded (browser windows has been closed) (Tested Not Working property, [2018-10-03])

Media Loading Process Events (in sequence) loadstart(ProgressEvent) -> durationchange -> loadedmetadata -> loadeddata -> progress -> canplay -> canplaythrough

Example :

ivalid / onreset / onsubmit

If you try to Submit a non-valid value it will turn Red
When you reset a message will display
Message will appear when form submit
CODE : 
    <fieldset>
        <form onreset="funcReset()" onsubmit="alert('Form Submits')">
            <input type="email" placeholder="E-mail" oninvalid="this.style.background='red'"> 
            <span> If you try to Submit a non-valid value it will turn Red </span><br>
            <input type="reset">
            <span> When you reset a message will display </span><br>
            <input type="submit">
            <span> Message will appear when form submit </span>
        </form>
    </fieldset>
<script>
	function funcReset(){
		alert('Form has been reset');
	};
</script>

search event

- enter something in the search filed than press Enter to seel the result

Your are Searching for ...

CODE : 
	<p> - enter something in the search filed than press Enter to seel the result </p>
	<fieldset>
		<input type="search" id="myInput" onsearch="funcSearch()" placeholder="Search..">
		<p> Your are Searching for <span id="search-box" style="text-decoration:underline; font-weight:bold;">...</span></p>
	</fieldset>
<script>
	function funcSearch() {
		var x = document.getElementById("myInput");
		document.getElementById('search-box').innerHTML = x.value;
	}
</script>
