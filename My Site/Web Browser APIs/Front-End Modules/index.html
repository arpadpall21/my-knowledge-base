Remember This :

- browsers make separate requests for each modules (as expected)

- frontend-modules run only from a server (so does not run locally, the file:// protocol not supported)

Description and Demonstration :

- Front-End modules work the same was as ES modules on the back-end (Node.js) (no CommonJS support)

- the top level module is the entry point of the application, it's responsible for initializing the application and importing other modules

- front-end modules are automatically deferred (no need to use a defer attribute) 

    <script src="scriptURL" type="module"></script>      // the requested script is treated as top-level ECMAScript module by the browser
    
    
    <script type="module">                                // this embeded script is treated as top-level ECMAScript module by the browser
        // JS code...
    </script>
    
// index.html (file) -----------------------------
    // ...
    
    <p id="hello"></p>
    <script type="module">                              // entry level module
    import hello from './modules/module1.mjs';                // imports sub module
    
    document.getElementById('hello').innerHTML = hello('World')
    </script>
    
    
// ./modules/module1.mjs (file) -----------------
    export default function hello(name) {
      return `Hello ${name}!`;
    }
