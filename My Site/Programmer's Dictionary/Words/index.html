Word Description
abstraction - hiding (complex) system parts, so on a higher level only essential parts are exposed
- that's why we say 'abstraction over a system|API|ect...'
agile developement - breaking the developement in pieces (sprints) so all these spirints can be developed separately
- Scrum, Kanban, XP (Extreme Programming)
argument - erv, erveles
attribute - tulajdonsag, jellegzetesseg (ahogyan viselkedik)
authenticated encrytion - encryption which also provides authentication (in addition of secrecy)
- like an encryption mode (block cipher mode) (ex: GCM, CCM, OCB, etc...)
authenticity - making sure that the message comes from the intended sender
big ball of mud - entangled code
binary (code) - file which is not readable by human (machine readable compiled program usually)
block | cluster (file system) - cluster of bytes on the disk, file system uses blocks in order to handle the disk efficiently (faster)
blockchain - blocks (nodes, objects, etc...) placed in a decentralized network chain in a way that they verify each other
- so changing one block will need the approval of all blocks in the chain
block cypher - encrypting one block at the time (ex: takes one 8 byte block at the time to encrypt, if the block is not a full 8 byte it will be padded)
canary (testing) - deploying a test code for a small group of users (who are unaware of this) for testing purpuses
code golf - express a code in a shortes possible way
compiling language-to-language translations in a deep level
concurrency VS parallelism - concurrency -> single thread that is passed where execution needed and freed up where is not (using single cpu) (async/await style programming)
- parallelism -> running multiple threads at the same time (using multiple cpu)
copy-on-write (CoW) - make copy link refference without creating a new file (if the new file is modified then it is created)
cipher - encryption algorithm
cipher suite - a set of algoritms that describes which algoritms should be used in TLS handshakes (ex: )
compiler - program what turns human readable code into machine readable code (soruce code to binary(executable))
cross-origin request - request to a resource which is hosted on an external origin
daemon - background process (service) on Unix-like systems
debouncing - limit (regulate) a frequently called task in order to gain performance
dependency - relation between modules
dinamically typed language - at compile time there's no type checking (varialbes are typeless)
distributed system - system whose components are located on different networked computers
end-to-end transport - network from the end client to the end server
ephemeral (TLS) - new certificate-key pair is used for each TLS session
encapsulation - hiding the actual data from the public (like in a class) then creating methods and conditions to access/modify this data
- so the data is not directly accessible to the public but only the methods that access/modify this data
endianness - CPU sequential byte order to represent a number (big endianness (most significants first) 0x00 0xFF = stored as 0x00 0xFF) (little endianness (less significant first) 0x00 0xFF = stored as 0xFF 0x00)
executable - a file containing an executable script or program (.exe executed by C++, .ps1 executed by PowerShell)
file descriptor - stream connected to a file or process (identified by a number) (on a process assigned by default (0 = stdin | 1 = stdout | 2 = stderr))
file handle - temporary file name or identifier assigned to an open file which is used by the operating system
file hash | hash | checksum - encrypted key generated with the original file
- it can be used by the client (which pairs the checksum and the original file) to verify that the original file has not been modified
firewall - security system that controls in/out netwrok traffic (prevents unauthorised connetions)
flooding (network flooding) - spread a message by telling other computers to resend it until all the computers get the message on the network
garbage collection - removing values that are not referenced my any varialbe
globbing - targeting files on the file system
hook - place in code that allows to tap in the API ('hook into the code')
- through a hook we can alter the behaviour of the API in question
hop-by-hop transport - network between two network nodes
hot swappable - hardware can be swapped while the computer is running (without stoping the computer)
hypervisor - virtual machine manager, (creates, runs and monitors virtual machines)
initialization vector (on Nonce) (in cryptography) - some data which is added to a key in order to generate a new unique key
- allows the reuse of the same
integrity (in cryptography) - making sure that the message has not been changed between the two parties
internet - network over the globe (not World Wide Web)
intrinsic / extrinsic (in CSS) - intisic = element is sized based on its content
- extrinsic = element is sized based on its parent
kernel (OS) - closest software layer to the hardware
load factor - a number from 0 to 1 indicating how empty of full something is
- (ex: 0.1 = 10% full / 0 = empty / 1 = full / 0.52 = 52% full)
media (network media) - communication channel in networking
memory leak (memory loss) - badly allocated memory (RAM) which is not used or cannot be accessed
metadata - data about data
meta programming - programming the programming language
microservice architecture - software architecture made of multiple independent 'micro' modules (services) that are loosely-coupled (monolitic VS microservice)
minifier - program that reduces file size
mixin (mix-in) - API or class which methods and properties are not directly implemented
- on runtime creates (mixes) an object, class, etc... from the intended sources
- (ex: the fetch API has the Body mixin implemented on both Request and Response, so the Body mixin looks like inherits from both classes)
mocking - faking a specific behaviour (like replacing a 'mocked' function with the original)
monolitic architecture - software architecture made of one large block (monolitic VS microservice)
namespace - container or scope
opaque response - Cross-Origin response with no CORS (headers)
Operating System - software between the hardware and the user software
optimization problem - a problem which requires the best solution (finding the minimal or maximal answer)
package - chunk of code (ready to install program)
parser - reads a piece of text and produces a data structure
persistent connection - live connection (estabilished connection)
polymorphism - overwriting properties/methods in the inheritance chain
polling (event handler system) - program put the event in an action queue than periodically check the queue and executes it
portable / standalone aplication ready to run program (does not need to be installed)
preflight request - a CORS pre request in order to figure out what kind of access the third party server provides for cross-origin requests
- an OPTION HTTP request with Access-Control-Request-Method Access-Control-Request-Headers and Origin headers
proprietary software owned sofrware (not open-source)
regression - predicting future data based on data in the past (AI) (I'm not 100% sure of this though!)
regression testing - testing the system after a change
repository "repo" - central storage location for packages, metadata, ect...
round-robin (fashion) - algorithm which distribute the load equally between members in a round fashion (used by nodejs cluster module)
runtime environment where the grogram is running (like the browser for JS)
run-to-completition - scheduling model in which each taks runs utnill it either finishes
salting (cryptography) - adding some data (pading the data) to the original data then hashing it, this way the generated hash is even harder to guess
scripting programming language script runs directly (does not need to be compiled) (interpreted by a program)
sharding dataset split and stored in multiple databases
- vertical sharding (scaling) upgrading the capacity of a single server
- horizontal sharding (scaling) distirbuting the dataset among multiple servers
socket communication endpoint
sourcecode human readable program (the program before compiled)
sourcemapping host recognizes the original file through a compiled or transpiled file
spying - spying on a specific behaviour (like spying on a function if it was called or not)
stabbing - faking a specific content (like faking some kind of content that exists)
statefull API having inernal storage (state) so it can remember previous states
stateless API not having internal storage (state) so it cannot remember any previous states (restarts default state every time it started)
statically typed language variable value type must be defined
stream cypher - encrypting one bit or byte at the time (encrypting the data as it streams)
swapping (in memory management) moving processes out|in the RAM to|from the disk to give space to other processes
syntax sugar easier to read syntax (code)
tarball Linux Archive file
TC39 committee evolving javascript
- stage 0 = strawman
- stage 1 = proposal
- stage 2 = draft
- stage 3 = candidate
- stage 4 = finished
transpiling - programming language-to-language translations
tree shaking - dead code elimination (dead code removal)
type casting - data type conversion (usually between 2 programming languages)
unit testing - testing a part of a software
vertext ('vertices' for multiple) - a point represented in 2D or 3D space
waterfall developement - sequential software developement
wildcard character - character that represent one or more other characters (ex: *)
