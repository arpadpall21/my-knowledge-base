Concepts
Concept Description
AI agent - Software system that can autonomously perceive its environment, make decisions, and take actions to achieve specific goals
classification - Predicting discrete labels or categories (ex: spam VS not spam)
cross-validation - Evaluating the performance of a model by splitting the data into multiple subsets and testing the model on different portions of the data. This helps ensure that the model generalizes well to unseen data and avoids overfitting
Hyperparameters - Tunable variables that are set before model training to control the learning process (external to the model)
LLM (Large Language Model) - A type of AI model designed to understand and generate human-like text based on vast amounts of textual data (ex: GPT-4, BERT)
model (ML Model) - Mathematical representation of a real-world system or process
model class - Blueprint or template that defines the structure and behavior of a machine learning model (ex: LGBMRegressor, RandomForestClassifier)
overfitting / underfitting - (Overfitting) When a model learns the training data too well, capturing noise and outliers, resulting in poor generalization to new data
- (Underfitting) When a model is too simple to capture the underlying patterns in the data, leading to poor performance on both training and new data
pattern (Model Component) - (weight) Relationships between X and Y learned from data
positive VS negative class - In binary classification, the positive class is positive outcome of the target variable VS negative class is negative outcome of the target variable (email spam ex: spam VS not spam)
regression - Predicting continuous numeric values (ex: house prices)
target variable - The variable (label) that the model is trying to predict or explain
token - Concept in natural language processing (NLP), it refers to a single unit of meaningful data (word, character, or subword)
X (Model Component) - Input Data (features)
Y (Model Component) - Class to be predicted (target variable)

Description and Demonstration :

- Machine Learning (ML) is a technique that allows computers to learn from data and make predictions or decisions without being explicitly programmed

- Deep Learning is a subset of ML that uses multi-layer neural networks to model complex patterns in data

- Artificial Intelligence (AI) a technique that computers to mimic human behaviour

- Generative AI is a type of AI that can generate new content (text, image, video, etc...)

Computer Science VS Machine Learning

- In Computer Science, at development time we pass the Formula (code) + Input to the computer which generates the output

- In Machine Learning, at Development (training) time we pass the Output + Input to the computer which generates the Formula (Model)

- In both Computer Science and Machine Learning, at Deployment time we pass the Formula (code/model) + Input to the computer which generates the output 

    Development Time                                               Deployment Time
    
// Computer Science 
    Formula (code) ->                                              Formula (code)  ->
                      [Computer] -> Output                                           [Computer] -> Output
    Input          ->                                              Input           ->
    
// Machine Learning 
    Input          ->                                              Formula (Model) ->
                      [Computer] -> Formula (Model)                                  [Computer] -> Output
    Output (data)  ->                                              Input           ->

Types of Machine Learning

- Supervised Learning:

- The model is trained on a labeled dataset, meaning that each training example is paired with an output label. The model learns to map inputs to the correct output (ex: classification (spam detection), regression (predicting house prices))

- Unsupervised Learning:

- The model is trained on an unlabeled dataset and must find patterns and relationships in the data without any guidance on what the output should be (ex: clustering (document clustering), annomaly detection)

- Reinforced Learning:

- The model learns by interacting with an environment and receiving feedback in the form of rewards or penalties. The goal is to learn a policy that maximizes cumulative rewards over time (ex: Game playing (AlphaGo), Robotics, Self-driving cars)

- Generative Learning:

- The model learns to generate new data that is similar to the training data (ex: image, text, and music generation)

Machine Learning Tools

- 1) AI Services

- Ready to use AI service for common tasks such as image recognition, natural language processing, and recommendation systems, accessible via APIs (ex: Azure Cognitive Services, AWS AI Services)

- 2) ML Services

- Offer tools and frameworks for building, training, and deploying custom machine learning models (ex: Azure Machine Learning, AWS SageMaker)

- 3) Frameworks, Libraries, and Infrastructure

- Facilitate the development and deployment of machine learning models (ex: TensorFlow, PyTorch, Scikit-learn)

MCP (Model, Context, Protocol)

- The MCP is essentially an LLM proxy protocol that defines how to interact with the model, how to structure the communication to optimize the model's performance and accuracy

- Features:

- Expose specific capabilities of the model to the user (ex: model trained for coding exposes such features)

- Remember Context (ex: user preferences, historical data, environmental conditions)

- Optimize the communication protocol to enhance the model's performance and accuracy (ex: structuring prompts in a way that the model can understand better)

- Provide a standardized way to interact with different models and services, making it easier for developers to integrate and utilize various AI capabilities

RAG VS MCP
Aspect MCP (Model, Context, Protocol) RAG (Retrieval-Augmented Generation)
External Retrieval - Does not rely on external data sources (solely on the model's internal knowledge) - Integrates with external data sources
Context Source - Focuses on optimizing input context - Combines externally retrieved data with model context
Scalability - Limited by model token capacity - Scalable with large external data
Dependency - No external dependencies - Requires a retrieval system (e.g., database)
Use Case - Best for tasks with consistent, structured context - Best for dynamic, knowledge-intensive tasks
