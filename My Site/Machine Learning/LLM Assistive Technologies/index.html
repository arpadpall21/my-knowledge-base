RAG VS MCP
Aspect MCP (Model, Context, Protocol) RAG (Retrieval-Augmented Generation)
External Retrieval - Does not rely on external data sources (solely on the model's internal knowledge) - Integrates with external data sources
Context Source - Focuses on optimizing input context - Combines externally retrieved data with model context
Scalability - Limited by model token capacity - Scalable with large external data
Dependency - No external dependencies - Requires a retrieval system (e.g., database)
Use Case - Best for tasks with consistent, structured context - Best for dynamic, knowledge-intensive tasks

MCP (Model, Context, Protocol)

- The MCP is essentially an LLM proxy protocol that defines how to interact with the model, how to structure the communication to optimize the model's performance and accuracy

- Features:

- Expose specific capabilities of the model to the user (ex: model trained for coding exposes such features)

- Remember Context (ex: user preferences, historical data, environmental conditions)

- Optimize the communication protocol to enhance the model's performance and accuracy (ex: structuring prompts in a way that the model can understand better)

- Provide a standardized way to interact with different models and services, making it easier for developers to integrate and utilize various AI capabilities

- Etc...

RAG (Retrieval-Augmented Generation)

- RAG is a technique that combines retrieval-based methods with generative models to enhance the quality and relevance of generated content

- Retrieval: Retrieving relevant information from external sources (knowledge bases, databases, documents, etc...)

- Augmented: augmenting your prompt with the retrieved information

- Generation: LLM generates a response based on the augmented prompt

Prompt Engineering

- Prompt engineering is a technique to optimize the user query to get better responses from the model ex:

- Without prompt engineering: "Explain photosynthesis"

- With prompt engineering: "Explain photosynthesis in simple terms for a 10-year-old child"

Query Validation

- For both MCP & RAG, validating the user query is crucial in order to provide relevant and accurate responses

- Techniques for query validation:

- Keyword Matching: Checking if the query contains relevant keywords

- Semantic Similarity: Checking if the query contains similar (relevant) text (ex: Car is similar to Automobile)

- 1) Convert the query into vector representation (embeddings)

- 2) Compare the query vector with vectors of potential relevant texts (like with a Vector Database)

- 3) Calculate similarity score (ex: cosine similarity)

- 4) Determine relevance based on similarity score

- Classifier Model: The model classifies the query (ex: Geography, Science, Technology, Irrelevant, etc..)
