AWS Concepts:

- Shared Responsibility Model

-> AWS is responsible for the security of the cloud

-> the customer is responsible for the security in the cloud

- Well-Architected Framework - best practices and guidelines developed by AWS having 6 pillars:

- Operational Excellence -> Security -> Reliability -> Performance Efficiency -> Cost Optimization -> Sustainability

AWS Global Infrastructure

- AWS Regions -> physical location in the world with multiple availability zones

- Availability Zone (AZ) -> one or more data centers, each with redundant power, networking and connectivity (to ensure redundancy within that region)

- Local Zones -> AWS infrastructure typically in large cities where no AWS region exists (to reduce latency)

- Wavelength Zones -> AWS infrastructure for 5G networks (to reduce latency)

- Edge Locations -> used caching content (to reduce latency) (CloudFront -> Amazon's CDN)

Cost & Support Management

- AWS provides the following tools to manage our costs:

- AWS Budgets (AWS resource) -> custom budgets with set alerts and limitations

- Migration Hub (AWS resource) -> track and manage migrations to AWS

- Cost Explorer (AWS resource) -> visualizes our costs and usages

- we can create organizations within the same account and apply budgets to them

- we can tag resources and apply budgets to them

- AWS pricing calculator to create budget estimates

- AWS provides the following supporting tools:

- AWS provides the following support plans (in level order) Basic (free) -> Developer -> Business -> Enterprise

- AWS Personal Health Dashboard (AWS service) -> dashboard that provides alerts and remediation guidance when AWS is experiencing events that might affect you

- AWS Trusted Advisor (AWS service) -> provides real-time guidance to help us provision our resources following AWS best practices

3 ways to interact with AWS resources

There are 3 ways to interact with AWS services:

- AWS Management Console -> GUI in the browser

- AWS Command Line Interface (CLI) -> full access to the same API as the AWS Management Console, it can be used in the AWS account browser or remotely

- SDK (Software Development Kit) -> programmatic access to AWS Services (supported by most programming languages ex: boto3 in Python) -> used to automate tasks

AWS Users

- AWS users can be created and managed in the IAM (Identity and Access Management) service

- There are 2 user types:

Root User -> has full access to all AWS services

IAM User -> has managed (limited) access to AWS resources

User -> individual IAM User

Group -> collection of users with the same permissions (a user can belong to multiple groups)

Role -> can be assigned to a user or an AWS service in order to allow to access to specific AWS resources

- an AWS Policy is a JSON file describing a set of access rules to AWS services

- AWS also supports Identity Federation (3rd party identification solutions like OIDC)

AWS Disaster Recovery

- AWS suggests 4 levels of Disaster Recovery strategies (Efficienty VS Cost):

(Cost +---) (Recovery Time ++++) -> Backup & Restore

- data is backed up in S3

(Cost ++--) (Recovery Time +++-) -> Pilot Light

- a backup system running with only critical components required by the application

(Cost +++-) (Recovery Time ++--) -> Warm Standby

- a scaled-down backup system running the fully functional environment

(Cost ++++) (Recovery Time +---) -> Multi Site

- a fully functional environment is running as a backup

AWS Services, Resources, Tools

Computing Services:

EC2 -> virtual machine

Concepts:

- Instance Type -> defines a virtual machine type (processor, memory, etc..) (ex: t3.medium, m5.large)

- Root Device Type -> storage for instance types

- Instance Store -> physical storage of the host machine (cannot be detached -> data will be lost when the instance is stopped)

- EBS (Elastic Block Store) -> network attached storage can be detached and attached to other instances

- AMI (Amazon Machine Image) -> template for an EC2 instances

Payment Plans:

- Dedicated Host -> physical server dedicated to us

- Reserved Instance -> we reserve EC2 instances for a specified duration, upfront payment for the capacity we reserve (1 or 3 years) -> lowest cost for long term

- On-Demand Instance -> we pay for the capacity we use per hour -> highest cost

- Spot Instances -> this is basically bidding for unused capacity on AWS, it's suitable for workloads that can be terminated instantly -> variable pricing (cheapest option with reliability trade offs)

Architecting in EC2:

- auto-scaling groups -> scaling rules that apply to EC2 instances belonging to the same group

- defines scaling options (min, max, desired), runs health checks on each instance in the group

- elastic load balancing -> intelligent traffic distribution

Access Control:

- security group -> firewall rules that apply to all EC2 instances belonging to the same group (works on EC2 instance level)

- Network ACL (Access Control List) -> controls inbound and outbound traffic (works on VPC level)

- VPN (Virtual Private Network) -> secure tunnel between our network and AWS (unlike Direct Connect this solution goes through the public internet)

Lambda -> serverless computing (pay per usage)

- the Lambda service has excellent integration with other AWS services (S3, DynamoDB)

Elastic Beanstalk -> ready-to-go platform for different programming languages

- Paas (Platform as a Service) -> you manage the application code and data -> AWS manages the rest (Monitoring, Deployment, Scaling, EC2 customization)

Container Management Services:

App Runner -> managed service for container management

Fargate -> serverless architecture for container management

ECS (Elastic Container Service) -> managed service for Docker containers

EKS (Elastic Kubernetes Service) -> managed service for Kubernetes containers

Network & Content Delivery Services:

Route 53 -> AWS Domain Name Service (global service)

- able to register domains and route AWS resources to these domains

VPC (Virtual Private Network) -> Logically isolated network with AWS

- this is basically a sub-network within an AWS account

Direct Connect -> network connection between AWS and private data centers

- a private connection that doesn't go through the internet

API Gateway -> front door access for AWS services

- API management service that integrates with other AWS services (like Lambda, VPC, etc)

CloudFront -> AWS CDN basically

- caching content closer to the user in order to enhance performance

Elastic Load Balancing -> AWS load balancer

- distributes incoming traffic across multiple targets (EC2 instances, containers, etc)

Global Accelerator -> accelerating traffic through AWS infrastructure

- the idea here is to use the AWS global infrastructure instead of the internet (which according to AWS is faster)

File Storage and Data Transfer Services:

S3 -> standard AWS file storage solution

- stores files as objects in buckets

- buckets a classified as classes (classes are basically settings for the specified bucket)

- files can be accessed through URLs (static hosting is available)

S3 Classes:

- S3 Standard -> for frequent access

- S3 Standard Infrequent Access -> for less frequent access

- S3 One Zone Infrequent Access -> files are stored in one availability zone only (cost-efficient but less resilient option)

- S3 Intelligent Tiering -> AWS automatically moves data to the most cost-effective tier

S3 Glacier -> file archival solution

S3 Glacier Classes:

- S3 Glacier Instant Retrieval -> archived data that needs to be accessed within milliseconds

- S3 Glacier Flexible Retrieval -> archived for data that needs to be accessed in a range of minutes to hours (depending on configuration)

- S3 Glacier Deep Archive -> archived data that needs to be accessed within 12 hours

Elastic Block Store (EBS) -> persistent block storage for EC2 instances

- can take snapshots of EBS volumes and offers encryption features

- several volume types can be used (SSD HDD, etc...)

Elastic File Syste (EFS) -> persistent block storage for EC2 instances (for Linux EC2 only)

- can be mounted on multiple EC2 instances at the same time

Snowball -> AWS service that physically transfers data into AWS (for petabyte-scale)

- AWS sends a physical drive where we save our data then we send the drive back, the data will be transferred to S3 at the end

Snowball Mobile -> AWS service that physically transfers data into AWS (for exabyte scale)

- AWS sends a truck to physically transfer your data to an AWS data center, the data will be transferred to S3 at the end

Database Services and Utilities:

Relational Database Service (RDS) -> AWS relational database management system service

- Launches within a VPC

- supports multiple DRMS servers (MySQL, PostgreSQL, etc...)

Aurora -> AWS RDMS server that supports both MySQL and PostgreSQL

DynamoDB -> AWS key-value pair and document-based database

- a serverless database that offers very high performance and scalability (according to AWS)

Redshift -> AWS data-warehousing service (petabyte scale)

- offers analytics, monitoring, etc... services for big data

Elasticache -> caching service

- serverless in-memory data (caching) service

- supports Reddis and Memcached

MemoDB -> Reddis compatible caching service

Migration Service -> database migration into AWS

DocumentDB -> MongoDb compatible document database

Neptune -> graph database

TimeStream -> time series database (saving data with timestamps)

App Integration Services:

Simple Notification Service (SNS) -> fully managed pub-sub messaging service

- organizes messages into topics, has great integration with other AWS services

- sends messages to multiple subscribers

- messages are short-lived, if the subscriber service is not available the message will be lost!

Simple Queue Service (SQS) -> fully managed queuing service

- queuing service that supports up to 256 Kb data payload, if the receiving service is not available the data is preserved in the queue for 14 days (great way to decouple services)

There are 2 queue types:

- Standard Queue -> does not guarantee message order

- FIFO Queue -> guarantees the message order

Step Functions -> serverless function orchestrator

- can coordinate multiple AWS services into serverless workflows

Management and Governance Services:

CloudTrail -> tracks user activity

- track user activity API usage and infrastructure changes

- continuously monitors account activity and logs into an S3 bucket or CloudWatch logs

CloudFormation -> provisioning infrastructure based on templates (without additional cost)

- creates, updates or deletes AWS resources (automates infrastructure deployment -> infrastructure as a code approach)

- supports JSON or YAML

CloudWatch -> monitors and manages AWS resources

- collects logs, metrics and events, we can set alarms based on predefined settings

Config -> tracks AWS resource configurations

- continuously monitors and records AWS resource configuration changes (provides a config history)

SystemsManager -> grouped operation over an infrastructure

- provides tools to manage a grouped infrastructure (like task automation)

- has a parameter store where parameter purpose data can be stored

ControlTower -> centralizes multiple AWS account management

- gives a high-level overview of multiple AWS accounts

OpsWorks -> configuration management service

- provides individual configurations to services in the stack

- configuration is defined as code, supports Chef and Puppet

AWS Organizations -> AWS organization management

- allows organizations to manage multiple accounts under a single account

Security Services:

Secrets Manager -> centralized service where secrets are stored and managed

- offers advanced secret storage features like auto-rotation etc...

Artifact -> AWS security and compliance reports

Well-Architected Tool -> review and improve your architectures with AWS best practices

- we enter our architecture parameters then AWS suggests best practices for implementation

Cognito -> identity and access management for applications

Guard Duty -> intelligent thread detection

Shield -> protects against DDoS (Distributed Denial of Service) attacks

Macie -> intelligent thread detection that uses Machine Learning

Inspector -> Automated vulnerability detector that continuously evaluates resources

Security Hub -> security overview across AWS accounts

CloudHSM (Hardware Security Module) -> a cloud-based physical security solution (to generate cryptographic keys, etc...)

Data Integration Services:

Storage Gateway -> storage in the cloud solution

DataSync -> moving data in the cloud solution

Data Processing Services:

Glue -> serverless extract, transform, load (ETL) services

- greatly integrates with AWS storage services (RDS, S3, Redshift, DynamoDB)

Elastic Map Reduce (EMR) -> big data analysis

- integrates big data workloads such as Presto, Hive, etc...

Data Pipeline -> data workflow orchestrator

- suports RDS, S3, Redshift and DynamoDB

Data Analysis Services:

Athena -> interactive query service

- uses SQL queries to data query

QuickSight -> business intelligence service

- dynamic data dashboard for data stored in AWS

CloudSearch -> managed search service

- search solutions in the cloud

AI and Machine Learning:

Rekognition -> Computer Vision service powered by Machine Learning

- recognizes things on image and video

Translate -> language-to-language translation service

Transcribe -> speach to text solution powered by Machine Learning

Pre-Defined Solutions:

Service Catalog -> set of services that can be deployed

Marketplace -> 3rd party catalogs in AWS

Dev-Tools:

CodeCommit -> managed source control service (uses Git)

CodeBuild -> fully managed continuous build and integration service

CodeDeploy -> fully managed deployment service to deploy custom applications

CodePipeline -> fully managed deployment service (higher level abstraction over other deployment tools)

CodeStar -> automation tool for other dev services
