Cloud Concepts

- Cloud Computing -> on-demand delivery of IT resources and applications via the internet with pay-as-you-go pricing

- Elasticity -> Scale up and down as you need

- Agility -> The ability to quickly scale up and down resources

- OpEx - Operational Expenditure (ongoing operational expenses)

- CapEx - Capital Expenditure (upfront capital investment)

- TCO - Total Cost of Ownership

Hardware Infrastructure Models

- on-premise (private) -> we own our hardware and software

- cloud (public) -> we use public cloud providers like AWS, Azure, etc...

- hybrid -> mix of on-premise and cloud

Shared Responsibility Models

- IaaS (Infrastructure as a Service): Azure VM, AWS EC2

- The customer is responsible for the operating system, applications, and data

- PaaS (Platform as a Service): Azure WebApp, AWS Elastic Beanstalk

- The provider is responsible for the underlying infrastructure, including servers, storage, networking, authentication, etc...

- The customer is responsible for the applications and data (App developer focused)

- SaaS (Software as a Service): Azure Office 365

- The provider delivers a fully working service, including infrastructure, middleware, and software updates

- The customer is simply using the service
