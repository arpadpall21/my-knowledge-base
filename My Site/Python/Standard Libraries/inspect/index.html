Method Description
inspect.getmembers(obj) - returns all members of obj in a list of tuples
|
inspect.typeCheckerMethod() - methods to check object type (like ismodule, isclass, etc...)
|
Retrieve Source Code
inspect.getfile(obj) - returns the absolute file path (in a string) where the obj is defined
inspect.getmodule(obj) - returns the moudle object where the obj is defined
|
inspect.getsource(obj) - returns the source code of the passed object
inspect.signature(callable) - return the call Signature object of the callable (object describing the call signature including types)
|
inspect.otherMethod() - other source code methods (ex: get docstring, comments, etc...)
|
Current State of Coroutines, Generators and Async Generators
inspect.getcoroutinestate(coroutine) - returns the current coroutine state

CORO_CREATED: Waiting to start execution

CORO_RUNNING: Currently being executed by the interpreter

CORO_SUSPENDED: Currently suspended at an await expression

CORO_CLOSED: Execution has completed

inspect.getgeneratorstate(generator) - returns the current coroutine state

GEN_CREATED: Waiting to start execution

GEN_RUNNING: Currently being executed by the interpreter

GEN_SUSPENDED: Currently suspended at a yield expression

GEN_CLOSED: Execution has completed

inspect.getasyncgenstate(asyncGenerator) - returns the current coroutine state

AGEN_CREATED: Waiting to start execution

AGEN_RUNNING: Currently being executed by the interpreter

AGEN_SUSPENDED: Currently suspended at a yield expression

AGEN_CLOSED: Execution has completed

Description and Demonstration :

- the inspect library provides live informations about objects (modules, classes, methods, functions, tracebacks, frame objects, and code objects)

- the following information can be retrieved:

- type checking

- source code

- inspecting classes and functions

- examining interpreter stack
