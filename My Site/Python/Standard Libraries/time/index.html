time.time() - returns the UTC time in seconds since epoch as float

time.strftime(fromat:str, struct_time) - formats the time (if passed) according to the format argument and returns the result in a string

time.monotonic() - returns the value as float of a monotonic clock (separated clock from the system)

time.perf_counter() - returns the time value as float of the performance counter (highes precision counter to measure performance)
time.perf_counter_ns() - returns the time value as int of the performance counter (highes precision counter to measure performance)

time.process_time() - returns how much time the process took to reach this point from start in seconds as float (calculates processing time only = excludes sleep)
time.process_time_ns() - returns how much time the process took to reach this point from start in nanoseconds as int (calculates processing time only = excludes sleep)
- more accurate version of process_time()
time.thread_time() - returns how much time the thread took to reach this point from start in seconds as float (calculates processing time only = excludes sleep)
time.thread_time_ns() - returns how much time the thread took to reach this point from start in nanoseconds as int (calculates processing time only = excludes sleep)
- more accurate version of thread_time_ns()
time.sleep(sec:int|float) - supsend the execution for the given secconds (float for more precise time)

time.tzset() - activates the newly set TZ (time zone) sysmtem variable value

time.daylight - DST timezone offset (in hour)

time.tzname - returns the name of the timezone in a tuple (local non-DST timezone, local DST timezone)
time.pthread_getcpuclockid(threadId) - returns the clock id of the thread specific CPU-time clock for the specified thread id

time.clock_setttime(clockId, time:float) - sets the time of the specified clock_id (currently only CLOCK_REALTIME accepted only)

Remember This :

Description and Demonstration :

- epoch is where the time starts (in Unix systems it's January 1, 1970 00:00:00), use time.gmtime(0) to find out in your platform

- DST (Daylight Saving Time) is calculated according to the local system

- in Python the local time is controlled by the TZ system variable, we manually set the TZ value which tells python the local timezone (remember using time.tzset() to 'activate' the new timezone)

DEMO

- pthread_getcpuclockid() 

    import time
    import threading
    
    current_threadid = threading.get_ident()
    clock_id = time.pthread_getcpuclockid(current_threadid)

- using the TZ system variable and the tzset() method (in this case the TZ system variable is not set 

    import time
    import os
    
    print( time.gmtime() )        // both returns UTC time 
    print( time.localtime() )
    
    os.environ['TZ'] = 'Europe/Budapest'    // timezone set 
    time.tzset()                            // we have to 'activate' the newly set timezone like this 
    
    print( time.gmtime() )      // returns UTC time 
    print( time.localtime() )   // returns 'locale' time (according to the set TZ value)

- mktime() 

    import time
    
    struct_time = (2022, 6, 8, 18, 20, 40, 2, 159, -1)      // the last argument (dst is -1) to use local time 
    
    print( time.mktime(struct_time) )     // -> 1654712440.0    // epoch
