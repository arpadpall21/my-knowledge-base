Pip [Ordered: Alphabetically]
Command Description
pip3 --version - prints the pip version
- (3) for python v3 (used if the system has both python v2 and python v3)
|
pip install packagesemVer ...
- installs the specified package(es) latest or the specified version
- use string for complex semantic versions (ex pip install "Django>=4.0.0,<4.0.4,!=4.0.3")
pip install --upgrade package ... - upgrades the installed package(es) to the latest version
pip install -r requirementsFilePath - install packages listed in the specified requirements file
pip uninstall package ... - uninstalls the specified packag(es)
pip uninstall -r requirementsFilePath [-y] - uninstall packages listed in the specified requirements file
- (y) yes for all uninstallation (won't prompt)
|
pip list [-v] [-o] - lists installed package
- (-v) also prints package locations
- (-o) outdated packages only (packages that have newer version available)
pip freeze - list installed packages in requirements format
- to remove all packages whe can use pip freeze > rem.txt ; pip uninstall -r rem.txt hack
|
pip show package - displays packages informations (version, location, package dependencies, etc...)

requirements.txt Semantic Versioning
Versioning Description
(nothing) - (no version specified) = latest version on CMD and in requirements.txt
* - latest version in Pipfile
|
== - pinned version
< | <= | > | >= - lower | lower or equal | higher | higher or equal version
!= - excluded version
|
semVer,... - multiple version conditions

Description and Demonstration :

- Pip (Pip Installs Packages|Python or Preferred Installer Program) is the default Python package installer for Python (included in Python v3.4)

- Pip has a large contribution on pypi.org where we can download from or publish Python packages

- Pip is kind-of a limited package manager, so we mostly use other tools (like Poetry)

DEMO 
// --version --------------------------------------------------------------------------    
    $ > pip --version                                                                  // pip version  
    
    
// install ----------------------------------------------------------------------------
    $ > pip install requests==2.26.0                                                   // installs the specified version   
    $ > pip install --upgrade requests                                                 // upgrades the request package to the latest version 
    $ > pip install -r requirements.txt                                                // installs packages listend the requirements.txt file  
    
    
// uninstall --------------------------------------------------------------------------
    $ > pip uninstall requests                                                         // uninstalls the specified package   
    $ > pip uninstall -r requirements.txt -y                                           // uninstalls packages listend the requirements.txt file  
    
    
// list -------------------------------------------------------------------------------
    $ > pip list                                                                       // lists all installed pacakge 
    $ > pip list -o                                                                    // lists outdated pacakges only  
    
    
// freeze -----------------------------------------------------------------------------
    $ > pip freeze                                                                     // lists all installed packages in requirements format 
    $ > pip freeze > requirements.txt                                                  // creates a requirements.txt file with the installed pacakges  
    
    
// show -------------------------------------------------------------------------------
    $ > pip show requests                                                              // displays pacakge info

The requirements.txt file

- the requirements.txt file lists packages in requirement format, Pip can use it to install or uninstall listed packages in this file (pip install -r requiremensFilePath)

- Pip never modifies the requiremenst.txt file (we do this manually), the requiremenst.txt it's just a naming convention, we can name this file as we like

- Pip doesnt have any lock file so does not support deterministic builds 

// requirements.txt (file) ------------------------------------------------------------
    packagesemVer                                                                       // requirement format
