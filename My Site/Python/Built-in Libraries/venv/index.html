Description and Demonstration :

- the built-in venv module (requireds python v3.3) allows us to create a lightweight virtual enviroment which is an isolated 'local' enviroment

- each virtual enviroment has its own python version and packages separated from the system and other virtual enviroments

- after creating the virtual enviroment we have to enter in it (and exit back to the system environment)

- the python version on creation time will be used in venv, there's a pyvenv.cfg (venv config file) in the venv root folder 

    python -m venv path

    
 - directory where to create the virtual enviroment (usually .venv in the current directory) 

  
                 // creates a virtual enviroment in path
  
    venvSrc venvActivate                 // enter in virtual enviroment (locally usually this is . .venv/bin/activate)    
  
  
// from virtual enviroment -----------------------------
    deactivate                           // exit from virtual enviroment
DEMO 
    > python -m venv .venv                        // creates a virtual enviroment in the current directory (creates a .venv directory where all venv staff is stored)
    
    > . .venv/bin/activate                        // enters the created virtual enviroment 
    
    // from virtual enviroment -----
    > deativate                                   // exit from virtual enviroment
