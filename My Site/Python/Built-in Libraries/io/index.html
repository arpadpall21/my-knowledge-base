Description and Demonstration :

- the io module provides low-level io interfaces, we usually not directly implement them, they are abstracted by other python interfaces like open(), shuttil lib, etc...

// IO hierachy ---------
    
    IOBase                            // abstract class for all IO classes
    │
    ├── RawIOBase                     // raw binary streams that provide low-level access to an underlying OS device or API (doesn't do any high level stream operations)
    │   └── FileIO                    // Represents an OS-level file containing bytes data
    │
    ├── BufferedIOBase                     // binary streams that support some kind of buffering
    │   ├── BufferedWriter                    // high-level access to a writeable non seekable raw binary stream
    │   ├── BufferedReader                    // high-level access to a readable non seekable raw binary stream
    │   ├── BufferedRWPair                    // high-level access to a readable and writable non seekable raw binary stream
    │   ├── BufferedRandom                    // high-level access to seekable raw binary stream, inherits from BufferedReader and BufferedWriter (readable writable)
    │   └── BytesIO                    // handles in memory binary data (like it were a file)
    │
    └── TextIOBase                     // interface for text based IO operations
        ├── TextIOWrapper                    // handles text encoding/decoding, wraps the BufferedIOBase (so basically works on top of BufferedIOBase)
        └── StringIO                    // handles in memory string data (like it were a file)

- few examples:

- a file-like object can implement TextIOWrapper, BufferedReader|BufferedReader or StringIO directly

- open(some_path, 'tr') uses:

- TextIOWrapper (text IO operations)

- BufferedReader (readable binary buffer)

- FileIO (writes to the actual file)

- open(some_path, 'bw') uses:

- BufferedWriter (writable binary buffer)

- FileIO (writes to the actual file)
