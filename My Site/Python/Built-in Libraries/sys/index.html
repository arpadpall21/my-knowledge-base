[Ordered: Alphabetically]
Method / Property / statement ect... Description
sys.argv - returns the command line arguments in a list (ex: sys.argv[0] is the script name)
sys.orig_argv - returns the command line arguments in a list including 1st executable (ex: sys.orig_argv is the executable name (python)) 3.10
sys.exit(exitCode:int

0 = normal exit code (any other than 0 considered abnormal exit)

) 		- exits the program (raises a SystemExit signaling exit to the interpreter)
sys.getrecursionlimit() - returns the call stack size for recursions
sys.setrecursionlimit(int) - sets the call stack size for recursions
sys.builtin_module_names - returns module names in a tuple that are compiled into this Python interpreter
sys.stdlib_module_names - returns all standard module names in a frozenset (main packages listed only) (Standard module = modules that we import into our scripts) 3.10
sys.modules - returns all modules loaded by the interpreter in a dictionary (name value pairs) (ex: sys.modules["time"] refers to the time module)
sys.path - returns a settable list which contains directory paths where python will search for modules [link]
sys.stdin - returns a process' standard-in as fileObj [link]
sys.stdout - returns a process' standard-out as fileObj [link]
sys.stderr - returns a process' standard-error as fileObj [link]
sys.getsize(any) - returns the size (memory consumption) of the passed object (in Python everything is an object)
sys.copyright - returns the Python interpreter copyright informations
sys.implementation - returns the Python interpreter implementation details in a named tuple
sys.version - returns the Python interpreter version info in a string
sys.platform - returns the platform identifier
sys.byteorder - returns the system byte order 'little' = little-endiannes (least-significatn byte first), 'big' = big-endiannes (most-significant bytes first)
sys.getdefaultencoding() - returns the default string encoding used by the interpreter
sys.hash_info - returns the hash implementation details in a named_tuple
sys.thread_info - returns the thread implementation info in a named tuple 3.3
sys.executable - returns the absolute path of the Python executable

DEMO

- built-in moudles VS standard modules 

    import sys
    
    sys.builtin_module_names          // -> ('_abc', '_ast', '_codecs', '_collections', ...           // modules at compile time 
    
    sys.stdlib_module_names           // -> ('_abc', '_ast', '_codecs', ...                           // standard modules
