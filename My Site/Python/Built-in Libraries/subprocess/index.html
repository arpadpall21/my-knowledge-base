Method Description
subprocess.run([arg, ...], otherArgumentsLink) - runs the passed arguments in a child process, the child process' streams are linked to the parent process by default
- once finished returns a completeProcess object 3.6
completeProcess.args - returns the arguments that the subprocess was started (in a list)
completeProcess.returncode - returns the status exit code of the subprocess

Remember This :

Description and Demonstration :

- the subprocess module has the ability to start a subprocess from a shell command

- Python v3.5 is great to run child processes, subprocess.run(...) handles all for us (executes the passed command and the child process' pipes are linked to the parent process)

- we can also use the older subprocess.Popen() class to basically do the same, it offers more control (timeout, kill, pipe routing, etc...) but also more complicated to use [link]

DEMO 
    import subprocess
    
    completeProcess = subprocess.Popen(['node', 'test.js'])       # runs the program in a child process (once the child process tarted there's no shell between the parent and child process)    
    
    print( completeProcess.args )                 // -> ['node', 'test.js']
    print( completeProcess.returncode )           // -> None

- Popen 

    import subprocess
    
    process = subprocess.Popen(['node', 'test.js'],
        stdin=subprocess.PIPE,
        stdout=subprocess.PIPE,
        stderr=subprocess.PIPE
    )
    
    std_out, std_err = process.communicate(input=b'data sent in standard in')
    
    print( std_out )            // -> bytes object containing all the subprocess' stdout
