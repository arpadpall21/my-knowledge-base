CMD Description
Lint
ruff check path [options] - lint checks all python files in the current directory (or targeted file)
[options]
- (--watch) continously watches for changes in targeted files
- (--fix) fixes linting errors (modifies targeted files)
Format
ruff format path [options] - formats all python files in the current directory (or targeted file) (modifies targeted files)
[options]
- (--diff) only displays intended file changes (does not modify targeted files)

Remember This :

- [2024-10-22] ruff doesn't properly support all linting rules, this makes things confusing! (ex: the W391 and E303 which are responsible for white spaces are not supported)

- to keep things very simple run 2 commans in order to check for PEP8 a compliant code: ruff check and ruff format --diff

Description and Demonstration :

- ruff is a very performant Python linter and code formater tool, with the intent of replacing all known Python linters (flake8, isort, pydocstyle, pyupgrade, autoflake,) and formaters (black)

- ruff linter & formater:

- linter: code analyzer configurable with (almost) all known Python linter rules (careful not all rules are properly supported check the docs!)

- by default Ruff enables Flake8's F rules, along with a subset of the E rules, omitting any stylistic rules that overlap with ruff format

- formater: formats the code to comply with Black style guidelines (PEP8 compliant)

Configuration [link]

- ruff looks for the closest pyproject.toml or ruff.toml file for configuration 

// pyproject.toml (file) -----
  // ...
  
  [tool.ruff]
  line-length = 120
  
  [tool.ruff.lint]
  extend-select = ["E501"]      // enforces line length
                                // to enable all E and W rules -> extend-select = ["E", "W"]

or 

// ruff.toml (file) -----
  line-length = 120
  
  [lint]
  extend-select = ["E501"]
