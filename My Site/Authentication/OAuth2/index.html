Description and Demonstration :

- OAuth 2.0 is an Authorization protocol meaning we can do specific actions in the behalf of a user but we don't know anyting about the user's credentials

- Open ID Connect (OIDC) is an Authentication extension on top of OAuth 2.0

- Concepts:

- Resource Onwer: User who owns the resource

- Authorization Server: Server handling authorization / authentication

- Resource Server: Server holding the owner's resources

- Client Application: Our app including fortend and backend

- Front Channel: "Less Safe" communication channel where the authorization grant is provided

- Back Channel: "Safe" communication channel where the client backend communicates with the resource server (explicit flow)

- Redirect URI: URL where the client frotnend application is redirected with the Authorization Grant (or token), on successful authentication (usually host/callback)

- Authorization Grant: Authorization code exchangable for Access & Refresh tokens (represents the resource owner's authorization) (explicity flow only)

- Authorization Scope: Scopes (specific actions) the resource owner authorizes the client application to use (ex: for OIDC the scope is openid)

- Response Type: Token types we want to receive (code = Authorization Code, token = Access Token, id_token = ID Token)

- Access, Refresh and ID Tokens: Tokens provided by the protocol (Access and Refresh Tokens for resource access) (ID Token for authentication)
