TAG's
Tag Attribute Description
<caption> defines the table's caption
  - must be inserted after the <table> tag
  - only one per table
<col> span="nr" (span columns for styling only)
 specifies a table's column in the sequence
  - must be child of the <colgroup> element
<colgroup> defines a group for the table columns for formating (nests <col> elements)
  - must be child element of the <table> element
  - must come after the <caption> element and before any <thead>, <tbody>, <tfoot>, <tr> elements
<table> defines a table
<tbody> groups body content in the table required! (you don't need to specify it but the browser automatically include one in the table when it is rendered)
  - used in conjonction with the <thead> and <tfoot> elements
  - must be child element of the <table> element
  - comes after the <caption>, <colgroup> and <thead> elements (according to W3C)
<td> colspan="nr" (span columns)
headers="id" (specified the header relation with other headers [no visual effect])
rowspan="nr" (span rows) 		defines a standard cell in the table
<tfoot> groups footer content in the table
  - used in conjonction with the <thead> and <tbody> elements
  - must be child element of the <table> element
  - comes after the <caption>, <colgroup> and <thead> elements (according to W3C)
  - comes before <tbody>, <th> elements (according to W3C)
<th> abbr="text (specifies an abreviated version of the content)
colspan="nr" (span columns)
headers="id" (specified the header relation with other headers [no visual effect])
rowspan="nr" (span rows)
scope="col / colgroup / row / rowgroup" (specifies header scope [no visual effect]) 		defines a header cell in the table
<thead> groups header elements
  - used in conjonction with the <tbody> and <tfoot> elements
  - must be child element of the <table> element
  - comes after the <caption> and <colgroup> elements
  - comes before <tbody>, <tfoot>, <tr> elements (according to W3C)
<tr> defines a table row

Remember This :

- the <colgroup> element has limited style posibilities, can only style background / border / width / visibility

- the <tfoot> element comes after the <thead> element but before the <tbody> element (according to W3C), I tested by mixing these element and the browser (Chrome) renders them in the right order (thead -> tbody -> tfoot [2018-08-16])

- at least one <tbody> element is required in a table if you don't specify it the browser automatically wraps the table's content in one

Description and Demonstration :

- the <table> element defines a table element which contains other children elements, these elements are independent therefore they can be targeted infividually

- ordinary elements can be rendered as table element by using the display property see: Rendering Elements (CSS) 

SYNTAX :    <table>
                <caption> captionTitle </caption>      // caption comes afther the "<table>" TAG
                <colgroup>                             // colgroup nest col TAGs and comes after the "<caption>" TAG
                    <col>                              // refers to the 1st column
                    <col span="2">                     // refers to the 2nd and 3rd columns
                </colgroup>
                <thead>                                // <thead>, <tfoot>, <tbody> are optional but they must be used in conjonction
                    <tr>                               // you can mix up these elements, browser will render them in the order of <thead> --> <tbody> --> <tfoot>
                        <th> content </th>
                        <th> content </th>
                        <th> content </th>
                    </tr>
                </thead>
                <tfoot>                                // <tfoot> comes after <thead> according to (W3C) but not by MDN
                    <tr>
                        <td> content </td>
                        <td> content </td>
                        <td> content </td>
                    </tr>
                </tfoot>
                <tbody>                                // groups body content Required! (you don't need to specify it but the browser automatically include one in the table when it is rendered)
                    <tr>
                        <td> content </td>
                        <td> content </td>
                        <td> content </td>
                    </tr>
                    <tr>
                        <td> content </td>
                        <td> content </td>
                        <td> content </td>
                    </tr>
                </tbody>
            </table>
    
    <table>
        <caption> myTable </caption>      
        <colgroup>

            <col style="background-color:white;">         // 2nd column background color styled to white

            </tr>
            <tr>
                <td> 1 </td>
                <td> 2 </td>

            </tr>
        </tbody>
    </table>

-- CODE (HTML) --
    <table class="first">
    <caption> Test Table (caption)</caption>

        </tr>
        <tr>
            <td> 1 </td>

            <td> 3 </td>
        </tr>
        <tr>

            <td> 5 </td>
            <td> 6 </td>
        </tr>
        <tr>
            <td> 7 </td>
            <td> 8 </td>
        </tr>
    </table>
