Description and Demonstration :

- global attributes are usable on any HTML elements, although some of them might have no effect in specific situations

- In addition to the basic HTML global attributes the following also exist

  • xml:lang / xml:base inherited from the XHTML specification
  • aira-* multiple attribute for assistive technology

The id / class / tilte / lang / style attributes

DEMO 
// id / class ----------------------------------------------------
-- HTML -- 
    <p id="@yd" class="myClass"> test_paragraph_1 </p>   // id must be unique 
    <p class="myClass"> test_paragraph_2 </p>            // same class identifiers identify multiple elements at once  
    <p class="myClass 2yourClass"> test_paragraph_3 </p> // element is identified by 2 different class ids 
    
-- CSS -- 
    #\@yd { color: green; }                              // escape character    // -! this makes kind of pointless and complicated to use weird identifier names 
    
// title / lang -------------------------------------------------- 
    <p title="myTitle"> this element has a title </p>
    <p lang="hu"> ez magyar nyelven van irval </p>
    
// style --------------------------------------------------------- 
    <p style="color:green;"> inline styled element </p>

The tabindex / accesskey attributes

DEMO

- tabindex 

    <p tabindex="2"> element </p>                                                      // this is a tabbing group element, (when the "Tab" button is hit these elements are the first to get focus on this page in the defined tabbing order!)    
    <p tabindex="3"> element </p>   
    <p tabindex="1"> element </p> 
    
    <p tabindex="0"> element </p>                                                      // element becomes focusable and the browser defines its tabbing order (document flow)
    <p tabindex="-1"> element </p>                                                     // element becomes focusable only! (not reachable by hitting the "Tab" or "Shift + Tab" buttons)  
    
    <a href="d" accesskey="l"> demo lingk </a>                                         // element gets focus when the browser shortcut combination in hit (in Chrome: Alt + key (Alt + l in this situation))     
    <input type="tel" accesskey="t">                                                   // element gets focus when the browser shortcut combination in hit (in Chrome: Alt + key (Alt + t in this situation))

The contenteditable / dir / hidden / inputmode / enterkeyhint / autocapitalize attributes

DEMO 
// contenteditable -------------------------------------------------------------------- 
    <p> uneditable content </p>                                                        // contenteditable value inherited from the parent element automatically    
    <p contenteditable> editable content </p>                                          // element's content is editable  
    <p contenteditable="true"> editable content </p>
    <p contenteditable="false"> uneditable content </p> 
    
// dir --------------------------------------------------------------------------------
    <p dir="rtl"> Writing direction changed to right-to-left </p>                      // element's content is displayed from right-to-left   
    
// contenteditable --------------------------------------------------------------------
    <p hidden> this element is hidden </p>                                             // element is hidden and removed from the layout but NOT removed from DOM   
    
// inputmode / enterkeyhint -----------------------------------------------------------
    <input inputmode="numeric">                                                        // on phones this brings up a numeric keypad (affect virtual keyboards only)   

    <input enterkeyhint="go">                                                          // on phones the numeric keypad the Enter key is displayed as 'Open' icon    
    
// autocapitalize ---------------------------------------------------------------------
    <input autocapitalize="sentece">                                                   // on phones when typing on virtual Keyboard new senteces will automatically start with capital letter

The data attribute

- the "data" attribute stores custom data (string only) which can be handeled through JS

- the set information is stored in the dataset property of the HTML element (called DOMStringMap) of the element

- the property should contain only lowercase letters and dashes are converted to (camel-case JS=> camelCase) in JS

DEMO 
    <p id="demo" data-demoProp1="myValue" data-demoProp2="2"> myParagraph </p>
<script>
    var element = document.getElementById("demo");
    element.dataset;                                                                   // -> {demoProp1:"myValue", demoProp2:"2"}
    
    typeof element.dataset.demoProp1;                                                  // -> 'string'       // data stored as string 
    typeof element.dataset.demoProp2;                                                  // -> 'string'
</script>
