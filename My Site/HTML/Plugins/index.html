Description and Demonstration

- Plugins are not supported anymore (because of security concerns), so all this is deprecated!

- Plug-Ins are additional softwares instaled in the browser in order to run content what the browser cannot run natively (like: Adobe Flash Player, Java Runtime Environment, ActiveX)

- Plug-Ins are not popular in these days (especially on modile browsers) because HTML5 supports these functionalities natively (video, audio, image, canvas, svg, ect..) so the user does not have to bother to download these additional craps in order tho run the enbeded content 

// <object> / <param> ------------------------------------------------------------- 
    <object data="flashPlayerData.swf" type="application/x-shockwave-flash" typemustmatch width="500" height="350">       // Flash player Plug-In required to run this type of content (we could also embed any type of content what is supported by an instaled Plug-In, video, audio, image, svg, html document)    
        <param name="autopaly" value="true">                  
        <param name="volume" value="45%">                     // parameters set 
        <param name="fullscreen" value="false">
        <p> Problem Occured </p>                              // fallback message in case the object is not supported or it cannot run the embeded content 
    </object>
        
// <embed> ------------------------------------------------------------------------ 
    <embed src="flashPlayerData.swf" type="application/x-shockwave-flash" width="500" height="350">                       // -! the <embed> does the same but it offers less tools
