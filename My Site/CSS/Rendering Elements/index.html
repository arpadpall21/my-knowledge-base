The position property

Position Can Be Positioned Affects the Layout Relative To
static - no - yes ---
relative - yes - yes - its original static position
fixed - yes - no - viewport
absolute - yes - no - its parent (the parent element must be positioned relative or absolute or fixed)
sticky - when fixed - yes - rendered with the flow until reaches the offset position by scrolling
- viewport when the scroll tries to hide it away but limited within the parent element's boundaries
Example :

static positiong

- rendered with the document (cannot be positioned)

"Static" Element
CODE : 
// HTML ----------------------------
    <div id="static"> "Static" Element </div>
    
// CSS ---------------------------
    #static{
        position: static;
        background-color: yellow;
        border: 2px solid black;
        padding: 2px;
        left: 100px;                            // -! ignored (static position cannot be positioned)
    }

- relative to its "static" position, can be positioned but leaves a hole because its original "static" place is not removed from the layout

"Relative" Element

    }

    }

sticky position

- the "sticky" element is positioned according to the normal flow of the document, a "sticky" element CANNOT be placed relative to the parent container

- once the sticky element reaches the set treshold (top and bottom) it sticks on the viewport to stay visible inside the parent lement's boundaries

- when the sticky element is in the middle of the screen is positioned relative (normal flow) becuse it is not forced to leave the screen
