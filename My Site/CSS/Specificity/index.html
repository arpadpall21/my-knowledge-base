Description and Demonstration :

- CSS specificity is a value of a css selector

- an element can be selected with a combination of css selectors, these combination add up (ex: p.someClass = precedence value 11)

- if an element is targeted with multiple selectors the one with the highest specivicity value takes precedence (the more specific one)

- 2 important things about specivicity order:

- style declaration or the file import order doen't matter (except when selectors have the same specificity)

- specificity levels cannot be overspecified (ex: one class selector (specificity value 10) is more specific than 11 element selectors (specificity value 11)) 

    Selectors                                                                Specificity Value
                                                                                                                        
    <htmlTag style="cssStyle">       (inline)                                1000
    #id                              (id)                                    100
    .class | [attr] | :pseudoClass   (class | attribute | pseudo-class)      10
    htmlTag | ::pseudoElement        (element | pseudo-element)              1
    *                                (universal selector)                    0
    
    
// !important property --
    - !important property                always take effect regardless of selector specificity
    
  <p id="myElement_1" style="color:green;"> myElement </p>    // specificity value 1000 (inline selector)
  
  
  #myElement { }                    // specificity value 100 (1 id selector)
  
  html body p.myElement { }       // specificity value 13 (3 element / 1 class selectors)
  
  body [id] { }                     // specificity value 11 (1 element / 1 atribute selectors)
  
  body p[id] { }                       // specificity value 12 (2 element / 1 attribute selectors)
  
  p#myElement { }                       // specificity value 101 (1 element / 1 id selectors)   
  
  *[data]                              // specificity value 10 (1 universal / 1 attribute selectors)
  
  
// sepcivicity level cannot be outspecified ------------
  p > span > span > span > span > span > 
  span > span > span > span > span > span { }     // specivicity value 12 (12 element selectors)
  
  .myElement { }                        // -! more specific because the class selector is a higher level selector (even if in this case sepcificity value is 10)
  
  
// !important property ----------
    p#test-test {
        color: blue;
    }
    
    #test-test {
        color: red !important;            // takes precedence (no matter if the selector has lower specificity)
    }

- css file import order or declaration import order doesn't matter 

    p#test-test { }   // style used because it has highest specivicity value 101 (declaration order doesn't matter)
    
    #test-tesst { }
    
// same specificity --
    #test-test { }
    
    #test-test { }                          // style used because both declaration has the same specificity value 100
