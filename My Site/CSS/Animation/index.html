Description and Demonstration :

- CSS allows us to animate most of the HTML elements

The @keyframes roule and animation-name property

- The @keyframes rule describes the animation step by step 

SYNTAX :                                                                                  43 -webkit- / 45  5 -moz- / 16  15 -webkit- / 30  12  4 -webkit- / 9 
            @keyframes animName {                                                      // describes an animation step by step 
                from|0% { cssStyles }                                                  // describes the start animation states  
                
                1%-99%{ cssStyles }                                                    // describes the animation at the 1%-99% stage    
                  
                to|100% { cssStyles }                                                  // describes the end animation states 
            }
DEMO 
// HTML -------------------------------------------------------------------------------
    <p class="myClass" > myParagraph </p>
    
// CSS --------------------------------------------------------------------------------
    .myClass {
        animation-name: myAnimation;                                                   // animation protocol (@keyframe) is called here 
        animation-duration: 4s;
        animation-iteration-count: infinite;
    }
    
    @keyframes myAnimation {                                                           // describes the animation steps by step 
        from {color: black;}
        50% {color:blue;}
        to {color: red;}
    }

The animation-name / animation-duration / animation-delay / animation-iteration-count properties

DEMO 
// HTML -------------------------------------------------------------------------------
    <p class="myElement"> myParagraph </p>
    
// CSS --------------------------------------------------------------------------------
    .myElement {
        position: relative;
        animation-name: myAnimation;                                                   // references the @keyframe roule  
        animation-duration: 10s;                                                       // animation duration (one iteration) is 10 seconds  
        animation-iteration-count: infinite;                                           // endless animation 
        animation-delay: 2s;                                                           // 2 seconds before the animation starts (if value is negative the animation is handeled as it is already playing for the set time)    
    }
    
    @keyframes myAnimation{
        from {left: 0;}
        to {left: 200px;}
    }

The animation-fill-mode property

- the animation-delay is technically already animationy

DEMO 
// HTML -------------------------------------------------------------------------------
    <p class="fillMode"> Fill Moded </p>
    
// CSS --------------------------------------------------------------------------------
    .fillMode {
        position: relative;
        left: 0;                                                                       // -! the element never uses its original position in this case   
        animation-duration: 5s;
        animation-delay: 3s;                                                           // during the delayed period the "left: 50px" is applied because of the "animation-fill-mode: both;"  
        animation-name: fillMe;
        animation-fill-mode: both;                                                     // both the first and last keyframes styles are applied
    }
    
    @keyframes fillMe {
        0% {left: 50px;}
        100% {left: 300px;}                                                            // at the end of the animation this style is preserved because of the "animation-fill-mode: both;"
    }
Example :

TEST

- note that : the animation has a 2 seconds delay before it starts, that is technically a running animation

- elements are set to left:50px; / animation start at left:0; finished left:500px;

animation-fill-mode: forwards

- first keyframe style is NOT applied, during the animation delay period the element stays on its original position left:50px;

- last keyframe left:500px; is applied when animation is finished

animation-fill-mode: backwards

- first keyframe left:0; is applided during the animation delay period

- the last keyframe style is NOT applied, when animation is finished (element jumps to its original position left:50px;)

animation-fill-mode: both

- first keyframe left:0; is applided during the animation delay period

- last keyframe left:500px; is applied when animation is finished
