Bitwise Operators
nr | nr - bitwise OR, performs a 32 bit signed integer bitwise OR operation

1) both nrs are converted to a 32bit signed integer

2) they are compared to each other

3) where one value has 1 and where both values have 1bit that bit is set to 1 in the result

EX: 

                21 | 68;       // -> 85  
                    
                    // 21 (decimal) = 00000000000000000000000000010101 (32 bit signed integer binary representation )   
                    // 68 (decimal) = 00000000000000000000000001000100 (32 bit signed integer binary representation )
                    // 85 (decimal) = 00000000000000000000000001010101 (32 bit signed integer binary representation )
on the nrs 		8

Remember This :

- comparison operators only can compare primitives, comparing objects will alwasy result false, if you want to compare objects use Object.is()

- comparison operators compare strings by their Alphabetical Order (numbers are placed before letters)

- performing bitwise operation on numbers that are larger then 32bit signed integers may result on unexpected behaviours

Description and Demonstration:

- operators using one value called Unary Operators ( typeof 1 )

- operators using two values called Binary Operators ( 1 + 1 )

- operator using three values called Ternary Operator ( (10 > 2) ? "ok" : "not-ok" ) (only one in JS)

Operator Precedence

- operator precedence describes defines the order operations are preformed (equal precedence operators are computed form left to right) 

    var x = 100 + 50 * 3;                                                              // -> 250   // the multiplication have higher precedence
    var x = (100 + 50) * 3;                                                            // -> 450   // precedence changed
    var x = 100 + 50 - 3;                                                              // -> 147   // equal precedence operators are computed from left to right

Comparison Operators == / === / != / !== / < / <= / > / >=

- comparison operators compare only primitive values, therefore 2 objects cannot be compared (comparign object always returns false)

DEMO 
    myObj == myObj                                                                     // -> false             // -! cannot compare complex data types (always returns false)    
    
    "5" == 5                                                                           // -> true
    "5" === 5                                                                          // -> false             // datatypes are not equal
    "5" != 5                                                                           // -> false             // both values are equal
    "5" !== 5                                                                          // -> true              // datatypes are not equal
    
    "3" < 5                                                                            // -> true              // no datatype test
    "5" <= 5                                                                           // -> true              
    "3" > 5                                                                            // -> false              
    "5" >= 5                                                                           // -> true
    
    "abc" < "def"                                                                      // -> true              // strings are compared in Alphabetical Order 
    "apa" < "anya"                                                                     // -> false
    "a" > 15                                                                           // -> false             // letter compared to a number always "false"
    "a" > "15"                                                                         // -> true              // numbers are placed before letters when it comes to an Alphabetical comparison

Logical Operators && / || / ! / ??

DEMO 
// logical AND (&&) -------------------------------------------------------------------
    "myStr" && "myStr2";                                                               // -> "myStr2"     // left exression evaluated true -> returns the right expression   
    
    0 && nothing;                                                                      // -> 0            // left exression evaluated false -> returns the left expression  
                                                                                                          // -! right expression 'nothing' never get evaluated (short circuiting)     
    
// logical OR (||) --------------------------------------------------------------------
    null || "myStr";                                                                   // -> "myStr"      // left expression evaluated false -> returns the right expression    
    
    "myStr" || nothing;                                                                // -> 0            // left expression evaluated false -> returns the left expression    
                                                                                                          // -! right expression 'nothing' never get evaluated (short circuiting)     
    
// logical NULLISH (??) ---------------------------------------------------------------
    null ?? "myStr";                                                                   // -> "myStr"      // left expression evaluated null or undefined -> returns the right expression    
    undefined ?? "myStr";                                                              // -> "myStr" 
    
    false ?? nothing;                                                                  // -> false        // left expression evaluated not null or undefined -> returns the left expression    
                                                                                                          // -! right expression 'nothing' never get evaluated (short circuiting)     
    
// logical NOT ------------------------------------------------------------------------
    !21                                                                                // -> false        // expression evaluated true -> returns false    
    !null                                                                              // -> true         // expression evaluated false -> returns true
