ArrayBuffer and SharedArrayBuffer object
Method / Property Description
ArrayBuffer
new ArrayBuffer(length:nr) - creates a 0 filled ArrayBuffer object instance of the specified size in bytes
- the ArrayBuffer object represents a fixed length RAW binary data buffer (called 'Byte Array' = 1 address represents 1 byte)
- it cannot be modified directly (the DataView object can get and set it or use one of the TypedArrays)
|
ArrayBuffer.isView(arg) - returns true if the passed argument is eighter a TypedArray or a DataView object, otherwise returns false
|
arrayBuffer.byteLength - returns the length of the ArrayBuffer in bytes
arrayBuffer.slice(st(inc):index, end(exc):index) - returns a new ArrayBuffer object copy whose content is between the specified byte range (end omitted = all remaining data)
|
SharedArrayBuffer 68 79 55 79 15.2 8.10
new SharedArrayBuffer(length:nr) - creates a 0 filled SharedArrayBuffer object instance of the specified size in bytes
- the SharedArrayBuffer object represents a fixed length RAW binary data buffer (called 'byte array' = 1 address represents 1 byte)
- it cannot be modified directly (the DataView object can get and set it or use one of the TypedArrays)
- unlike the ArrayBuffer the SharedArrayBuffer shares its memory between threads and cannot be transfered from one thread to another

SharedArrayBuffer

- shared memory (ex: main.js worker_1.js worker_2.js sees the same underlying memory 'SharedArrayBuffer')

- not transferable (ex: the SharedArrayBuffer cannot be transfered from main.js to worker_1.js only shared!)

ArrayBuffer

- cloned memory (ex: main.js worker_1.js worker_2.js gets individual copies 'clones' of the send ArrayBuffer)

- transferable memory (ex: the transfered ArrayBuffer from main.js to worker_1.js is moved so it won't be visible in main.js anymore)

|
sharedArrayBuffer.byteLength - returns the length of the SharedArrayBuffer in bytes
|
sharedArrayBuffer.slice(st(inc):index, end(exc):index) - returns a new SharedArrayBuffer object copy whose content is between the specified byte range (end omitted = all remaining data)

DataView object
Method / Property Description
new DataView(arrayBuffer, byteOffset:index, byteLength) - returns a DataView object which is a low-level interface for reading and writing an ArrayBuffer (or part of it)
- converts integers, BigIntegers and floating point numbers to binary data and vice versa (handles endianness regardless of the platform's architecture)
|
dataView.buffer - returns the original ArrayBuffer object referenced by this DataView object
dataView.byteLength - returns the length in bytes of this DataView object
dataView.byteOffset - returns the offset in bytes of this DataView object (index where the original ArrayBuffer is viewed from)
|

TypedArray
Method / Property Description
TypedArray - Array-like view of the underlying raw data (ArrayBuffer)
- ex: Int8Array gets or sets the underlying raw data (ArrayBuffer) as 8bit Integers
new TypedArray() - empty (new in ES2017)
new TypedArray(length:nr) - item length (ex: new Int8Array(1) = allocates 1 byte / new Int32Array(1) = allocates 4 bytes)
new TypedArray(TypedArray) - the passed TypedArray passes its items and not the underlying ArrayBuffer
new TypedArray(iterableObj) - object impelmenting the 'iterable' symbol interface
new TypedArray(arrayBuffer, byteOffset:index, elementLength:nr) - ArrayBuffer object shares the memory with the newly created TypedArray
|
TypedArray types
Int8Array - the underlying binary data is represented as 8bit Signed Integer value (-128 to 127 (decimal numbers are floored when set)) (1 byte = 1 Array item)
Uint8Array - the underlying binary data is represented as 8bit Unsigned Integer value (0 to 255 (decimal numbers are floored when set)) (1 byte = 1 Array item)
Uint8ClampedArray - the underlying binary data is represented as 8bit Unsigned Integer value (0 to 255) (1 byte = 1 Array item)
- setting an off range value will be clamped to the closest valid value (ex: any negative will be set as 0), decimal numbers are rounded when set
|
Int16Array - the underlying binary data is represented as 16bit Signed Integer value (-32768 to 32767 (decimal numbers are floored when set)) (2 bytes = 1 Array item)
Uint16Array - the underlying binary data is represented as 16bit Unsigned Integer value (0 to 65535 (decimal numbers are floored when set)) (2 bytes = 1 Array item)
|
Int32Array - the underlying binary data is represented as 32 bit Signed Integer value (-2147483648 to 2147483647 (decimal numbers are floored when set)) (4 bytes = 1 Array item)
Uint32Array - the underlying binary data is represented as 32 bit Unsigned Integer value (0 to 4294967295 (decimal numbers are floored when set)) (4 bytes = 1 Array item)
Float32 - the underlying binary data is represented as 32 bit Floating Point value (1.2*10**38 to 3.4*10**308) (4 bytes = 1 Array Item)
|
BigInt64Array - the underlying binary data is represented as 64bit Signed BigInteger value (-9223372036854775808n to 9223372036854775807n (no decimals allowed)) (8bytes = 1 Array Item) 67 68 54 79 15 10.4
BigUint64Array - the underlying binary data is represented as 64bit Unsigned BigInteger value (0 to 18446744073709551615n (no decimals allowed)) (8bytes = 1 Array Item) 67 68 54 79 15 10.4
Float64Array - the underlying binary data is represented as 64bit Floating Point value (5.0*10**-324 to 1.8*10**308) (8bytes = 1 Array Item)
|
Properties and Methods
typedArray.buffer - returns the underlying binary data (ArrayBuffer object)
|

|
typedArray.arrayMethod - TypedArrays are basically Arrays, so any Array method works on them as well Array methods [link]
|

typedArray.byteLength - returns the length in bytes of the typed array
|
typedArray.byteOffset - returns the byte offset from the underlying ArrayBuffer (when TypedArray is created by an ArrayBuffer)
typedArray.set(iteralbeObj|typedArray, stIndex(inc)) - sets multiple values (by using the passed object's values) from the specified index
|
typedArray.subarray(stIndex(inc), endIndex(exc)) - returns a typedArray which is a view (of the targeted region) of the TypedArray

ArrayBuffer / SharedArrayBuffer / DataView

DEMO 
    var buffer = new ArrayBuffer(128);                                                 // we allocated 128 bytes of RAW data (empty = 0 filled)
    var dataView1 = new DataView(buffer);                                              // this data view sees the full ArrayBuffer 
    var dataView2 = new DataView(buffer, 4, 64);                                       // this one views the ArrayBuffer from 4th index (included) 64 bytes   
    
// (ArrayBuffer) isView() / byteLength / slice() --------------------------------------
    ArrayBuffer.isView(dataView2);                                                     // -> true                 // passed object is a DataView or TypedArray 
    buffer.byteLength;                                                                 // -> 128                  // ArrayBuffer contains 128 bytes 
    newBuffer = buffer.slice(32, 64);                                                  // -> ArrayBuffer(32)      // slices 32 bytes from the ArrayBuffer and returns it in a new ArrayBuffer object   
    
// (DataView) buffer / byteLength / byteOffset ----------------------------------------
    dataView2.buffer;                                                                  // -> ArrayBuffer(128)     // returns the referenced ArrayBuffer object (ArrayBuffer which will be read and written)   
    dataView2.byteLength;                                                              // -> 64                   // the byte length of this data view 
    dataView2.byteOffset;                                                              // -> 4                    // this dataView object views the referenced ArrayBuffer from 4th index  
    
// read / write the ArrayBuffer -------------------------------------------------------
    dataView1.setUint8(0, 241);                                                        // writing the 1st 8 bits of the ArrayBuffer as Unsigned Integer value 
    dataView1.getUint8(0);                                                             // -> 241                  // reads the 1st 8 bits of the ArrayBuffer as Unsigned Integer value (returns the set data)   
    dataView1.getInt8(0);                                                              // -> -15                  // the same data is interpreted differently if we read as an 8bit Unsigned Integer  
    
    dataView1.setFloat64(1, 145.2244, true);                                           // writing 64 bits from the 2nd byte of the ArrayBuffer as 64bit Floating point    // -! written as little endianness 
    dataView1.getFloat64(1, true);                                                     // -> 145.2244             // read the written data                                // -! read as little endianness 
    
    dataView2.setBigUint64(0, 6549874654165165n);                                      // writing 64 bits from the 4th byte of the ArrayBuffer as 64bit Big Integer       // -! as BigInt datatype  
    dataView2.getBigUint64(0);                                                         // -> 6549874654165165n    // read the written data                                // -! as BigInt datatype  
    
// CAVEATS ----------------------------------------------------------------------------
    dataView1.setUint16(0, -100);                                                      // writting an invalid value will give unexpected result (Unsigned Integer = 0 to 65535)    
    dataView1.getUint16(0);                                                            // -> 65436                // unexpected value 
    
    dataView1.setUint16(0, 2.999);                                                     // decimal numbers will be floored if set as integers 
    dataView1.getUint16(0);                                                            // -> 2                    // value floored

ArrayBuffer VS SharedArrayBuffer 

// ------------------------------------------------------------------------------------
// main.js ----------------------------------------------------------------------------
    var workers = require('worker_threads');
    
    var worker_1 = new workers.Worker('./worker_1.js');
    
    var shArrBuf = new SharedArrayBuffer(8);                                           // SharedArrayBuffer created 
    new DataView(shArrBuf).setFloat64(0, 1111.1111);                                   // set data to the SharedArrayBuffer
    
    worker_1.postMessage(shArrBuf);                                                    // the posted SharedArrayBuffer memory is shared between threads (NOT cloned)
        
    setTimeout(function(){
        console.log( new DataView(shArrBuf).getFloat64(0) );                           // -> 9999.9999     // -! since the SharedArrayBuffer memory is shared between threads modifications in 'worker_1.js' also visible here   
    }, 1000);
    
// ------------------------------------------------------------------------------------
// worker_1.js ------------------------------------------------------------------------
    var workers = require('worker_threads');
    
    workers.parentPort.on('message', function(buf){
        new DataView(buf).setFloat64(0, 9999.9999);                                    // modifying the received SharedArrayBuffer 
    });
    
// ------------------------------------------------------------------------------------
// main.js ----------------------------------------------------------------------------
    var workers = require('worker_threads');
    
    var worker_1 = new workers.Worker('./worker_1.js');
    
    var shArrBuf = new ArrayBuffer(8);                                                 // ArrayBuffer created 
    new DataView(shArrBuf).setFloat64(0, 1111.1111);                                   // set data to the ArrayBuffer
    
    worker_1.postMessage(shArrBuf);                                                    // the thread receives a clone of this ArrayBuffer (NOT a shared memory)    
    
    setTimeout(function(){
        console.log( new DataView(shArrBuf).getFloat64(0) );                           // -> 1111.1111     // -! since the ArrayBuffer is cloned each thread has a copy of this ArrayBuffer, so the ArrayBuffer in this trhead cannod be modified by another thread     
    }, 1000);
    
// ------------------------------------------------------------------------------------
// worker_1.js ------------------------------------------------------------------------
    var workers = require('worker_threads');
    
    workers.parentPort.on('message', function(buf){
        new DataView(buf).setFloat64(0, 9999.9999);                                    // modifying the received ArrayBuffer 
    });

TypedArrays

- TypedArrays are Array-like view of the underlying binary data buffer (the underlying ArrayBuffer)

- there is not directy visible TypedArray constructor, instead there are global properties what implements typed Arrays for specific element types

- TypedArrays use the platform's hardware endianness to get or set the underlying ArrayBuffer

DEMO 
    new Uint8ClampedArray(4);                                                    // -> Uint8ClampedArray[0, 0, 0, 0]    // new TypedArray created with the specified length (empty)  
    
    new Int16Array(new Int16Array([475, 1244, -474]))                            // -> Int16Array[475, 1244, -474]      // new TypedArray created by using another TypedArray  
    
    new Float32Array([415.1, 124.22, 142.03]);                                   // -> Float32[415.1, 124.22, 142.03]   // new TypedArray created by using an iterable object   
    
    new BigUint64Array(new ArrayBuffer(16));                                     // -> BigUint64Array[0n, 0n]           // new TypedArray created by passing an ArrayBuffer   
    
    var uint8Clamped = new Uint8ClampedArray(4);
    var float32 = new Float32Array([142.12, 12.4, 145.2]);
    
// set() / subarray() / other array methods ------------------------------------
    uint8Clamped.set([-1454, 300, 32.8], 1);                                    // set to = [0, 0, 255, 33]             // sets the typedArray values from the 1st index (clamped array = clamps offset values and rounds decimals!)  
    var subArr = uint8Clamped.subarray(1, 3);                                   // -> Uint8ClampedArray[0, 255]         // returns a view of the targeted region
    subArr[1] = 0;                                                              // set TypedArray items through the view   
    uint8Clamped;                                                               // -> Uint8ClampedArray[0, 0, 0, 33]    // original TypeArray is modified 
    
    uint8Clamped.includes(33);                                                  // -> true                              // all Array methods are available on TypedArrays as well!   
    
// BYTES_PER_ELEMENT / name / buffer / byteOffset / byteLength / length --------
    BigUint64Array.BYTES_PER_ELEMENT;                                           // -> 8                                  // constructor builds a typedArray which contains 8 underlying bytes per array item    
    BigUint64Array.name;                                                        // -> 'BigUint64Array'                   // constructor's name 
       
    float32.buffer;                                                             // -> ArrayBuffer(12){}                  // returns the underlying ArrayBuffer object (binary raw data)   
    float32.byteOffset;                                                         // -> 0                                  // typedArray offset relative to the underlying ArrayBuffer    
    
    float32.length;                                                             // -> 3                                  // number of array items 
    float32.byteLength;                                                         // -> 12                                 // number of bytes of the underlying binary data
