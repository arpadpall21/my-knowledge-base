Description and Demonstration:

- JavaScript is an OOP (Object Oriented Programing) language officialy ECMA-262, developed to powers web browser scripting

- JavaScritp is a high-level single threated programming language (does 1 thing at the time only)

- JS executes tasks what are queued in the "Call Stack"

- JS send tasks to APIs (they opperate on their own thread!), once the API finishes the result is placed in the "Callback Queue"

- there are 2 calback queues in the browser JavaScript, Micro Task Queue and Macro Task Queue

- Micro Task Queue has promises and has higher precedence over Macrotask Queue

- Macro Task Queue all other async fucntions (DOM events, Ajax calls, etc...)

- the "Event Loop" opperates between the "Call Stack" and the "Callback Queue", if the "Call Stack" becomes empty the "Event Loop" moves the next queued task form the "Callback Queue" to the "Call Stack" (callled 1 Tick), once the task is executed the next task is moved to the Call Stack untill the Evelnt Loop becomes empty

Variables and Scoping

- JavaScript variables are identified by unique names, these unique names are called "identifiers"

- JavaScript uses Lexical scope only (does not use any Dynamic scopes)

- Lexical Scope is created at write-time, whereas Dynamic Scope (and this) are created at runtime 

SYNTAX :    var|let|const identifier

                
 - can contain a-z|A-Z (letters) / 0-9 (digits) / $ (dollar signs) / _ (underscore) 

                
 - case sensitive and cannot start with a number 

                
 - reserved words (keywords) cannot be used (like: for) 

            
 = data                                           // link to var let const keywords [link]
            
            var|let|const varName [= var|let|const varName = ...] = val                // assingns a value to multiple variables

- a scope is basically an isolated place where variables are isolated from other scope variables

- any child scope can access its parents variables, but parent scopes cannot access their children sope variables

- we have 3 scope types in JavaScript, global scope, function scope and block scope

- the var variable is function scoped, the let and const variables are block scoped

- scope 

// var --------------------------------------------------------------------------------
    var x = 1;                                                                         // global scope 
    
    function test(){                                                                   // function scope (also block scope)
        var y = 2;                                                                     // function scoped 
        x;                                                                             // -> 1
    }
    test();
    
    
    {                                                                                  // block scope 
        var z = 3;                                                                     // -! the var keyword is not block scoped    
        x;                                                                             // -> 1        
    }

Hoisting

- in JavaScript all declarations are hoisted (moved to the top of the current scope) (initializations, definition are not hoisted)

- it's recommended to place variables on the top of the current scope 

    a;                                                                                 // -> undefined             // variable declaration is hoisted
    b;                                                                                 // -> undefined             // -! variable declaration is hoisted but the value initialization is not
    c;                                                                                 // -> throws a Reference Error because the let is not a declaration (therefore is not hoisted)    
    
    var a;                                                                             // this is a declaration;
    var b = 22;                                                                        // declaration + initialization (the initialization part is not hoisted)     
    const c = 23;                                                                      // not a declaration

Strict Mode 13 4 12.1 10 6 0.1.14

- JavaScript strict mode is a mode to better check for errors, it basically turns "bad syntax" into real errors 

SYNTAX :    "use strict"                                                               // 5 enables strict mode at the beginning of the current scope
                                                                                          undeclared variables are not allowed (var keyword required)   
                                                                                          deleting variables, objects or functions with the delete keyword is not allowed 
                                                                                          deleting undeletable property is not allowed (ex: delete Object.prototype)    
                                                                                          duplicate arguments is not allowed (ex: function(a, a){})   
                                                                                          octal literals or octal escape characters are not allowed (ex: 010)    
                                                                                          setter or getter only properties are not allowed 
                                                                                          using the eval() method to create variables is not allowed 
                                                                                          using the with() method is not allowed
DEMO 
    "use strict";                               // strict mode in global scope (local sopes are running in strict mode to in this case!)
    
    function myFunc() {
        "use strict"                            // using strict mode in local scope (in this case the global scope is NOT running in strict mode)
    }

- words not allowed as variable names in strict mode 

    arguments       interface       private         static
    eval            let             protected       yield
    implements      package         public

- undeclared variables will cause error in Strict Mode 

    x = "this string";                          // throws an error in strict mode
    y = {Mark:"Audi", Model:"A8"};              // throws an error in strict mode

- deleting variable, object or function is Not Allowed in Strict Mode 

    var x = 21;
    var y = {p1:10, p2:20};
    function z(a, b){};
    
    delete x;                                   // throws an error in strict mode
    delete y;                                   // throws an error in strict mode
    delete z;                                   // throws an error in strict mode

- duplicating parameters is Not Allowed in Strict Mode

    function myFunc(a, a){}                     // throws an error in strict mode

- octal literals 010 and octal escape characters \"010 are Not Allowed in Stirct Mode 

    var x = 010;                                // throws an error in strict mode (octal literal)
    var y = "\045";                             // throws an error in strict mode (octal escape literal)

- set read-only and get-only properties are Not Allowed in Strict Mode 

// read-only property -------------------- 
    var myObj = {prop1:"myVal"};
    Object.defineProperty(myObj, "prop1", {writable:false});    // this won't throw an error (it is OK to have read-only property)
    myObj.prop1;                                // -> "myVal"   // we can read read-only property in strict mode no problem!
    myObj.prop1 = "modifiedVal";                // throws an error in strict mode (read-only property cannot be modified)
    
// get-only property ---------------------
    var myObj = {get prop1() {return "value what you get"}};    // the "prop1" is set as getter-only propearty (no "set" specified)
    myObj.prop1;                                // -> "value what you get"      // we can read the get-only property no problem!
    myObj.prop1 = "set new value";              // throws an error in strict mode (because the property has no setter function)

- deleting an undeletable property in Strict Mode is Not Allowed 

    delete Object.prototype;                    // throws an error in strict mode

- using the eval() function to create variables is Not Allowed in Strict Mode 

    eval("var x = 'this variable'");            // throws an error in strict mode

- using the with statement is Not Allowed in Strict Mode 

    with (Math){                                // throws an error in strict mode  
        x = cos(3 * PI) + sin (LN10)    
        y = tan(14 * E)  
    }

Explicit VS Implicit data coerction

- Explicit coerction when we intentionally convert a data type

- JavaScript tries to convert the 'wrong' data type into the 'right' data type on operation this is called implicit coerction in JavaScript

- JavaScript uses the + operatior for both additon and concatenation, addition for Numbers and concatenations for Strings

- JavaScript Operates from left to right, any number after a string will be concatenated in a statement!

// explicit coerction -----------------------------------------------------------------
    var x = '1';
    
    var y = Boolean(x);                                                                // -! explicit coerction (we intended to convert the data type)
    
    y;                                                                                 // -> true 
    typeof y;                                                                          // -> 'boolean'
    
// implicit coerction -----------------------------------------------------------------
    var x = '2' * 2;                                                                   // -!
    
    x;                                                                                 // -> 4 
    typeof x;                                                                          // -> 'number'
    
// the + operator ---------------------------------------------------------------------
    1 + 1 + 1;                                                                         // -> 3              // when used with numers performs addition 
    
    1 + '1'                                                                            // -> '11'           // data after the '+' operator is concatenated  
    1 + 1 + '1' + 1 + 1;                                                               // -> '2111'
