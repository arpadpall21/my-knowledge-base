DNS Reccords
Record Description
A - (Address) maps a domain name to an IPv4 IP address
AAAA - (Quad A Record) maps a domain name to an IPv6 IP address
CNAME - (Canonical name) - resolves a domain or subdomain to another domain (basically aliases a domain to a another)
- comon usage is to resolve a subdomain to domain (ex: www.exampleSilte.com aliases to exampleSite.com)
|
MX - (Mail Exchanger) maps your domain to an email servcie provider (ESP) domain
- priority level can be set, the lower value has higher prioritiy (ex: 0 = highest priority)
- there are usually several records with different ESPs and priorities set
|
SRV - (Service) contains a record for a specific service
- it's used along with other records (like A, MX records) in order to point to a specific service
- formated as _service._proto.name. TTL class type of record priority weight port target. (ex: _xmpp._tcp.example.com. 86400 IN SRV 10 5 5223 server.example.com.)
|
TXT (Text) simple text records (used for authentication since you are the only one who can access to your DNS records)
|
PTR - (Pointer) maps an IP address to a fully-qualified domain name (oposite of the A record)
- used in email networking in order to determin if an email server is legit
|
NS - (Name Server) stores an Authoritative Name Server domain (where these DNS records are stored)
- it's usually configureb by the domain registrar
SOA - (Start Of Authority) stores administrator informations about the DNS zone (managed by DNS administrators)
- contains the primary name server domain, mail for the domain administrator and the domain serial number
CAA records - (Certification Authority Authorization) specifies which Certificate Authorities are allowed to issue certificates for a domain
- if nothing is specified all Authorities are allowed to issue certificats

Hierarchy of DNS Servers

- DNS Resolver

- the client request this server for domain name resolution

- it's job is to request Root -> TLD -> Authoritative servers for domain name resolution and return the result to the client

- also work as cache (caches DNS records specified by the record's TTL)

- DNS Server Hierarchy :

- domain names are resolved from right to left by a set of DNS hierarchy network

- Root Servers store TLD Server IP addresses (there are only 13 in the word [2024-06-11])

- TLD Servers (Top Level Domain) store all sites ending with the top level domain code (ex: the .com server stores all .com sites, the .net server stores all .net sites, ect... )

- Authoritative Name Server (Hierarchy) contain the actual DNS records (this is where Registrars register domains)

DNS resolution flow

- request 1 - the user makes a DNS request to the DNS Resolver (if the DNS resolver finds the requested domain in its cache responds to the client response 8)

- request 2 & response 3 - the Root Server provides the apropriate TLD Servers IP address to the DNS Resolver

- request 4 & response 5 - the TLD Server provides the apropriate Authoritative Servers' IP address to the DNS Resolver

- request 6 & response 7 - the Authoritative Servers provide the acutal IP address of the requested site to the DNS Resolver

- response 8 - the DNS Resolver sends the domain's IP address to the client

Example resolution of www.someSite.com.

- . - resolved by a Root Server (the . after the domain name is hidden by the user)

- .com - resolved by a .com TLD (Top Level Domain) Server

- www.someSite - resolved by the Authoritatieve Server (this is where the domain is registered)

- DNS Zones

- DNS Zones are hierarchy level logical groupings of domain name parts (these are basically namespaces at different hierarchy level)

DNS Records

- DNS records are data entries stored in a DNS server's database that associate domain names with IP addresses
