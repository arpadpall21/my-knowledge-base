IPv4
Address Description
[0-255].[0-255].[0-255].[0-255] - IPv4 address represented by 4 octets
|
10.0.0.0/8 - (Class A) private IP range (16,777,216 addresses available)
172.16.0.0/12 - (Class B) private IP range (1,048,576 addresses available)
192.168.0.0/16 - (Class C) private IP range (65,536 addresses available)
224.0.0.0 to 239.255.255.255 - (Class D) Multicasting address range
- IANA defined that a multicast address must start with 1110 bits
- (244 = 111000000 bit, 239 = 11101111 bits) that's the reason for the funny address range
|
127.0.0.0/8 - Loopback IP range (large range but we only use 127.0.0.1 = localhost)
- network adapter self test (packets never sent through the network)
|
169.254.0.0/16 - Link-Local IP range = APIPA (Automatic Private IP Addressing)
- the host self assigns it (in case there's no DHCP)
|
firstIpAddress - the first IP address is the network identifier (mostly used to calculate the available IPs on the network)
- ex: in a subnetwork 192.168.0.0/16 the IP 192.168.0.0 is the network id
lastIpAddress - the last IP address is the (local) broadcast address
- ex: in a subnetwork 192.168.0.0/16 the IP 192.168.0.255 is the broadcast address
- broadcasting messages will be sent to all addresses on the network (UDP protocol)
|
0.0.0.0 - alias for the current host address
255.255.255.255 - alias for the network broadcast address

IPv6
Address Description
4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f] IPv6 represented by 8 blocks of 4 hextets
|
(2|3)ggg:gggg:gggg:ssss:iiii:iiii:iiii:iiii - Global Unicast is a public IPv6 address routable on the internet
- 2|3 - prefix (3bits)
- g - global routing (45bits) = globally assigned by IANA (global hierarchical assignement)
- s - subnet id (16bits) = identifies the subnet, assigned by the Administrator (network hierarchies can be defined at this level)
- i - interface id (64bits) = unique host identifier (calculated randomly (on Windows), based on the MAC address (on Linux))
|
(fc|fd)00:gggg:gggg:ssss:iiii:iiii:iiii:iiii - Unique Local is a private IPv6 address, used in private networking like IPv4 Class A|B|C (not routable on the internet)
- fd00|fc00 - prefix (8bits)
- g - global id (40bits) = assigned by the Administrator
- s - subnet id (16bits) = identifies the subnet, assigned by the Administrator
- i - interface id (64bits) = indivudual node addresses on the subnet, assigned by the Administrator
|
fffs:gggg:gggg:gggg:gggg:gggg:gggg:gggg - Multicast Address
- ff - prefix (8bits)
- f - flag (4bits)
- s - scope (4bits)
- g - group id (112bits)
|
fe80:0000:0000:0000:iiii:iiii:iiii:iiii%id - Link-Local IP range = APIPA (Automatic Private IP Addressing)
- the host self assigns it (in case no DHCP)
- fe80 - prefix (10bits)
- i - interface id (64bits) = device id
- %id - scope id (on windows)
|
::1 - loopback address, network adapter self test (packets never sent through the network)
:: - alias for the current host address

Description and Demonstration :

- MAC (Media Access Control) address:

- physical address of a network interface (statically assigned -> unchangable)

- used on OSI Layer 2 packet delivery (hop-to-hop)

- IP (Internet Protocol) address:

- digital address of a network interface (dynamically assigned)

- IANA (Internet Assigned Numbers Authority) is the authority that decides IP address range distribution (ex: 172.16.0.0/12 is a class B private IP range)

- used on OSI Layer 3 packet delivery (end-to-end)

IPv4

- an IPv4 is made of 32bits divided into 4 blocks, each block is composed of 4bits (each block is represented by one octet (0-255) 

    [0-255].[0-255].[0-255].[0-255]                                                // IPv4 address represented by 4 octets

Subnet

- IANA defined special IP addresses what can be used for private networks, private IP addresses are not routable on the internet

- There are 3 IPv4 subnet classes (Class A 10.0.0.0/8 / Class B 172.16.0.0/12 / Class C 192.168.0.0/16)

IPv4 Subnet Mask

- the subnet mask basically divides the IP addres into 2 parts, expressed in bits counted from left

- the left part identifies the network and the right part identifies the device

- this is where the sender device knows if the destination is on its network or on another network (sends to the gateway) 

    ipAddress/subnetMask                                                           // the subnetMask defines how many bits identifies the network (counted from left), the right part is the device identifier    
        
    [0-255].[0-255].[0-255].[0-255]                                                // subnet mask octet representation   
    
    10.1.1.102/8                                                                       // IP has an 8bit subnet mask (10 = network id) (1.1.102 = host id)
    255.0.0.0                                                                          // subnet mask   // underlying bits -> 11111111.00000000.00000000.00000000
    
// ------------------------------------------------------------------------------------
    10.1.1.102/16                                                                      // IP has a 16 bit subnet mask (10.1 = network id) (1.102 = host id)
    255.255.0.0                                                                        // subnet mask   // underlying bits -> 11111111.11111111.00000000.00000000
    
// ------------------------------------------------------------------------------------
    10.1.1.102/20                                                                      // IP has a 20 bit subnet mask (10.1.1 =network id) (1.102 host id)  // -! the 3rd octet (1) is halved    
    255.255.240.0                                                                      // subnet mask   // underlying bits -> 11111111.11111111.11110000.00000000
    
// ------------------------------------------------------------------------------------
    10.1.1.102/32                                                                      // IP has a 32bit subnet mask    // -! point to point link
    255.255.255.255                                                                    // subnet mask   // underlying bits -> 11111111.11111111.11111111.11111111
Example :

- our device IP address is 192.122.1.102/16

- if we send data to an address 192.122.1.104 the sender device will know that the destination is in our LAN because the target's subnet mask portion corresponds with the sender device

- if we sent data to an address 192.123.1.104 the sender device will know that the destination is not in our LAN because the target's subnet mask does not correspon with the sender device

Broadcasting

- the network's last IP is used for broadcasting, sending (requesting) UDP packets to the broadcast address will be forwarded to all addresses on the network

Broadcasting types:

Local Broadcast - broadcasting messages on the current network

Directed Broadcast - broadcasting messages on a foreign network

Multicasting

- multicasting is basically a pub/sub pattern on a IPv4 network (clients can subscribe to different channels)

- the IGMP (Internet Group Management Protocol) is used to enable multicasting, it's configured on OSI layer 2 and 3 (we configure the switch (enable IGMP Snooping))

- TTL (Time To Live) in broadcasting determines how many hops the message is allowed to travel

IPv6

- an IPv6 is made of 128bits divided into 8 blocks, each block is composed of 16bits (each block is represented by 4 hextets (0-f))

- the IPv6 is a different network, so we cannot send data to an IPv4 network from an IPv6 network (without transition technolgies like 6to4, ISATAP, Teredo) 

    4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]                // IPv6 represented by 8 blocks of 4 hextets  
        
// IPv6 abbreviation rules ---------------------
  // block leading 0's can be omitted ----------------------------------
    48c7:0004:0000:a4d4:0000:0000:0c00:0001                                        
    48c7:4:0000:a4d4:0000:0000:c00:1                                               
    
  // block containing consecutive 0's abbreviated to one single 0 ------
    48c7:0004:0000:a4d4:0000:0000:0c00:0001
    48c7:0004:0:a4d4:0:0:0c00:0001                                                 
        
  // two or more blocks containing consecutive 0's are replaced with :: (abbreviation allowed only once!) (the longer 0 sequence is abbreviated if there are several!) 
    48c7:0004:0000:a4d4:0000:0000:0c00:0001
    48c7:0004:0000:a4d4::0c00:0001                                                    
        
  // ex: fully abbreviated IPv6 ----------------------------------------
    48c7:0004:0000:a4d4:0000:0000:0c00:0001
    48c7:4:0:a4d4::c00:1

IPv6 Routing on the Internet

- we use IPv6 Global Unicast address type to route traffic on the internet, because each device has its own IP address no DHCP and NAT/PAT needed 

    (2|3)ggg:gggg:gggg:ssss:iiii:iiii:iiii:iiii                                        // Global Unicast (has fixed subnet mask /64)
    
    2|3ggg:gggg:gggg      // globally assigned by IANA (ISP provide us this part)
    ssss                  // assigned by the administrator (network hierarchies can be assigned here)
    iiii:iiii:iiii:iiii   // unique interface id

IPv6 Transition Technologies

- ISATAP (Intra-site Automatic Tunnel Addressing Protocol) router allows an IPv6 network to communicate with an IPv4 network (and vice versa) private networks only

- 6to4 allows the IPv6 traffic through an IPv4 network (cannot tunnel IPv4 traffic through IPv6 network)

- Teredo from an IPv4 network we connect to a Teredo Server/Relay which transform the traffic to IPv6
