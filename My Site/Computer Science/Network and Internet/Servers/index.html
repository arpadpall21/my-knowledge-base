DNS Reccords
Record Description
TTL (Time To Live) defines how long (seconds usually) the DNS record should be cached by the requested host (fully explained

- when a machine requests the DNS a domain name resolution the record's TTL tells the requested machine for how long it should keep the DNS record cached

- let's say the www.someSite.com domain has a 1440 TTL (expressed in seconds which is 1 day)

- in this case the requested machine caches the www.someSite.com for one day, so it won't make any new DNS request for this domain name for one day

- once the record's TTL is expired the machine will make a new DNS request for this domain name again

A reccords - (Host IPv4) specifies domains or subdomains pointing to the set server(s)
- blank or @ = base/naked domain | * = all unspecified subdomain
AAAA records - (Host IPv6) specifies domains or subdomains pointing to the set server(s)
- blank or @ = base/naked domain | * = all unspecified subdomain
CName records (Canonical name) points to a domain or subdomain to another hostname
MX reccords - (Mail Exchanger) specifies which email server(s) to use when a mail is sent to the specified domain
- server priority can be set, the lower value has higher prioritiy (ex: 0 = highest priority)
NS records (authoritative Name Server) authoritative DNS server(s) where these DNS records are stored
TXT reccords (Text string) simple text records associated with the domain
PTR records ((domain name) Pointer) resolves an IP address to a fully-qualified domain name (does the oposite as A records) [NOT TESTED: 2020-02-25]
SRV records (Service) points service servers (like MX records but for other services) [NOT TESTED: 2020-02-25]
SOA records - (Start Of Authority) record appears at the beginning of the DNS zone
- contains the Authoritative Name Server, contact details for the domain Administrator and the domain serial number
CAA records - (Certification Authority Authorization) specifies which Certificate Authorities are allowed to issue certificates for a domain
- if nothing is specified all Authorities are allowed to issue certificats
NAPTR (Name Autority Pointer) used in internet telephony for mapping servers to users

Description and Demonstration :

- a Server is a computer on the network which provides services to other Computers on the network

- any computer can be a server but there are special Operating Systems (Windows Server Editions, Linux Ubuntu or Red Hat) and special hardware (Server processors, ECC RAM, ect...) made for servers, this special gear is more reliable and secure (obviously)

- there are many server types out there, File Server, Database Server (SQL), Email Server (Windows Exchange), ect...

Authentication Server (Domain Controller)

- the Domain Controller controls other computers on the network, gives them defined privileges to access computer or network functinalities

- once a computer or user is identified by the DC, the DC sends it an "access key", the user can access only to functionalities defined by this key

- Domain Controllers can have specified "trust" between them, this defines permissions between Domain Controllers

- Windows runs Active Directory as DC, we can have multiple mirror Active Directory DCs on the network for reliability and security purposes

- Linux runs Samba as DC, we can have only one DC and one backup DC on the network

DNS Server (Domain Name Server/Service)

- maps domain names to IP addresses

- DNS Network on Internet basics function in a nutshell

- Example: 1) the computer X on the network requests the DNS to identify the someDomain name

2) the DNS checks its reccords and maps the someDomain name to the 1.2.3.4 IP address then sends back this information

3) now the compuer X can make a request to the 1.2.3.4 IP address

Domain Registrar

- the Domain Registrar is were you can purchase domain names, this is usually a business (like GoDaddy.com)

- through Domain Registrars you can manage your (Authoritative) name server, DNS records, ect...

- changing internet DNS records takes time because the changes must be copied to all (13) Nameservers around the word, you can check this on the www.whatsmydns.net website

Domain Name Servers and Domain Name Resolution

- a DNS Resolver is the closest DNS server which caches IP addresses and does domain forward lookups, its main purpuse is to remove the load from the global DNS network

- domain names are resolved from right to left by a set of DNS hierarchy network

- DNS Server Hierarchy :

- Root Servers store TLD Server IP addresses (there are only 13 in the word [2020-02-24])

- TLD Servers (Top Level Domain) store all sites ending with the top level domain code (ex: the .com server stores all .com sites, the .net server stores all .net sites, ect... )

- Authoritative Server (Hierarchy) contain the actual DNS records (this is where Registrars register domains)

1) The user makes a DNS request to the DNS Resolver, if the DNS resolver finds the requested domain in its cache responds to the client (step 8)

2 / 3) the Root Server provides the apropriate TLD Servers IP address to the DNS Resolver

4 / 5) the TLD Server provides the apropriate Authoritative Servers IP address to the DNS Resolver

6 / 7) Authoritative Servers provide the acutal IP address of the requested site to the DNS Resolver

8) the DNS Resolver sends the domain's IP address to the client

Example : the www.someSite.com. = (. = resolved by a Root Server) (.com = resolved by a .com TLD Server) (someSite = resolved by a 1st level Authoritative server) (www = resolved by a 2nd level Autoritative Server this is where the domain is registered)

DHCP Server (Dynamic Host Configuration Protocol)

- dynamically assigns IP addresses to new devices connecting to the network

- it does this by issuing 'leases' to the connecting devices, once the lease is expired (or before expiration) the connected device reuquests a new lease in order to get a new (or keep the) connection to the network

- a DHCP server has limited range of IP addresses to assign (100 for a simple soho router), that's why leases are important (2 hours expiration for a simple soho router)

Web Server

- serves web pages (static or dynamic = web application)

- IIS (Internet Infromation Server) is a web server for Windows, it supports ASP.NET in order to develop web applications

- Apache is a very popular platform free web server, it supports multiple scripting languages in order to develop web applications (PHP, SQL, Python, ect...)

- NGINX (engineX) is a free web server which can be used as a proxy, reverse proxy and mail server, supports HTTP, HTTPS, IMAP, POP3 and SMTP protocols and has SSL support (as reverse proxy)

- Node.js can run as standalone web server, it supported on multiple platforms and has powerful tools and library support

Proxy and Reverse Proxy Servers

- both the proxy and the reverse proxy servers sit between the client and the server, they manage the traffic passing through and offer several features

- the proxy server benefits clients behind it, isolate clients | caches data | filters and blocks unwanted traffic

- the reverse proxy server benetits servers, isolate servers | balances the load between servers
