Description and Demonstration :

- the Operating System is the main program on the computer, its responsibility is to control the underlying hardware

- the Operating System runs on top of the hardware, and other aplications (programs or processes) are running on top of the OS

- applications (or their child processes) never interact directly with the hardware, they request the OS to do the requested task

- the Operating System has a crap-ton of task, it uses several 'managers' to achive these tasks (Memory Manager = manages the RAM | Process Manager = Managers running processes and CPU power | File Manager = reads nd writes secondary memory (HDD, SDD) | Device Manager = manages peripherals | Network Manager = manages the network | ect... )

Windows VS UNIX Systems

- Windows is less efficient secure and reliable than Linux, this is because far more people uses it, it has to deal with more weight

- Linux is more efficient secure and reliable than Windows because is lighter and deals with far less programs

- a Linux OS is composed of the Linux Kernel (the core of the OS) and of a Linux Distribution ('Distro')

- Linux is Case-Sensitive VS Windows is Case-Insensitive

File System

- Windows uses a more physical file system files in folders are stored on drives

- Linux uses a logical file system, disks and partitions can be "mounted" logically Linux File Sytem [link]

Users

- Windows is a single user system (only one user is allowed in the system at the time)

Windows User Profiles are Administrator | Regular | Child | Guest

- Linux is a multi user system (multiple users are allowed in one system (through the network))

- Linux User Profiles are Administrator | User | Service (uses a 9bit file ownership permission system [rwx][rwx][rwx] (owner | group | others) which is awesome!))

License

- Windows License types (Windows license can be checked by typing the slmgr -dli on the command line)

- OEM (Original Equipement Manufacturer) = Allowed only on the installed computer (not transferable or reusable)

- Retail = The Person own the software (can be transfered to other computers but cannot be used on more than one machine at the time)

- Volume = Can be used on multiple machines, it's for enterpries use (Usually it has time limit or other license tricks)

- CAL = (Client Access License) allows a device or user to access to a Windows Server

UNIX / POSIX / LINUX & LINUX kernel & Distributions / GNU & GNU Project / GNOME

UNIX

- UNIX is an Operating System released in 1973 (fist of its kind)

- a UNIX-like Operating System doesn't mean sharing the same UNIX source-code, it means following the same UNIX philosopy

POSIX (Portable Operating System)

- POSIX is an OS standard, which describes the minimum API requirements to follow in its family

- MacOS (from v10.5), Solaris, AIX and HP-UX are POSIX complient

- LINUX Distros, Android, FreeBSD, OpenBSD and VMWare follow most POSIX standards but they are POISIX certified

LINUX & LINUX kernel & LINUX Distributions

- LINUX and LINUX kernel are somewhat interchangeable, it's a UNIX-like Operating system, it fully controls the underlying harware and can be upgraded (but has no User Interface)

- a LINUX Distribution is a package which provides a User Interface on top the Linux kernel

GNU & GNU foundation & GNOME

- GNU is a free and open-source collection of software for LINUX (bash is a GNU software)

- GNOME is a free and open-source desktop environment

- the Ubuntu LINUX distribution uses both the above products

Environment (System) Variables

- environment variables are basically variables (key=value pairs) available on the system or session level, so every process can access these values

- environment variable keys are usually written with uppercases and separated by underscore (ex: NUMBER_OF_PROCESSORS)

Soft-Links / Hard-Links / Junctions

- a Soft-Link is a shortcut items what points to a file or directory (can point anywhere on the local system or on the network through SMB path)

Targets files or directories through absolute or relative paths

- a Hard-Link is a mirror copy of the original file (and each other), Hard-links are basically multiple files pointing to the same underlying data (available in the same volume)

the underlying file's content can be modified through any of the linked Hard-Links (because technically they are the same file, in linux hard-links have same inode numbers)

deleting a Hard-Link (or even the original file) does not modify the underlying data (data stays accessible through the remaining Hard-Links)

- a Junction is a shortcut item which points to a directory (junctions point to their targed directory through absolute paths) (can point anywhere on the local computer)

Available on Windows Only

Targets directories through absolute paths

Virtual Machine (VM)

- a Virtual Machine is an abstraction over a real OS and hardware, VMs has their own resources and are completely isolated from each other overt the same system

- the Hypervisor what is responsable for creating, running and monitoring VMs on the system

Windows Registry

- the Windows Registry is a low-level database which is used to store Operating System configurations, the Registry can be accessed through the Registry Editor

- applications also use this Registry to store their configurations but this is not required, (some applications use XML files or executables for this purpose)

- the SAM, SECURITY, SOFTWARE, SYSTEM, DEFAULT and other registry files are stored in %SystemRoot%\System32\Config folder

- Linux and macOS do not have Registry

CLI (Command Line Interface) VS GUI (Graphical User Interface)

- in programming the CLI prefered over GUI because it is more reliable and less vulnerable

Inter Process Communication (IPC)

- there are 2 ways processes can communicate with each other within a system:

- Shared Memory -> use of an allocated shared memory on the system to read/write to/from it

- File Socket -> use pipes of domain sockets (on Linux) named pipes (on Windows)

- we also can use this technique to create communication between services within the same system (sockets allow simple 2 way communication without the http overhead (no hostname, port))
