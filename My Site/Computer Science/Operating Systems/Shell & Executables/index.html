Description and Demonstration :

Shell

- the Shell is a command processor (PowerShell, Bash, ect..), typed commands are processed by the shell then passed to the underlying OS to 'ask' the specified task to be performed

- the Terminal is an access tool (window) to the shell (TTY = terminal), the terminal is most traditionally used on Unix-like systems because multiple user can be in the same system through 'terminals'

- the Shell Script basically a shell executable (on Windows is a *.ps file) (on Linux is a file starting with #!)

- the Command is basically the name of the shell script (programs are executed from left-to-right and the data can be piped between them with the | (pipe) character)

Executables

- an executable is file which contains a program, instructons to the OS (hence the name 'executable' = can be executed)

- when an executable is executed (ex: by a shell) requires the OS to perform the specified task

- on Windows executables are *.exe, *.bat, *.com files

- on Linux pretty much any file can be an executable, however the user needs executable privileges to 'run' the file

How Commands are interpreted by the Shell (TESTED with PowerShell)

- a command on the terminal is basically an executable name

- the shell looks for executables in directories specified by the PATH environment variable

- once ther's a match the command (executable or shell script) is executed

- processes can have their own environment variables set (on top of environment variables set by the system) 

    $ > gulp                                                                           // PowerShell check if the 'gulp.ps1' (file) can be found through the paths specified in the PATH environment variable    
    // gulp command executed                                                           // the 'gulp.ps1' file is found in 'C:\Users\Faszagyerek\AppData\Roaming\npm\gulp.ps1' and executed! (if the command is not found = ERROR message)
