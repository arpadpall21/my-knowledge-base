Remember This :

- Unicode and Encoding are two different concepts!

- the Unicode is an International encoding Standard, basically it is a big list which provides a unique (hexadecimal) number for each character

- the Encoding is the way how electronics encode these numbers (8bits / 16bits / ...)

Description & Demonstration :

- electronics can store data only in bits (binary code) basically in 0 and 1

- 8 bits = 1 byte

- in 1 byte (8 bits) we can store 256 different characters

CHARACTER SET
Char Set Compatibility Description
ASCII
(American Standard Code for Information Interchange) 		- uses 1 byte (7 bits = characters, 1 bit = transmission)
- encodes 128 characters - first standardized encode for web
ANSI / Windows-1252 compatible with ASCII - uses 1 byte (8 bits = 1 character)
- encodes 256 characters
- was the default character set for windows up to windows 95
- since ANSI was compatible with windows it is supported in all browsers
Latin1 / ISO-8859-1 - compatible with ASCII
- compatible with ANSI except range (128-159) (32 less characters) 		- uses 1 byte (all 8 bits are used)
- encodes 224 characters (range is 0-255 but 128-159 characters are empty)
- default encoding for HTML 4.0
base64 NO - uses 6 bits per character (ex: 4 characters are encoded in 3 bytes)
- encodes 64 ASCII characters
- used to send string encoded data through the network (instead of raw binary data)
UTF-8
(Unicode Transformation Format)
(8 stands for 8 bit blocks)		 compatible with ASCII - uses 1-4 bytes (when needed) per character
- encodes any character on the unicode table (can encode 4294967296 characters)
- default encoding for HTML5
- supported by all major languages and Operating Systems
UTF-16
(Unicode Transformation Format)
(16 stands for 16 bit blocks)		 NO - uses 2-4 bytes (when needed) per character (uses at least 2 bytes for encode 1 character)
- encodes any character on the unicode table (can encode 4294967296 characters)
