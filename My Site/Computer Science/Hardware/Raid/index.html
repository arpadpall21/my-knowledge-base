RAID level Description
RAID 0 - Spreads the data on 2 or more disks
- Increases Read/Write speed
- Decrease data redundancy if one (any) drive fails in the array all data is lost
RAID 1 - Duplicates the data on 2 or more disks
- Offers the best RAID Data Redundancy, can survive one disk failure
- Increases Read speed
- No Read/Write speed increase
- Only up to 50% of the total disk capacity is usable (2+ disks are just for redundancy)
RAID 10 - Combines RAID 1 and RAID 0 uses 4 or more disks
- Offers the best RAID Data Redundancy
- Increases Read/Write speed
- Only 50% of the total disk capacity is usable
RAID 5 - Parity is distributed on the drives uses 3 or more disks
- theoretically we can use unlimited disks but max 7 is recommended
- Can survive 1 disk failure
- Increases Read speed (not as efficien as RAID 0) (Write speed is not Increased)
- We lose only 1 disk space of the Array
RAID 6 - Double Parity is distributed on the drives uses 4 or more disks
- max 32 disks
- Can survive 2 disk failure
- Increases Read speed (not as efficien as RAID 0)
- Decreases Write speed (Because 2 Parity is written)
- We lose only 2 disk space of the Array

Description and Demonstration :

- RAID (Redundant Array of Independent Disks) Storage Technology using multiple disks, it is used for data redundancy, performance improvement, or both (RAID is not data backup)

- in order to use RAID we need a RAID controller, this can be a built in controller in the motherboard or a separate PCIe card

- RAID protects against physical disk failures, RAID does not protect against data corruption (such as viruses), data erase (not a backup) and the RAID controller can be a weakpoint (a solid RAID controller PCIe card is the best choice)
