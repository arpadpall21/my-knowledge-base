Git client CLI
Command Description
Initializing Git Repositories
git init dirPath - initializes a git repo (creates a .git dir)
|
git config [options] user.name userName - gets or sets the user name for the local Git repo
[options]
- (--global) for global user (used when no local user is set)
git config [options] user.email userEmail - gets or sets the user email for the local Git repo
[options]
- (--global) for global user (used when no local user is set)
|
git status
- checks the status for the Modified Area (files modified after the last commit) and the Stagin Area
- also returns the current branch we are on, if the HEAD is detached etc...
|
git log [options] - list the version history (commits) of the current branch
[options]
- (--oneline) one line per commit
- (--follow filePath) tracks the version history of the specified file
- (--oneline --all --graph) prints the git treey graphically
- (--stat) prints the change status for each commit
|
git reflog - displays the git command history log
|
Staging Area and Commit Management
git add [options] path - add files from the Modified area (or untracked files) to the Staging area (and updates the git history)
[options]
- (--all) all files
- (-f) add path focefuly (ex: ignored files can be added like this)
- (--) avoids conflicts when file names contain - (hiphens) (ex: git add -- * will add all files to the staging area without problem)
- (-p) display file content differences between the last commit and the Modified Area (uses the diff command) (does not add untracked files to the Stagin Area)
|
git rm [options] path - delete path from the file system and updates deleted status in the Staging Area
- [options]
- (-r) recursive delete (all files in dir)
- (--cached) untracks the specified item(s) and puts as deleted status in the Stagin Area (Git won't track this file any more)
git mv srcPath trgPath - moves/renames srcPath to trgPath (directory or file)
- must be used when renaming (otherwise git tracks srcPath as deleted and trgPath as new)
|
git commit [options] - creates a new commit on the top of the current branch (changes in the Stagin Area are commited) (opens the default text editor to input the commit message)
[options]
- (-a) also commits files from the Modified Area (Untracked files are not commited)
- (-m message) commit message (in this case the default text editor is not opened)
- (--amend) modifies the current commit (instead of creating a new)
- (--on-edit) does not modify the most recent commit's message
|
git tag [options] tagName commitId - adds a tag to the most recent commit (or the specified commit)
[options] - (-d) deletes the specified tag
|
git checkout master|branchName|tagName|commitId - navigates to the specified branch or commit point (files at that commit point are on the file system now, on the Git Tree the HEAD indicates where you are)
git checkout -b branchName - creates a new branch (from the current commit) and navigates to it
|
git revert tagName|commitId - reverts (undoes) the current (specified) commit by creating a new 'revert commit' (careful about conflicts)
|
git reflog - prints the git history log
git reset [options] tagName|commitId - removes all commits on the branch aftert the specified tagName|commitId (commits are removed and modifications placed back in the Modified Area)
[options]
- (--hard) also destroys file modifications after the specified commit
|
git reset --hard remoteVar/branch - pulls the branch from the remote repository and overwrites the current branch
|
git reset [--hard] path - empties the Staging Area by putting files back to their previoues place (or only the specified file)
- (--hard) also destroys file modifications
|
git cherry-pick commitId - pircks the specified commitId and put at the top of the current branch
|
Stash
git stash [options] - stashes all the current changes
[options]
- (-m name) adds a name to the stash entry
- (push path) stashes only the specifed file/directory
|
git stash list - prints the stash list
|
git stash pop stashId - copies the most recent (or the specified) stashed changes back to the Modified Area deletes the stash
|
git stash apply stashId - copies the most recent (or the specified) stashed changes back to the Modified Area
|
git stash drop stashId - deletes the most recent (or the specified) stash entry
|
Branches and Merging Branches
git branch [options] - lists all local branches
[options]
- (-a) remote branches are also listed
- (-D branchName ... ) deletes the specified local branch(es)
git branch branchName commitId|tag - creates a new branch from the current commit (or from the specified commit)
git branch -m oldBranchName newBranchName - renames the specified oldBranchName branch (if oldBranchName is omitted the current branch is renamed)
git push -d remoteVar branch - deletes the specified branch from remote repo (so careful with this)
|
git diff [options] - prints change diffs between the current Commit and the Modified Area and Stagng Area
[options]
- (--staged) only prints diffs between the previous Commit and the Staging Area
- (branch|commitId branch|commitId) prints the diffs between the specified branches or commits
|
git merge branchName [-m message] - merges the specified branch into the current branch by creating new 'merge' commit on the current branch
[options]
- (-m message) adds a message to the new commit
git merge --abort|--continue - (used on merge conflict) aborts or continues the merge process
|
git rebase [options] branch - rebases the current branch to branch
[options]
- (-i) (interactive mode) open the default text editor to finegrain the rebase process (commits to pick, exclude, etc...)
git rebase --abort|--continue - (used on rebase conflict) aborts or continues the rebase process
|
Remote Repo (on GitHub)
git clone repoUrl - clones the remote repository in the current local directory
|
git remote [options] - lists remote repository variables
[options]
- (-v) variables are printed with their URLs (the origin is the default and set when the repo is cloned)
git remote add remoteVar URL - sets a remote variable
git remote rename oldVar newVar - renames the specified remote variable
git remote remove remoteVar - deletes the specified remote variable
|
git push URL|remoteVar [options] - pushes local changes to the remote repo
[options]
- (-f) force push (override remote changes)
- (--delete branchName) deletes the remote branch (so careful with that)
|
git pull URL|remoteVar

- origin if not specified

- pulls changes from the remote repo and updates the local repo (this is basically fetch + checkout)
- (git reset --hard remoteVar/branch to 'force pull' (override the local branch with the remote))
|
git fetch remoteVar - fetches changes from the remote repo without making any changes locally

GitHub

- GitHub is a hosting service for Git repositories and Gists, on GitHub we can manage our repositories just like we do locally with Git (we can get 1GB free space apparently [2020-04-19])

- a GitHub repo is called 'remote repo', team members working on the same project connect to and synchronize this repository

- a Gits is a code snippet on GitHub, GitHub's Gist service is a lightweight hosting service to distribute and version control code snippets

Git

- Git is a distributed version control system, it tracks and stores repository history during the developement stages

- a Git Commit is basically a project status, when you 'chechout' a commit Git shows you the files at that commit stage on the file system

- files can be in 1 of the following 3 states:

- Modified - changes since the last commit

- Staging Area - changes that will be commited

- Committed - saved changes in a commit

- other than the above 3 sates changes can be stashed (it's a separated area for putting changes aside)

Stagin Area and Commit management add / rm / commit / tag / checkout / revert / reset

DEMO 
// add ------------------------------------------------------------------------------------
    $ git add *                                                                            // add all files (including nested onces) from the Modified Area to the Stagin Area      // -! also adds untracket files    
    $ git add -p *                                                                         // add all files (including nested onces) from the Modified Area to the Stagin Area      // -! does not add untracked files 
        diff --git a/index.html b/index.html                                                 // the above command compares the Previous Commit to the Modified Area by using the diff command automatically  
            // file comparison here ...
    
// rm -------------------------------------------------------------------------------------
    $ git rm -r SomeFeature                                                                // -! recursively removes the directory and all its itmes from the file system and puts them as deleted in the Stagin Area (also works on already deleted files)  
    $ git rm --cached someFile                                                             // untracks the 'someFile' file (Git won't track it any more) and marks it as deleted in the Stagin Area    
    
// commit ---------------------------------------------------------------------------------
    $ git commit -m "commit message"                                                       // commits files form the Stagin Area (creates a new commit with a commit message) 
    $ git commit -a -m "commit message"                                                    // commits files from the Stagin Area and the Modified Area too! (creates a new commit with a commit message)   
    $ git commit --amend -m 'new commit message'                                           // modifies the most recent commit's message   
    
// tag ------------------------------------------------------------------------------------
    $ git tag myTagName 6e6d51c                                                            // tags the specified commit (adds a tagname to it)  
    $ git tag -d myTagName                                                                 // removes the specified tag from the commit   
    
// checkout -------------------------------------------------------------------------------
    $ git checkout branch1                                                                 // navigates to the branch specified as 'branch1' (we can see 'branch1' files on the file system now)    
    $ git checkout 0088d40                                                                 // navigates to the specified commit (this commit's files are on the file system now) 
                                                                                             // -! if we are not on the top of the branch here we can only stash or put in a new brach modifications from here (we cannot coomit)   
    
    $ git checkout -b branch2                                                              // creates a new branch 'branch2' from the current commit and navigates to it    
    
// revert ---------------------------------------------------------------------------------
    $ git revert 0088d40                                                                   // adds a new commit which reverts (disables) this commit 
                                                                                           // if we want to reenable the reverted commit we just revert the revert commit 
// reset ----------------------------------------------------------------------------------
    git reset 11e1676                                                                      // removes all commits after this one on the branch     // -! file modifications after this commit are kept in the Modified Area   
    git reset --hard 11e1676                                                               // -! destorys all commits and file modifications after this one on the branch   
    
    git reset HEAD                                                                         // empties the Stagin Area by placing files back to their previous place

Stash stash save / stash list / stash pop / stash apply / stash drop

- the Stash is an area where you can put modifications aside, so this clears the Modified and Stagin areas (for someting else)

- we can have multiple stash entries and recall them when we need them, the Stash cannot be pushed to the remote repository

DEMO 
// stash save -----------------------------------------------------------------------------
    git stash save                                                                         // moves all files to the Stash from the Modified and the Stagin areas   
    git stash save -u                                                                      // does the same as above but also moves untracket files    
    
    git stash -p                                                                           // -! copies all files to the Stash from the Modified and the Stagin areas (uses the diff program to compare brevious commit changes to the current ones)    
    
// stash list -----------------------------------------------------------------------------
    git list                                                                               // returns the stash list   
    
// stash pop  / stash apply ---------------------------------------------------------------
    git stash pop                                                                          // moves the most recent stash files to the Modified Area    
    git stash apply stash@{1}                                                              // copies the 2nd stash entry files to the Modified Area    
    
// stash drop -----------------------------------------------------------------------------
    git stash drop stash@{3}                                                               // deletes the 4th stash entry

Branches and Merging Branches branch / diff / merge / rebase

- commit hashes are basically computed from the content of the commit, every commit contains the hash which is ahead of it, this is how they form the commit chain (or tree)

- rebasing is basically going back to a specific commit then comparing the commits ahead, so git recomputes the hashes and check if there's any change compared to the content what it has now

DEMO 
// branch ---------------------------------------------------------------------------------
    git branch -a                                                                          // lists all local and remote branches (remote branches are listed red)   
    git branch feature1 6d667f8                                                            // creates the 'feature1' branch from the 6d667f8 commit  
    git branch -D feature1                                                                 // deletes the 'feature1' branch   
    
// diff -----------------------------------------------------------------------------------
    git diff --staged                                                                      // compares and prints file differences between the last commit and the Modified and Staging areas    
    
// merge ----------------------------------------------------------------------------------
    git merge 'feature1' -m 'feature1 merged'                                              // merges the 'feature1' branch into the current branch and creates a new merge commit (if there's no prompt the merge is successful)   
                                                                                           // -! if there's a merge conflict it has to be resolved and commited or aborted   
    git merge --abort                                                                      // aborts a pending merge conflict    
    
// rebase ---------------------------------------------------------------------------------
    git rebase master                                                                      // moves the current branch on top of the master branch 
                                                                                           // -! if there's a rebase conflict it has to be resolved and continued or aborted   
    git rebase --continue                                                                  // continues a pending rebase conflict  
    git rebase --abort                                                                     // aborts a pending rebase conflict    
    
// top the current branch with another one ------------------------------------------------// !! in this case branches are not separated branches, the 'master' is simply behind the the 'feature1' on the same branch    
    git merge 'feature1'                                                                   // moves the current 'master' branch on the same level as 'feature1' 
    git rebase 'feature1'                                                                  // moves the current 'master' branch on the same level as 'feature1'

Remote Repository (no GitHub) clone / remote / push / pull / fetch

DEMO 
// clone ----------------------------------------------------------------------------------
    git clone git@github.com:gulpjs/gulp.git                                               // clones the Gulp repo into the current local direcoty

    git remote -v                                                                          // list remote repository variables // -! set for the local repository

    git remote rename backup pallBackup                                                    // renames the remote variable

    git push origin --all                                                                  // uploads all local branches to the 'origin' remote repository    
    git push origin --delete feature2                                                      // deletes the 'feature2' branch in the 'origin' remote repository (does not delete the branch locally)    
    
// fetch ----------------------------------------------------------------------------------
    git fetch origin                                                                       // fetches the commits from the remote (origin) repository  
    git diff master origin/master                                                          // displays the difference between the local master and the remote master branches 
    
    git merge                                                                              // merges the fetched commints into the local branch  
    
// pull -----------------------------------------------------------------------------------
    git pull origin                                                                        // downloads and updates all branches from the 'origin' remote repository   
                                                                                           // pull = fetch + merge commands

- Git and Docker do fundamentally different thins, Git track only files (for version control) VS Docker deals with directory structures for container file system managament

- this is one of the reasons why globs are used differently in these files! But there are other tricky things around globs as well

DEMO 
    // -! in .gitignore golb files and directories are recursively checked, if there's a match the WHOLE PATH IS IGNORED (because git tracks only files)
    // -! in .dockerignore there's no recursive check and only the matching part of the path is ignored (because docker copies directory structures)
    
    // glob (in ignore file)        // original path                // file ignored by Git          // watched part by Docker           // ignored part by Docker 
// --------------------------------------------------------------------------------------------------------------------------------------
