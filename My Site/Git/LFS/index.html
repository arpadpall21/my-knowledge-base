Install
- Git LFS is a software that is installed on OS level (git-fls.github.com)
|
Commands
git lfs help cmd - print LFS commands and help in general
- (cmd) help for the specified command
git lfs install - installs (adds and clean up filter configs) Git LFS in your current repo
|
git lfs track 'glob' - (without 'glob') lists tracked paths (LFS paths are stored in .gitattributes)
- (with 'glob') add a new LFS file track path (same glob pattern as .gitignore)
git lfs untrack 'glob' - removes the specified path from the LFS track list
|
git lfs status - prints the git status (just like git status) but also displays which files are tracked by LFS or Git (in paranthesis)
|
git lfs fetch [--recent] [--all] - fetches LFS files for the current commit only from the remote repo without making any changes to the local repo
- (--recent) 'recent' files (younger than the last 7 days (by Default))
- (--all) all the LFS file history
git lfs pull - pulls LFS files for the current commit only from the remote repo (this is basically fetch + checkout)
git lfs clone repoUrl - first calls git clone then clones LFS files for the current commit (hight performance clone for large LFS enabled repos)
|
git lfs prune [--dry-run] - deletes 'old' LFS files

- files older tahn 7 days (by Default) (not 'recent' files)

- files not belonging to this commit

from the local LFS cache (from .git/lfs)

Description and Demonstration :

- Git LFS is a version control extension that handles large (binary) files within a Git repository (tracked in .git/lfs)

- Git LFS is version control agnostic (compatible with various version control systems that support Git LFS, including GitHub, Bitbucket, GitLab, Azure, and others)

- only downloads files belonging to the current commit, it doesn't do any diff display (not needed for binary files)

- paths of tracked files are listed .gitattributes file located in the root of the project (add remove paths with git lfs track & git lfs untrack)

- once Git LFS is installed and configured, seamlessly work with git (does its thing in the background)

Example : 
    $ git init
        Initialized empty Git repository in /home/apall/Documents/TestRepo/.git/
        
    $ git lfs install                                                                      // after initializing git we set up git LFS 
        Updated git hooks.
        Git LFS initialized.
                                                                                           // -! Git LFS files must be tracked before .git, so the intended large files are tracked by LFS not by .git    
    $ git lfs track 'Pictures/**'                                                          // files in the Pictures/** and Videos/** directories are handled by git LFS from no on   
    $ git lfs track 'Videos/**'    
    
    $ git lfs track                                                                        // list LFS tracked paths    
        Listing tracked patterns
            Pictures/** (.gitattributes)                                                   // paths are listed in .gitattributes file 
            Videos/** (.gitattributes)
        Listing excluded patterns
    
    $ git lfs status                                                                       // prints all tracked file's status but we can also see here which files are hadled by LFS 
        
        Git LFS objects to be committed:
        
          Project/test.txt (Git: e3b0c44)                                                  // file handled by git   
          Pictures/v1.jpg (LFS: d404fad)                                                   // file handled by git LFS (mind the paranthesis)  
          Videos/v1.webm (LFS: 73ca609)
        
        Git LFS objects not staged for commit:
