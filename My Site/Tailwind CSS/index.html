|

Description and Demonstration :

- Tailwind CSS is a utility-first CSS framework, this approach encourages us to use (Tailwind) class utilities instead css components (style groups) (like bootstrap)

- Inline styling means no css files to maintain, no style namespace mix risk

- Not using css components means we won't end up with a bunch of css component to remember and maintain

- Tailwind docs agrees that writing repetitive inline styles make our HTML ugly, but maintaining css files and css components is worse!

Support:

- Well supported by frontend frameworks (natively supported by Next.js) [setup link]

- for browser support the docs says "tested on latest browsers", also transpiles vendor prefixes (or we could use autoprefixer for this)

- Tailwind removes default styles from cretain html elements, this is for consistency reasons (this feature is called preflight)

Selectors & Styling Based on Other Element's State 

    <htmlTag class="tailwStyle ...">           // styles applied on the element itself
    <htmlTag class="*:tailwStyle ...">         // styles applied on direct child elements
    
// styling based on parent element state (hover, active, etc...) ----
    <htmlTag class="group/groupName ...">
      <htmlTag class="group-status/groupName:tailwStyle">    // tailwStyle applied on element if group/groupName parent element has status
    </htmlTag>
    
    
// styling based on previous sibling element state (hover, active, etc...) ----
    <htmlTag class="peer/peerName ...">
    <htmlTag class="peer-status/peerName:tailwStyle">    // tailwStyle applied on element if peer/peerName previous sibling element has status
    
    
// styling based on child element state (hover, active, etc...) / or presence ----
    <htmlTag class="has-[pseudoSel|htmlTag]:tailwStyle ...">      // pseudoSel: tailwStyle applied if any child element has status (ex: :active)
      <...>                                                        // htmlTag: tailwStyle applied if this element has any htmlTag child element
    </htmlTag>

CSS Components / Element Base styles / Custom Tailwind styles (utilities)

- Tailwind also supports css components, but it's discouraged in Tailwind docs (only for highliy reused components) 

// input.css (file) --------------------------
    @tailwind base;
    @tailwind components;
    @tailwind utilities;
    
    
// css component ---
    @layer components {
      .className {
        cssStyes
      }
    }
    
    .className {              // this is also valid but not recommeded because of css specificity (use @layer components instead)
      cssStyes
    }
    
    
// element base style ----
    @layer base {
      htmlTag {              // apply base style on all htmlTag elements
        @apply tailwStyle;
      }
    }
    
    
// custom tailwind style (utility) ----
    @lary utilities {
      .tailwUtil {           // creates a custrom tailwUtil class utility
        cssStyes
      }
    }

Configuration

- the tailwind.config.js configures the tailwind transpiler, but most importantly we can use it to change or extend existing taiwind utility classes 

// tailwind.config.js (file) -----
    /** @type {import('tailwindcss').Config} */
    module.exports = {
      content: [globPath],               // source files to transpile
      theme: {
        utilityClass   // default class utilities (ex: colors default color palette / spacing default spacing)
                                    // -! overriding one property like: color overrides all default colors 
        extend: {                   // extends existing class utilities (ex)
          ...          // ex: adding color property with new colors extends the existing color palette
        },
      },
      other configs...
    }
    
    
// theme() function -------------------
    theme(configThemeKey)            // reference any config theme key (ex: theme(colors.red.400))
                                                      // can be used in: class interpolation (ex: <div class-"bg-[theme.colors.blue.400]">)
                                                      // can be used in: input.css file

    /** @type {import('tailwindcss').Config} */
    module.exports = {
      content: ['./src/**/*.{html,js}'],
      theme: {
        colors: {                                   // this overrides all default color utility classes

          secondary: '#5d634c',
          someColor: {
            light: '#ccbc91',
            DEFAULT: '#ad9961',
            dark: '#635736',
          }
        },
        extend: {
        },
      },
      plugins: [],
    }

Extending class utilities 

// HTML --------------
    <div class="h-[43px] bg-cyan-100"></div>            // existing colors are not overriden
    <div class="h-[43px] bg-myColor"></div>
    <div class="h-[43px] bg-someColor"></div>
    <div class="h-[43px] bg-someColor-light"></div>
    <div class="h-[43px] bg-someColor-dark"></div>
    
    
// tailwind.config.js --------------
    /** @type {import('tailwindcss').Config} */
    module.exports = {
      content: ['./src/**/*.{html,js}'],
      theme: {
        extend: {
          colors: {                     // extends existing color palette
            myColor: '#141E8C',         // custom color
            someColor: {                // custom color with different shades
              light: '#ccbc91',
              DEFAULT: '#ad9961',       // default shade
              dark: '#635736',
            }
          }
        },
      },
      plugins: [],
    }

The theme() function 

// HTML --------------
    <div class="h-10 bg-[theme(colors.red.100)]"></div>     // references default theme value
    <div class="h-10 bg-[theme(colors.myColor)]"></div>     // references a custom value
    
// input.css -------------
    @tailwind base;
    @tailwind components;
    @tailwind utilities;
    
    @layer component {
      .some-component {
        color: theme(colors.red.400);                   // references theme values 
        background-color: theme(colors.amber.400);
        height: theme(spacing.10);
      }
    }
    
    
// tailwind.config.js --------------
    /** @type {import('tailwindcss').Config} */
    module.exports = {
      content: ['./src/**/*.{html,js}'],
      theme: {
        extend: {
          colors: {
            myColor: '#734845'
          }
        },
      },
      plugins: [],
    }
