Description and Demonstration :

- Tailwind CSS responsive modifies apply css styles at specified screen widths, out of the box 5 responsive modifiers are supported 

    <meta name="viewport" content="width=device-width, initial-scale=1.0">      // place this in document head
    
// -----------
    <htmlTag class="respMod:tailwStyle..." >     // tailwStyle styles applied if screen width fullfils respMod
    
    <htmlTag class="min|max-[cssLength]:tailwStyle..." >     // specifies custom screen width
  
// mobile first design approach --------------------------
    <div class="bg-white"></div>        // on code level we basically defaults to the smallest screen size
    
    <div class="sm:bg-white"></div>     // same as above (if we use mobile first design approach)
    
    
// appliy tailwind styles between a specified screen width range ----
    <div class="md:max-lg:bg-gray-200"></div>
Example :

Mobile First Approach 

    <div class="w-full h-[40px] bg-green-500 md:bg-yellow-500 lg:bg-red-500"></div>       // default background color is for sm media modifier

Setting Custom Breakpoints 

    <div class="h-[40px] min-[1000px]:bg-gray-200"></div>

Apply style between a specified range 

    <div class="w-full h-[40px] bg-green-500 md:max-lg:bg-yellow-500"></div>            // yellow background color applied between md and lg ranges

Customization

- by modifying the tailwind.config.js we can modify breakpoints but also customizing breakpoint modifier names 

// tailwind.config.js (file) ----------------------------------
    
    /** @type {import('tailwindcss').Config} */
    module.exports = {
      theme: {
        screens: {                                                  // overides all breakpoints (default are removed)
          'respMod': cssLength,   // for respMod we can use custom names (ex: mobile)
          ...
        },
        extend: {                                                   // extend breakpoints
          screens: {
            'respMod': cssLength,     // override existing or add new breakpoint
            ...
          }
        }
      }
    }
