Description and Demonstration :

- The REpresentational State Transfer is an Api describing the communication between client-server (however there's no clear specification for REST Api)

- The REST Api has the following constraints:

- must be a client-server architecture

- stateless (the server doesn't keep data about the clients)

- cachable resource (the response is marked as cachable or not cachable)

- layered (services are scalable between the client-server)

- The URL is the address of the resource (indicates where a specific resource is located)

- a HTTP verb indicates the action that the client wants to perform on the resource

- The Client requests te server with the following HTTP verbs (basic verbs):

- GET (read resource, has no request body)

- POST (send resource)

- PUT (update resource)

- DELETE (delete resource, has no request body)

- The Server responds with the following status codes

- 1** (Information, like Protocol upgrade)

- 2** (Success)

- 3** (Redirection, redirect to a different URI)

- 4** (Client Error, server received an unexpected request like malformed request body or unknown client id)

- 5** (Server Error, something's wrong on server side)

URL or URI

- Uniform Resource Locator (locates the resource)

- Uniform Resource Identifier (locates and identifies the resource)
