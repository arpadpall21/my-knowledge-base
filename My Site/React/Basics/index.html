Description and Demonstration :

- React is a JavaScript library to build single page application, it's an application development methology

- it provides a full development environment then we can 'build' the application and serve it by a web server (building is basically translating the react code into a browser readabe code)

Using React directly in the HTML document (through CDN) 

    <html>
        <head>
            <script src="https://unpkg.com/react@16/umd/react.production.min.js"></script>              // include these 3 CDN links to the head of the HTML document and you're good to go   
            <script src="https://unpkg.com/react-dom@16/umd/react-dom.production.min.js"></script>
            <script src="https://unpkg.com/babel-standalone@6.15.0/babel.min.js"></script>
        </head>
        <body>
            <div id="testElement"></div>            // target HTML element wher to render the React Component    
        
            <script type="text/babel">
                function TestComponent(){           // React Code 
                    return <h1>Hello World!</h1>
                }
                
                ReactDOM.render(<TestComponent />, document.getElementById('testElement'));
            </script>
        </body>
    </html>

- serving the React App with express.js 

    var express = require('express');
    var path = require('path');
    var app = express();
    
    app.use(express.static(path.join(__dirname, 'build')));                            // serving static files (the page's .css and .js files)
    
    app.get('/', function(req, res){
        res.sendFile(path.join(__dirname, 'build', 'index.html'))                      // serving the main file 
    });
    
    app.listen(1000, '127.0.0.1');                                                     // served on 'localhost:1000'

React core mechanic explained

- the React application is composed by components, each component contains its own JSX code that is rendered on the DOM

- components have their own states, by setting the state React will rerender the component and (re)paints the DOM part for which the component is responsible (if necessary)

- components are structured in hierarchy (triggering a component will rerender all its child components)

- React does not manipulates the DOM directly, instead it maintains a Virtual DOM and only (re)renders HTML elements when needed, this makes React fast!
