Paths

canvasCtx2D.bezierCurveTo(x1Cp, y1Cp, x2Cp, y2Cp, xEnd, yEnd)
canvasCtx2D.path2DObj.bezierCurveTo( ↑ same as above ↑) 		creates a "cubic Bézier" curve between two points
canvasCtx2D.clip() clips the current shape's region (all further drawings are limited to this region only)
canvasCtx2D.closePath()
canvasCtx2D.path2DObj.closePath() 		closes the current path from the current point to the starting point (creates the last closing stroke of the path)
canvasCtx2D.ellipse(x, y, xRadius, yRadius, rotation(radian), startAng(radian), endAng(radian), antiClockWhise_boolean)
canvasCtx2D.path2DObj.ellipse( ↑ same as above ↑) 		creates an ellipse (has its own starting coordintes) (no IE Support)
canvasCtx2D.fill(path2DObj, "nonzero/evenodd") fills the current path (optionally the path from the path object), (optionally can fill every second sub-path nested shapes)
canvasCtx2D.isPointInPath(x, y) returns a boolean value indicating if the given coordinates are in the current path
canvasCtx2D.lineTo(x, y)
canvasCtx2D.path2DObj.lineTo( ↑ same as above ↑) 		creates a line from the last point to the specified point
canvasCtx2D.moveTo(x, y)
canvasCtx2D.path2DObj.moveTo( ↑ same as above ↑) 		specifies a point without creating a line
canvasCtx2D.quadraticCurveTo(xCp, yCp, xEnd, yEnd)
canvasCtx2D.path2DObj.quadraticCurveTo( ↑ same as above ↑) 		creates a "quadratic Bézier" curve between two points (same as "cubic Bézier" curve but it has only 1 control point)
canvasCtx2D.stroke(path2DObj) draws the current path (optionally the path from the path object)

canvasCtx2D.strokeText("text", x, y, max_width(nr)) draws a stroke (not filled) text on the canvas

canvasCtx2D.textBaseline = "alphabetic(default)/top/bottom/middle/ideographic/hanging" gets or sets the vertical (baseline) position of the text of the canvas context

Transformation

canvasCtx2D.translate(x, y) oves the canvas grid on the "x" and "y" axis relative to the 0, 0 (top left corner) coordinates

Remember This :

- the canvas object cannot be created by using constructor (like new Canvas()), you must use or create a canvas HTML tag

    - to save up performance
  1. ) draw images off canvas as cash images rather than scale them which uses considerable resources
  2. ) you can use multiple canvas elements as layers (like in games [background, UI, game layer]) to prevent rendering unnecessary regions
  3. ) turn off canvas transparencey by using canvasContext2D.getContext("2d", {alpha:false}) attribute

Description and Demonstration :

- the <canvas> tag has no drawing abillities on its own! but provides a container (by default 300 X 150 pixels) to draw graphics on the fly via scripting (usually JavaScript)

- the canvas API is a Drawing Interface (bit-map) once is drawn the only way to manipulate it is to redraw the canvas

- the canvas object cannot be created by using constructor (like new Canvas()), you must use or create a canvas HTML tag

- the getContext() method returns an object which provides methods and properties to draw the canvas 

SYNTAX :    <canvas id="canvasId"></canvas>

            <script>
                var canvas = document.getElementById("canvasId");          // get the canvas element as JS object
                var contextId = canvas.getContext("context", attribute)    // the contextId provides methods and properties to draw the canvas 
            </script>
            
    // ---------------------------------------------------------------------- 
            HTMLCanvasElement.getContext("context", attribute)
                - "context" : sets the drawing interface context ("2d" / ...) 
                - attribute : context attribute ({alpha:false} sets the 2d context fully opaque = default canvas background black (performance upgrade) (no EDGE / IE support))
            
            HTMLCanvasElement.toDataURL("type", quality(0 - 1))            // returns a "dataURL" of the current canvas (resolution 96 dpi)  dataURL [link]
                - "type" : image format (default "image/png")
                - quality(0 - 1) : image quality (default 0.92)
                
            canvasCtx2D.save()                                             // saves the current context state (context styles like lineWidht ect.. / clipping region / transformation matrix / dash list)  
            canvasCtx2D.restore()                                          // restores the previously saved context state

- the {alpha:false} 2d context attribute sets the canvas to fully opaque (background color black), this is intended to speed up performance 

    <canvas id="canvas1">
        <img src="backupImage.jpg" alt="">                                 // -> backup image specified in case canvas is not supported
    </canvas>
    <canvas id="canvas2">
<script>
    var myCanvas = document.getElementById('canvas1');
    ctx = canvas1.getContext("2d");                                        // creating a "2d" drawing context
    ctx.save();                                                            // save the current (default) canvas state
// create and style the desired line  
    ctx.lineWidth = 5;
    ctx.strokeStyle = "green";
    ctx.moveTo(0, 0);
    ctx.lineTo(300, 150);
// draw the actual line    
    ctx.stroke();
    
    ctx.restore();                                                         // context state restored (all further context style reset to default [to previously saved state])

// {alpha:false} attribute
    var ctx2 = document.getElementById('canvas2').getContext("2d", {alpha:false});      // canvas fully opaque (canvas background color black)
    ctx.strokeStyle = "green";                                             // line color set because of the black canvas background 
    ctx.moveTo(0, 0);
    ctx.lineTo(300, 150);
    ctx.stroke();
<script>

save() / restore() TEST

CODE : 
    <canvas id="saveRestore" style="border:solid 1px black"></canvas>
<script>
    var saveRestoreFn = function() {
        var ctx = document.getElementById("saveRestore").getContext('2d');
        ctx.save();                             // saves the current context state (default styles)
        
        ctx.lineWidth = 10;
        ctx.strokeStyle = "red";
        ctx.beginPath();
        ctx.moveTo(20, 10);
        ctx.lineTo(20, 140);
        ctx.stroke();
        
        ctx.save();                             // curent context state saved 
        
        ctx.lineWidth = 5;
        ctx.strokeStyle = "orange";
        ctx.beginPath();
        ctx.moveTo(40, 10);
        ctx.lineTo(40, 140);
        ctx.stroke();
        
        ctx.beginPath();                        // same styles used here too
        ctx.moveTo(60, 10);
        ctx.lineTo(60, 140);
        ctx.stroke();
        
        ctx.restore();                          // previously saved context state restored
        
        ctx.beginPath();                        
        ctx.moveTo(80, 10);
        ctx.lineTo(80, 140);
        ctx.stroke();
        
        ctx.restore();                          // previously saved context (default) state restored
        
        ctx.beginPath();                        
        ctx.moveTo(100, 10);
        ctx.lineTo(100, 140);
        ctx.stroke();
    }
    saveRestoreFn();
</script>

dataURL() method TEST

Tested with Edge because the toDataURL() method doesn't allow to get cross origin images (if there is no image used in canvas this problem does not persist!) and Chrome sees local sites as cross origin [2018-11-18]

CODE : 
    <canvas id="imageData_getIt" style="border:solid 1p black;"></canvas>
    <img id="dataSource" alt="dataUrl source Test">
<script>
    var getImageDataFn = function() {
        var ctx = document.getElementById("imageData_getIt").getContext("2d");
        var img = new Image();
        img.src = "Pictures/camaro.jpg";
        
        img.onload = function() {
            ctx.drawImage(img, 140, 80, 300, 150, 0, 0, 300, 150);
            var dataURLimage = document.getElementById("imageData_getIt").toDataURL();      // canvas returned as data URL 
            document.getElementById("dataSource").src = dataURLimage;                       // image source dataURL printed 
        }
    }
    getImageDataFn();
</script>
