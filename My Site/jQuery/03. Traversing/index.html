Method Description
jQ_obj.add(jQuery_selector, context) adds elements to the jQuery object
jQ_obj.addBack(jQuery_selector) adds back the previous element to the selected stack
jQ_obj.children(filter_jQuery_selector) returns the direct chidren of the selected element
jQ_obj.closest(filter_jQuery_selector) returns the selected element itself or the closest targeted ancestor
jQ_obj.contents() returns all direct (1st level) children including text and comment nodes
jQ_obj.each(fn(index, element)) runs a function for each element in the jQuery object but cannot return anything (the original jQuery object is not modified)
jQ_obj.end() ends the most recent filtering operation in the current chain (basically cancels the recent filtering operaton so you can reuse the previous selection state)
jQ_obj.eq(index) returns the element at the index position from the jQuery object
jQ_obj.filter(jQuery_selector)
jQ_obj.filter(fn(index, element)) 		returns the elements that match the set criteria from the jQuery object
jQ_obj.find(filter_jQuery_selector) returns all decendent children elements of the selected element
jQ_obj.first() returns the first element from the jQuery object
jQ_obj.has(jQuery_selector) returns the elements which have the specified elements inside them from the jQuery object
jQ_obj.is(jQuery_selector)
jQ_obj.is(fn(index, element)) 		checks if the specified element is part of the selected group and returns a boolean value
jQ_obj.last() returns the last element from the jQuery object
jQ_obj.map(fn(index, element)) runs a function for each element in the jQuery object and returns its values in a new jQuery object (the original jQuery object is not modified)
jQ_obj.next(filter_jQuery_selector) returns the next sibling element of the selected element
jQ_obj.nextAll(filter_jQuery_selector) returns all next sibling elements of the selected element
jQ_obj.nextUntil(stop(excluded)_jQuery_selector, filter_jQuery_selector) returns all next sibling elements between the selected element and the targeted element
jQ_obj.not(jQuery_selector)
jQ_obj.not(fn(index, element)) 		returns all elements which do not match the set criteria from the jQuery object
jQ_obj.parent(filter_jQuery_selector) returns the direct parent element of the selected element
jQ_obj.parents(filter_jQuery_selector) returns all the parent elements
jQ_obj.parentsUntil(stop(excluded)_jQuery_selector, filter_jQuery_selector) returns all parent elements between the element itself and the targeted element (excluded)
jQ_obj.prev(filter_jQuery_selector) method returns the previous sibling element of the selected element
jQ_obj.prevAll(filter_jQuery_selector) returns all previous sibling elements of the selected element
jQ_obj.prevUntil(stop(excluded)_jQuery_selector, filter_jQuery_selector) returns all previous sibling elements between the selected element and the targeted element
jQ_obj.siblings(filter_jQuery_selector) returns all sibling elements of the selected element
jQ_obj.slice(start_index(included), end_index(excluded)) reduces the selection to a specified range

Navigation (UP) select Ancestors parent() / closest() / parents() / parentsUntil()

- the parent() method returns the direct parent element of the selected element

- the closest() method returns the selected element itself or the closest targeted ancestor

- the parents() method returns all the parent elements

- the parentsUntil() method returns all parent elements between the element itself and the targeted element (excluded)

SYNTAX :    jQ_obj.parent(filter_jQuery_selector)  
            jQ_obj.closest(filter_jQuery_selector)
            jQ_obj.parents(filter_jQuery_selector)
            jQ_obj.parentsUntil(stop(excluded)_jQuery_selector, filter_jQuery_selector)
                - filter_jQuery_selector : narrows down the search (targets specific elements)
                - stop(excluded)_jQuery_selector : indicates where to stop the search (element excluded)

    $(document).ready(function(){
    // parent()    
        $("#demo").parent();                        // returns "li" the direct parent element  
        $("#demo").parent("span");                  // doesn't return anything because the "span" element is not direct parent!
    // closest()
        $("#demo").closest("p");                    // returns the element itself because this method starts the search with the element itselft
        $("#demo").closest("span");                 // returns "span" the closest match (if there would be other "span" elements only this element would be returned too!)
    // parents()
        $("#demo").parents();                       // retunrs all parents "li, span, div, body, html" (event the html is returned)

    // parentsUntil()
        $("#demo").parentsUntil();                  // returns all parents "li, span, div, body, html" (event the html is returned)
        $("#demo").parentsUntil("div");             // returns "li, span" all parent elements between the selected and "div" element (excluded)       
        $("#demo").parentsUntil("div", "[style~='width:220px;']");
                                                    // returns "span" element because this is the only match between the selected one and "div"
    });
</script>

Navigation (SIDEWAYS) select Siblings siblings() / next() / nextAll() / nextUntil() / prev() / prevAll() / prevUntil()

- the siblings() method returns all sibling elements of the selected element

- the next() method returns the next sibling element of the selected element

- the nextAll() method returns all next sibling elements of the selected element

- the nextUntil() method returns all next sibling elements between the selected element and the targeted element

- the prev() method returns the previous sibling element of the selected element

- the prevAll() method returns all previous sibling elements of the selected element

- the prevUntil() method returns all previous sibling elements between the selected element and the targeted element 

SYNTAX :    jQ_obj.siblings(filter_jQuery_selector) 

            jQ_obj.next(filter_jQuery_selector)
            jQ_obj.nextAll(filter_jQuery_selector)
            jQ_obj.nextUntil(stop(excluded)_jQuery_selector, filter_jQuery_selector)
            
            jQ_obj.prev(filter_jQuery_selector)
            jQ_obj.prevAll(filter_jQuery_selector)
            jQ_obj.prevUntil(stop(excluded)_jQuery_selector, filter_jQuery_selector)
                - filter_jQuery_selector : narrows down the search (targets specific elements)
                - stop(excluded)_jQuery_selector : indicates where to stop the search (element excluded)

    $(document).ready(function(){

        $("#demo").siblings();                          // returns 2"div" 2"li" and 2"span" all sibling elements all 6 elements

        $("#demo").prevAll();                           // returns 1"span" 1"li" 1"div" all previous sibling elements

        $("#demo").prevUntil("li");                     // returns 1"span" all previous elements until "li" (excluded)
        $("#demo").prevUntil("div", "span");            // returns 1"span" element the previous targeted element between the element itself and "div"
    });
</script>

Navigation (DOWN) select Children children() / find() / contents()

- the children() method returns the direct chidren of the selected element

- the find() method returns all children elements of the selected element

- the contents() method returns the direct (1st level) children including text and comment nodes 

SYNTAX :    jQ_obj.children(filter_jQuery_selector)
            jQ_obj.find(filter_jQuery_selector)
            jQ_obj.contents()
                - filter_jQuery_selector : narrows down the search (targets specific elements)
    
    <p id="demo">in myParagraph <span>this span is <b>BOLD</b></span> and <span>that span is <i>ITALIC</i></span></p>
<script>
    $(document).ready(function(){
    // children()    
        $("#demo").children();                          // returns the 2 "span" elements only because they are direct children
        $("#demo").children(":eq(1)");                  // returns the 2nd "span" element only 
    // find()
        $("#demo").find("*");                           // returns the 2 "span" the 1 "b" and the 1 "i" elements (all children elements) (argument required)
        $("#demo").find("span:first, i");               // returns the 1st "span" element and the "i" element targeted children 
    // contents()
        $("#demo).contents();                           // returns 4 nodes "text, span, text, span" (only direct nodes)
        $("#demo).contents()[1];                        // returns the 1st "span" element as a DOM element!
    });
</script>

Filtering

the first() / last() / eq() / filter() / has() / not() methods

- jQuery filtering methods select specific elements from a group of elements

- the first() method returns the first element from the jQuery object

- the last() method returns the last element from the jQuery object

- the eq() method returns the element at the index position from the jQuery object

- the filter() method returns the elements that match the set criteria from the jQuery object

- the has() method returns the elements which have the specified elements inside them from the jQuery object

- the not() method returns all elements which do not match the set criteria from the jQuery object 

SYNTAX :    jQ_obj.first()
            jQ_obj.last()
            jQ_obj.eq(index)
            jQ_obj.has(jQuery_selector)
            
            jQ_obj.filter(jQuery_selector)
            jQ_obj.filter(fn(index, element))
            
            jQ_obj.not(jQuery_selector)  
            jQ_obj.not(fn(index, element))
                - jQuery_selector : element targeted by the jQuery selector 
    
    <ol id="demo">
        <li>item_1</li>
        <li>item<b>_2</b></li>
        <li class="target">item_3</li>
        <li class="target">item_4</li>
        <li>item_5</li>
    </ol>
<script>
    $(document).ready(function(){
    // first()
        $("#demo *").first();                           // returns "item_1" element
    // last()
        $("#demo *").last();                            // returns "item_5" element
    // eq()
        $("#demo *").eq(1);                             // returns "item_2" element                 
        $("#demo *").eq(3);                             // returns "item_4" element 
    // has()
        $("#demo *").has("*");                          // returns "item_2" element, (returns the the element which has any children elements)
    // filter()
        $("#demo *").filter(".target");                 // returns "item_3, item_4" (the have the "target" class)
    // not()
        $("#demo *").not(".target");                    // returns "item_1, item_2, <b>, item_5" (the <b> nested element is also returned because it is a separate element even if it is nested)
    });
</script>

    the add() / addBack() / slice()

    - the add() method adds elements to the jQuery object

    - the addBack() method adds back the previous element to the selected stack

    - the slice() method reduces the selection to a specified range 

    SYNTAX :    jQ_obj.add(jQuery_selector, context)
            jQ_obj.addBack(jQuery_selector)
            jQ_obj.slice(start_index(included), end_index(excluded))
                - start_index(included) : element to start the selection with
                - end_index(excluded) : element to end the selection with (if omitted all elements are selected after the first argument)   
    
        <p id="demo"> myParagraph </p>

        $("#demo").add(addIt).css("color", "white");                    // adding additional item to the selection with JS object
        $("#demo").add($addIt).css("color", "white");                   // adding additional item to the selection with jQuery object
    // addBack
        $("#demo").next().addBack().css("color", "white");              // the element itself is selected too

</script>

    the end() / is() methods

    - the end() method ends the most recent filtering operation in the current chain (basically cancels the recent filtering operaton so you can reuse the previous selection state)

    - the is() method checks if the specified element is part of the selected group and returns a boolean value 

    SYNTAX :    jQ_obj.end()

            jQ_obj.is(jQuery_selector)
            jQ_obj.is(fn(index, element))
    
        <div id="demo_parent">
        <p id="demo"> myParagraph <span>nestedSpan</span> </p>
        <p> myParagraph <span>nestedSpan</span> </p>
    </div>
<script>
    $(document).ready(function(){
    // end()
        $("#demo")
            .find("span")                       
            .css("textDecoration", "underline")
            .end()                                      // makes possible to "reuse" the selected element again 
            .css("border", "solid 1px black");              // cancels the "find()" method after this point in this situation
    // is()
        $("#demo_parent p").is("#demo");                // -> "true"        // the "demo" element is part of the selected group
    });
</script>

    the each() / map() methods

    - the each() method runs a function for each element in the jQuery object but cannot return anything (the original jQuery object is not modified)

    - the map() method runs a function for each element in the jQuery object and returns its values in a new jQuery object (the original jQuery object is not modified) 

    SYNTAX :    jQ_obj.each(fn(index, element))
            jQ_obj.map(fn(index, element))
                - index : current element's index
                - element : current element passed by the function
    
        <p class="demo"> myParagraph_A </p>
    <p class="demo"> myParagraph_B </p>
<script>
    $(document).ready(function(){
        var $Array = $([10, 20, 30]);                                   // array wrapped in jQuery object
    // each --------------------------    
        var eachTest = $Array.each(function(index, val){
            return val += 1;                                            // the "return" has no effect in the "each" method (this doesn't do anything here)
        });
        eachTest                                                        // -> $([10, 20, 30])       // the "each" method does NOT return the passed values therefore the passed jQuery object is inchanged  
        $Array                                                          // -> $([10, 20, 30])       // the original "$Array" is not modified
        
    // map ---------------------------    
        var mapTest = $Array.map(function(index, val){
            return val += 1; 
        });
        mapTest                                                         // -> $([11, 21, 31])       // the "map" method RETURNS the passed values therefore the passed jQuery object is updated 
        $Array                                                          // -> $([10, 20, 30])       // the original "$Array" is not modified
    });
</script>
