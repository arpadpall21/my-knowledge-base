Remember This :

- in MySQL there's no outer join, cross join can be used for this purpose

Description and Demonstration :

- 2 or more tables can be joined together based on a related column between them (a column containing the same data in both tables)

- the related colums don't necessarily have to be in PRIMARY KEY - FOREIGN KEY relationship (but it's common to use joins this way)

- in MySQL there's (INNER) JOIN, LEFT JOIN and RIGHT JOIN but there's no OUTER JOIN

    SELECT *|table.col, ... FROM leftTable

        
 - here we can specify all the columns from the final joined table what we want to return 

        
 Ex : 

        
 -> SELECT myTable1.col_1, myTable2.col_1 ... = the returned result will show 'col_1' from 'myTable1' and 'col_1' from 'myTable2' 

    
 JOIN rightTable ON leftTable.col=rightTable.col

        
 - columns in both tables on which the join is based 

        
 leftTable.col = column from the left (source) table 

        
 rightTable.col = column from the right (target) table 

    
 moreJoins...

        
 - additional joins (any join type JOIN | LEFT JOIN | RIGHT JOIN) (syntax after the JOIN keyword) 

        
 - multiple joins are chained from left to right 

        
 Example: 

        
 - we want to join 4 tables in this order table_1 -> table_2 -> table_3 -> table_4 

        
 - first table_1 and table_2 are joined and returned 

        
 - then the returned table is joined to table_3 and returned 

        
 - then finally the returned table is joined to table_4 and the final joined table is returned

 [WHERE condition]
        // joins 2 (or more) tables together based on a column in each tables containing the same data    
        // the returned table contains reccords where the compared columns match their data and the right table's records (right join)   
    
    
    SELECT *|table.col, ... FROM table [AS aliasA], table [AS aliasB]
        // (self join) the table is joined with itself (each record is combined with itself and with every other records)    
  
  
    SELECT *|tAlias.col, ... FROM table_1 CROSS JOIN table_2
        // joins table_1 and table_2 together
DEMO 
// inner join -------------------------------------------------------------------------
    mysql> SELECT * FROM myTable1                                                      // inner joins 'myTable1' and 'myTable2' based on columns 'myTable1.col1' and 'myTable2.col1'    
         > JOIN myTable2 ON myTable1.col1=myTable2.col1;                                 // the returned table contains records where values match between 'myTable1.col1' and 'myTable2.col1'
    
    mysql> SELECT myTable1.col1, myTable1.col2, myTable2.col2 FROM myTable1            // only specified columns are returned from the joined table   
         > JOIN myTable2 ON myTable1.col1=myTable2.col1;
    
// left join --------------------------------------------------------------------------
    mysql> SELECT * FROM myTable1                                                      // left joins 'myTable1' and 'myTable2' based on columns 'myTable1.col1' and 'myTable2.col1'    
         > LEFT JOIN myTable2 ON myTable1.col1=myTable2.col1;                            // the returned table contains records where values match between 'myTable1.col1' and 'myTable2.col1' and all 'myTable1' records (left table)    
    
// right join -------------------------------------------------------------------------
    mysql> SELECT * FROM myTable1                                                      // left joins 'myTable1' and 'myTable2' based on columns 'myTable1.col1' and 'myTable2.col1'    
         > RIGHT JOIN myTable2 ON myTable1.col1=myTable2.col1;                           // the returned table contains records where values match between 'myTable1.col1' and 'myTable2.col1' and all 'myTable2' records (right table)    
    
// cross join -------------------------------------------------------------------------
    mysql> SELECT * FROM myTable1 CROSS JOIN myTable2;                                 // joins myTable1 and myTable2 together 
    
    
// self join --------------------------------------------------------------------------
    mysql> SELECT * FROM myTable1 A, myTable2 B;                                       // the table 'myTable1' is joined with itself (each record is combined with itslef an with every other records)
                                                                                         // table aliases allow self join 
    
// multiple join (chain join) ---------------------------------------------------------
    mysql> SELECT * FROM myTable1 
         > JOIN myTable2 ON myTable1.col_1=myTable2.col1                               // first 'myTable1' and 'myTable2' are inner joined and returned 
         > JOIN myTable3 ON myTable1.col_1=myTable3.col1;                              // then the returned table is inner joined to 'myTable3' (and so on...)    
    
    mysql> SELECT * FROM myTable1 
         > LEFT JOIN myTable2 ON myTable1.col_1=myTable2.col1  
         > LEFT JOIN myTable3 ON myTable1.col_1=myTable3.col1;                         // the 'myTable1' will have all its records in the final join  
    
// table aliases ----------------------------------------------------------------------
    mysql> SELECT * FROM myTable1 AS a                                                 // table aliasses for ('myTable1' = A)('myTable2' = B)
         > JOIN myTable2 AS B ON A.col1=B.col2
    
    mysql> SELECT * FROM myTable1 A                                                    // does the same as above (without the 'AS' keyword)   
         > JOIN myTable2 B ON A.col1=B.col1; 
         
    mysql> SELECT A.col1, A.col2, B.col1 FROM myTable1 A                               // table aliases used when targeting table columns 
         > JOIN myTable2 B ON A.col1=B.col1;
