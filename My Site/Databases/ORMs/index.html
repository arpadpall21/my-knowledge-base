ORMs

MySite> Databases> ORMs

- ORM stands for Object-Relationoal Mapping, it's a technique for converting data between incompatible systems using object-oriented approach

- in ORM systems, usually each class (Model) maps to a table|document in the underlying database

- there are 2 coupling methos how ORMs are mapped to the database:

- Active Record pattern (easy and straightforward to use):

- the model is directly coupled, each model instance represent a row in the databse (there's no layer between the model and the database)

- each model has its own methods for CRUD operations with the database

- Data Mapper pattern (flexible):

- the model is losely coupled with the databse (there is a mapper layer between the model and the database)

- the model is separated from the database handling logic
