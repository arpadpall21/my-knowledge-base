Method / Property Description
path.win32 - returns a path that explicitly handles paths as windows path 15.3 (directly can be required as "path/win32")
path.posix - returns a path that explicitly handles paths as posix path 15.3 (directly can be required as "path/posix")
|
path.sep - returns the os path segment separator (windows = \ | posix = /)
path.delimiter - returns the os path-delimiter (windows = ; | posix = :)
|
path.dirname(path:str) - returns the directory path of the passed path
path.basename(path:str, suffix:str

- optional suffix to remove

) 		- returns the last path segment
path.extname(path:str) - returns the file extension
|
path.parse(path:str) - returns a pathObj:obj object whose properties represent the passed path's elements (object properties : dir / root / base / name / ext)
path.format(pathObj:obj

dir: str - if provided root ignored

root: str

base: str - if provided name and ext ignored

name: str

ext: str

) 		- from the passed pathObj:obj returns a path (pathObj:obj created by path.parse())
|
path.normalize(path:str) - normalizes (resolves) the .. character as parent directory and the . character as current directory (""(zero-length string) same as .)
- multiple path segment separators will be reduced to one platform specific (/ on POSIX and \ on windows)
- trailing separators are preserved
path.join(path:str, ...) - joins path segments together by using the platform specific path segment separator
- normalizes the joined path with the normalize() method
path.resolve(path:str, ...) - joins path segments together by resolving them into an absolute path (trailing path segment separators are removed)
- path segments are processed from left to right until an absolute path is constructed
- if no arguments are passed the current absolute path will be returned
- normalizes the returned absolute path with the normalize() method
|
path.relative(fromPath:str, toPath:str) - returns the relative path calculated from fromPath to toPath ("" = current directory)
path.isAbsolute(path:str) - returns true if the passed path is an absolute path, otherwise returns false
path.matchesGlob(path:str, globPattern:str) - returns true if passed path matches globPattern:str 23.10 (sitll experimental)

Description and Demonstration :

- the path module provides utilities for working with file and directory paths

- by default the os is detected, but we can use the path.posix VS path.win32 to specify the os path to handle 

    import path from 'node:path';
    
    path.basename('C:/Program Files/node/node.js');           // auto detects the os file system
    
    path.win32.basename('C:/Program Files/node/node.js');     // explicitly handles the passed path as windows path
    path.posix.basename('/Users/pall/node.js');               // explicitly handles the passed path as posix path

The win32 / posix / sep / delimiter properties

DEMO 
    const path = require('path');
    
// win32 / posix ----------------------------------------------------------------
    path.isAbsolute('d:\\Safe\\CODE +\\My Site\\');                              // -> true         // detected platform = windows 
    path.win32.isAbsolute('d:\\Safe\\CODE +\\My Site\\');                        // -> true         // this is an absolute path on windows 
    path.posix.isAbsolute('d:\\Safe\\CODE +\\My Site\\');                        // -> false        // not an absolute path on posix (because the path is windows specific)   
    
// sep / delimiter --------------------------------------------------------------
    path.win32.sep;                                                              // -> '\'          // windows path segment separator 
    path.posix.sep;                                                              // -> '/'          // posix path segment separator   
    
    path.win32.delimiter                                                         // -> ';'          // windows path delimiter 
    path.posix.delimiter                                                         // -> ':'          // posix path delimiter

The dirname() / basename() / extname() methods

DEMO 
    const path = require('path');
    
// dirname() -------------------------------------------------------------------- 
    path.win32.dirname('C:\\Program Files\\node\\node.js');                      // -> 'C:\Program Files\node'          // directy path of the passed file path returned    
    path.win32.dirname('C:\\Program Files\\node\\');                             // -> 'C:\Program Files'               // -! not a file path = previous directory path returned 
    
    path.posix.dirname('/dev/node/node.js');                                     // -> '/dev/node'
    path.posix.dirname('/dev/node/npm/color_module/bin');                        // -> '/dev/node/npm/color_module'     // -! not a file path = previous directory path returned 
    
// basename() -------------------------------------------------------------------
    path.win32.basename('C:\\Program Files\\nodeFolder\\node.js');               // -> 'node.js'                        // returns the last portion of the path  
    path.win32.basename('C:\\Program Files\\nodeFolder');                        // -> 'nodeFolder'
    path.win32.basename('C:\\Program Files\\nodeFolder\\node.js', '.js');        // -> 'node'                           // removes the extension of the returned result 
    path.win32.basename('C:\\Program Files\\nodeFolder', 'er');                  // -> 'nodeFold'                       // the 2nd argument is basically removed from the end of the returned result   
    
    path.posix.basename('/dev/nodeFolder/node.js');                              // -> 'node.js'
    path.posix.basename('/dev/nodeFolder');                                      // -> 'nodeFolder'
    path.posix.basename('/dev/nodeFolder/node.js', '.js');                       // -> 'node'
    path.posix.basename('/dev/nodeFolder', 'deFolder');                          // -> 'no'                             // the 2nd argument is basically removed from the end of the returned result   
    
// extname() --------------------------------------------------------------------
    path.win32.extname('C:/Program Files/nodeFolder/node.js');                   // -> '.js'                            // returns the file extension 
    path.win32.extname('C:/Program.Files/node.Folder');                          // -> '.Folder'                        // basically returns the string portion after the '.'(period) character in the last portion of the path    
    
    path.posix.extname('dev/nodeFolder/node.js');                                // -> '.js'
    path.posix.extname('dev/Program.Files');                                     // -> '.Files'

The parse() / format() methods

DEMO 
    const path = require('path');
    
// parse() ----------------------------------------------------------------------
    path.win32.parse('D:\\serv\\html\\index.html')                               // -> {root:'D:\\', dir:'D:\\serv\\html', base:'index.html', ext:'.html', name:'index'}
    path.posix.parse('/srv/html/index.html');                                    // -> {root:'/', dir:'/srv/html', base:'index.html', ext:'.html', name:'index'}
    
// format() ---------------------------------------------------------------------
    path.win32.format({dir:'D:\\serv\\html',                                     // -> 'D:\serv\html\index.html'
                       root:'D:\\', 
                       base:'index.html', 
                       name:'index', 
                       ext:'.html'});
    
    path.posix.format({dir:'/serv/html',                                         // -> '/serv/html/index.html'      // -! the 'dir' property overrides the 'root' property 
                       root:'\\',                                                                                   // this is ignored 
                       base:'index.html',                                                                           // -! the 'base' property overrides the 'name' and 'ext' properties   
                       name:'something other',                                                                      // this is ignored
                       ext:'.css'})                                                                                 // this is ignored

The normalize() / join() / resolve() methods

- join() simply joins path segments together VS resolve() resolves path segments into an absolute path

DEMO 
    const path = require('path');
    
// normalize() ------------------------------------------------------------------
    path.win32.normalize('C:\\Program Files\\Call of Duty\\..');                 // -> 'C:\Program Files'                           // normalized to the parent directory 
    path.win32.normalize('C:\\Program Files\\Call of Duty\\.');                  // -> 'C:\Program Files\Call of Duty'              // normalized to the current directory  
    path.win32.normalize('C:////Program Files\\\\Call of Duty///');              // -> 'C:\Program Files\Call of Duty\'             // multiple path segment separators are reduces (normalized) to one platform specific and trailing allowed   
    
    path.posix.normalize('dev/appliatons/Call of Duty/..');                      // -> 'dev/appliatons'                             // normalized to the parent directory 
    path.posix.normalize('dev/appliatons/Call of Duty/.');                       // -> 'dev/appliatons/Call of Duty'                // normalized to the current directory  
    path.posix.normalize('dev///applications/////Call of Duty//');               // -> 'dev/applications/Call of Duty/'             // multiple path segment separators are reduces (normalized) to one platform specific and trailing allowed   
    
// join() -----------------------------------------------------------------------
    path.win32.join('/D:', '//Program Files\\\\', 'Call of Duty', 'run.exe//');  // -> '\D:\Program Files\Call of Duty\run.exe\'    // joins path segments together then uses the 'normalize()' method on the final result 
    
    path.posix.join('/dev', '//Program Files////', 'Call of Duty', 'run.exe//'); // -> '/dev/Program Files/Call of Duty/run.exe/'   // joins path segments together then uses the 'normalize()' method on the final result 
    
// resolve() --------------------------------------------------------------------
    path.win32.resolve('\\', 'Program File', 'Call of Duty', 'run.exe');         // -> 'D:\Program File\Call of Duty\run.exe'       // resolves to an absolute path then uses the 'normalize' method on the final result 
    path.win32.resolve('C:', 'Program File\\', 'Call of Duty', 'run.exe//');     // -> 'C:\Program File\Call of Duty\run.exe'       // no segment separator trailing (obviously this is an absolute path)   
    path.win32.resolve();                                                        // -> 'current absolute path'                      // if no arguments are passed the current absolute path will be returned  
    
    path.posix.resolve('/srv', '/dev', 'app', 'Call of Duty', 'run.exe');        // -> '/dev/app/Call of Duty/run.exe'              // -! on posix the last path segment starting with the '/'(backslash) character will be the root 
    path.posix.resolve('/srv', '/dev', 'app', 'Call of Duty', 'run.exe////');    // -> '/dev/app/Call of Duty/run.exe'              // no segment separator trailing (obviously this is an absolute path)   
    path.posix.resolve();                                                        // -> 'current absolute path'                      // if no arguments are passed the current absolute path will be returned

The relative() / isAbsolute() / matchesGlob() methods

DEMO 
    const path = require('path');
    
// relative() ------------------------------------------------------------------
    path.win32.relative('C:\\Program Files\\Call of Duty', 'C:\\');              // -> '../..'                       // returns the relative path between the passed paths assuming the 1st argument as current directory   
    path.win32.relative('C:\\Program Files', 'C:\\Program Files\\Call of Duty\\mode');  // -> 'Call of Duty\\mode'
    path.win32relative('', 'C:\\');                                              // -> '..'                          // the ''(zero-length string) is assumed as current directory (we are in 'C:\\windows')   
    path.win32.relative('C:\\Program Files', 'C:\\server');                      // -> '..\server'                   // -! navigates back to the root directory then to the target path 
    path.win32.relative('C:\\Program Files', 'D:\\server');                      // -> 'D:\\server'                  // -! because the root elements are different the target path is returned 
        
    path.posix.relative('/dev/app', '/');                                        // -> '../../'                      // returns the relative path between the passed paths assuming the 1st argument as current directory   
    path.posix.relative('/dev/app', '/dev/app/Mario/level1');                    // -> 'Mario/level1'
    path.posix.relative('', '/');                                                // -> '../'                         // the ''(zero-length string) is assumed as current directory (we are in '/srv')   
    path.posix.relative('/dev/some folder', '/srv');                             // -> '../../srv'                   // -! navigates back to the root directory then to the target path 
    
// isAbsolute() -----------------------------------------------------------------
    path.win32.isAbsolute('C:\\Program Files\\Call of Duty');                    // -> true             
    path.win32.isAbsolute('Program Files\\Call of Duty');                        // -> false
                
    path.posix.isAbsolute('/dev/Call of Duty');                                  // -> true
    path.posix.isAbsolute('dev/Call of Duty');                                   // -> false 
    
// matchesGlob() -----------------------------------------------------------
    import path from 'node:path/posix';
    
    path.matchesGlob('/foo/bar', '/foo/*');                                 // -> true
    path.matchesGlob('/foo/bar', '/**/bar');                                // -> true
    path.matchesGlob('/foo/bar/main.mjs', '/**/*.mjs');                     // -> true
