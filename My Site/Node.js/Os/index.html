Method / Property Description
os.constant - returns the os module available constants 6.3.0
|
os.arch() - retunrs the CPU architecture (same as process.arch)
os.cpus() - returns informations about each logical CPUs in an Array
os.endianness() - returns the endianness of the CPU (for which the Node.js binary was compiled) (endianness = order of bytes to represent a number, BE (big endianness) = most significant bytes first | LE = less significant bytes first)
os.totalmem() - returns the total amount of system memory in bytes
os.freemem() - returns the free system memory in bytes
|
os.platform() - identifies the operating system (same as process.platform)
os.type() - returns the operating system name in a string
os.release() - returns the operating system release informations in a string
os.hostname() - returns the operating system name in a string
os.tmpdir() - returns the default temporary files directory of the system in a string
os.homedir() - returns the home directory of the current user
|
os.EOL - returns the Operating System's end-of-line marker
os.devNull - platform specific file path to null device 14.18
os.uptime() - returns the system uptime in seconds (time from the system started)
os.userInfo({ encoding:encode:str }) - returns an object which contains current user informations 6.0.0
os.networkInterfaces() - returns active network interfaces in an object
os.loadavg() - returns an Array containing the loading averages in the last 1, 5 and 15 seconds

- each number indicates the average number of processes waiting to be executed in the last 1, 5 and 15 minutes

- normal values = below the number of processors the system have

- system overload = above the number of processors the system have

Example of a 12 core system

- 0.25, 0.27, 0.33 = normal values

- 17.11, 14.55, 13.02 = system overloaded

(UNIX only on windows is always [0, 0, 0])
os.availableParallelism() - returns an estion of how many parallelism a program should use 18.14
|
os.setPriority(pid:nr, priority:nr) - attempts to set the scheduling priority to the targeted process (any process on the system) (use the os.constants.priority property in order to get the available priority values of the system) 10.10.0
os.getPriority(pid:nr) - returns the scheduling priority of the targeted process 10.10.0

Description and Demonstration :

- the os module provides a number of operating system related utility methods

The arch() / cpus() / endianness() / totalmem() / freemem() methods

DEMO 
    const os = require('os');
    
// cpus() / arch() / endianness() -----------------------------------------------
    os.cpus();                                                                   // -> [ list of cpus with details... ]         // logical CPUs of the system 
    os.arch();                                                                   // -> 'x64'                                    // CPU architecture
    os.endianness();                                                             // -> 'LE'                                     // CPU uses little endianness
        
// totalmem() / freemem() -------------------------------------------------------
    os.totalmem();                                                               // -> 17129938944                              // total memory of the system (in bytes)   
    os.freemem();                                                                // -> 11014127616                              // free memory of the system (in bytes)

The platform() / type() / release() / hostname() / tmpdir() / homedir() methods

DEMO 
    const os = require('os');
    
// platform() / type() / release() / hostname() ---------------------------------
    os.platform();                                                               // -> 'win32'                              // platform id
    os.type();                                                                   // -> 'Windows_NT'                         // sytem type
    os.release();                                                                // -> '10.0.17763'                         // release info
    os.hostname();                                                               // -> 'Faszagyerek'                        // name of the system 
    
// tmpdir() / homedir() ---------------------------------------------------------
    os.tmpdir();                                                                 // -> 'C:\Users\Pall\AppData\Local\Temp'   // temporary files directory
    os.homedir();                                                                // -> 'C:\Users\Pall'                      // user home directory

The constants / loadavg() / EOL / uptime() / userInfo() / networkInterfaces() methods and properties

DEMO 
    const os = require('os');
    
// userInfo() / EOL / uptime() --------------------------------------------------
    os.userInfo();                                                               // -> { User informations... }     // current user informations 
    os.EOL;                                                                      // -> '\r\n'                       // end-of-line markers of the system 
    os.uptime();                                                                 // -> 4237                         // time in seconds since the system is on 
    
// constants / loadavg() --------------------------------------------------------
    os.constants;                                                                // -> { OS Constants... }          // Operating System constants   
    os.loadavg();                                                                // -> [0, 0, 0]                    // loading averages (on windows this is always [0, 0, 0])
    
// networkInterfaces() ----------------------------------------------------------
    os.networkInterfaces()                                                       // -> { active networks... }       // retunrs the active networks and their details

The setPriority() / getPriority() methods

DEMO 
    const os = require('os');
    
    os.constants.priority;                                                       // -> available priority values of the system      // this returns the available priority values of the system    
    
    os.setPriority(9680, -7);                                                    // sets the scheduling priority to the tageted process  // -! the 9680 PID is some other program's id (not necessarily a Node process)   
    
    os.getPriority(9680);                                                        // -> -7                                           // get scheduling priority of the targeted pid 
    os.getPriority(process.pid);                                                 // -> 0                                            // current process priority (Default value)
