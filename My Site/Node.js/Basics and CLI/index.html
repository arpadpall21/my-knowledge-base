--no-warnings nodeFile.js - silences all warnings (including deprecation warnings) 6.0.0
--print | -p jsExpression - evaluates the following argument as JavaScript expression and prints the result (throws error if any) 0.6.4
--help | -h - print the available node command line options and Environment Variables 0.1.3
--insecure-http-parser nodeFile.js - accepts invalid HTTP headers (Security risk!) [TESTED: 2020-04-29] Not working custom HTTP headers accepted anyways! 13.4.0
--secure-heap=nr

- the passed value must be a power of 2

- initializes an OpenSSL secure heap

OpenSSL's secure heap prevents sensitive data from leaking due to pointer overruns or underruns

of the specified size (in bytes) (must be at least 2)
- fixed size and cannot be resized at runtime
- the OpenSSL secure heap is disabled by default 15.6.0
--secure-heap-min=nr

- the passed value must be a power of 2

- (when --secure-heap used) specifies the minimum allocation from the secure OpenSSL secure heap 15.6.0
--title=titleName nodeFile.js - sets the process.title on startup 10.7.0
--tls-cipher-list="cipherSuite:..." nodeFile.js - specifies an alternative default TLS cipher suite list (overrides the default node cipher suite list) 4.0.0
--v8-options - prints the available V8 command line options 0.1.3
--version | -v - returns the node.js version number 0.1.3
--zero-fill-buffers nodeFile.js - zero fills newly allocated buffers [TESTED -> link] 6.0.0

Node.js program structure

- in Node.js the global scope is not designed to run tasks on it, instead we use modules what have their own scopes and can communicate with each other if we want to

- when we run a .js file we are in that scope (module)

- the require() method which basically links other modules to the same global scope we are running

- the global object is the global object (conviniently), it is available from anywhere

- module structure (each module looks like this) 

SYNTAX :    global                                                                     // references the global object (scope), accessible anywhere in Node.js (contains all JavaScript constructors)
DEMO 
// main.js (file) ------------------------------------------------------------------------------------------------------------------
    // access global object's default properties --------------------------------------
    this                                                                               // -> {}                    // does not references the global scope because we are in the 'main.js' module here 
    global.Object;                                                                     // -> [Function: Object]    // we can reference any properties form the global object 
    Object;                                                                            // -> [Function: Object]    // we don't have to call the 'object' object if we want to access a property of the global object 
    
    // module's object ----------------------------------------------------------------
    __filename;                                                                        // -> (path and file name)  // object belongs to this module 
    global.__filename;                                                                 // -> undefined             // not a global object 
    
    // declare properties in the global scope -----------------------------------------
    require('./module1.js');                                                           // the 'module1.js' module will have the same global object (scope) as this module 
    
    global.var1 = 'value1';                                                            // declare a global property (not a recommended practice)
    var2 = 'value2';                                                                   // this does the same       // -! in strict mode this is not allowed
    var localVar = 'locale value';                                                     // local variable 
    
    // call global properties ---------------------------------------------------------
    global.var1;                                                                       // -> 'value1'              // global object property 
    global.var2;                                                                       // -> 'value2'              // same as 'global.var2'
    global.var3;                                                                       // -> 'value3'              // global object property declared in 'module1.js' 
    global.var4;                                                                       // -> RefferenceError       // -! because we did not called the 'required()' method on the 'module2.js' so it does not have the same global object    
    
    var1;                                                                              // -> 'value1'              // same as 'global.var1'
    localVar;                                                                          // -> 'locale value'        // !! local properties are called the same way as the global properties can be called careful! 
    globa.localVar;                                                                    // -> undefined 
    
// module1.js (file) ---------------------------------------------------------------------------------------------------------------
    global.var3 = 'value3';                                                            // global property declared 
    
// module2.js (file) ---------------------------------------------------------------------------------------------------------------
    global.var4 = 'value4';                                                            // global property declared

    require('./module1.js');                                        // we required the first module (this way 'module1.js' have the same global object at this module)
    
    global.mainVar1 = "main variable one";                          // variable declared on the global scope
    mainVar2 = "main variable two";                                 // this is a global variable (not recommended and in strict mode it will fail)
        
    var localVar = 'local variable';                                // local variable accessbile anywhere from this module

      // global variables declared in this module ----
